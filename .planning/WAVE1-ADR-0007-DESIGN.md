## Calibration Gate Logic

### Gate Function: `validarCalibracaoEquipamento(labId, equipId)`

**Blocking Rules:**
1. If `proximaCalibracaoPrevista` is in the past → **BLOCK all CIQ runs on this equipment**
2. If `status` = 'quebrado' → **BLOCK immediately**
3. If `status` = 'em_manutencao' → **BLOCK** (can mark maintenance window)
4. If no calibration history → **BLOCK** (unqualified equipment)

**Return:**
```typescript
{
  allowed: boolean;
  reason?: string;  // e.g., "Calibração vencida em 2026-05-01"
  dataVencimento?: Timestamp;
}
```

**Integration:** Every CIQ module (hematologia, imunologia, etc.) must call this gate **before saving run**.

## 5 CIQ Module Integration

Each module adds calibration validation before **run save**:

| Module | Check | On Block | On Success |
|--------|-------|----------|-----------|
| **Hematologia** | validarCalibracaoEquipamento(labId, equipId) | Throw error | Save run + denormalize equipamentoId |
| **Imunologia** | Same | Same | Same |
| **Coagulacao** | Same | Same | Same |
| **Uroanalise** | Same | Same | Same |
| **Bioquimica** | Same | Same | Same |

**Denormalization in Run Doc:**
```typescript
interface HematologiaRun {
  // ... existing fields ...
  equipamentoId: string;
  equipamento?: {
    nome: string;
    numeroSerie: string;
    ultimaCalibracaoData: Timestamp;
  };
}
```

This is **immutable after creation** — links run to exact equipment + calibration state at time of test.

## Refactor: Calibração Module

**Current State:**
- Calibração data scattered in ControleQualidade or per-module
- No central equipamento record
- Manual tracking of dates

**New State (ADR 0007):**
- `/labs/{labId}/equipamentos` — single source of truth
- `/labs/{labId}/equipamentos/{equipId}/calibracoes` — immutable audit trail
- `/labs/{labId}/equipamentos/{equipId}/manutencoes` — maintenance history
- Cloud Function: `agendarProximaCalibracacao()` — RT-only, sets `proximaCalibracaoPrevista`
- Scheduled validator: Check daily for overdue calibrations → create NC automatically if overdue

## NC Auto-Creation on Overdue Calibration

**Scheduled Function (runs every 6 hours):**
```
checkOverdueCalibracoes():
  - Query all equipamentos where proximaCalibracaoPrevista < now()
  - For each overdue:
    - Check if NC already open for this equipamento
    - If not: openNaoConformidade({
        origem: 'equipamento',
        origemId: equipId,
        descricao: `Calibração vencida desde ${dataVencimento}`,
        severidade: 'grave',  // Blocks operations
        bloqueiaOperacoes: true
      })
    - Log: "Overdue calibration detected for {equipId}, NC opened {ncId}"
```

## Backfill Strategy (Equipamentos Legados)

### Current State
- Equipamentos registrados em ControleQualidade.setups ou inlined
- Sem data de calibração centralizada
- Sem manutenção tracked
- Sem fornecedor qualificado

### Migration Plan

**Script:** `functions/scripts/backfill-equipamentos.mjs`

```javascript
// 1. Query all legacy setups from ControleQualidade
// 2. For each setup:
//    - Create Equipamento doc in /equipamentos
//    - Extract calibration date if available
//    - Set proximaCalibracaoPrevista = dataCalibracacao + 12 meses (default)
//    - Set status = 'ativo' (assume all legacy are active)
//    - Map fornecedorId from legacy data or mark as 'unknown'
//    - HMAC-sign via ADR 0005
//    - Create initial Calibracao entry (denormalized from legacy)
// 3. Update all CIQ runs to reference new equipId
// 4. Validate 1:1 mapping (count before = count after)
// 5. Report: migrated count, errors, duration
```

**Safety Measures:**
- Dry-run first on test lab
- Verify 1:1 mapping
- Keep old setups as reference (_migrated: true)
- Halt if >5% anomalies detected
- Full audit trail (who, when, count, hash)

## Firestore Rules (ADR 0007 Patch)

```firestore
match /labs/{labId}/equipamentos/{equipId} {
  // Read: anyone can view equipamentos
  allow read: if request.auth.uid != null;
  
  // Create: admin/RT can create equipamentos
  allow create: if request.auth.uid != null &&
                   (request.auth.token.admin == true ||
                    request.auth.token.responsavelTecnico == true);
  
  // Update: RT only, but only certain fields
  allow update: if request.auth.uid != null &&
                   request.auth.token.responsavelTecnico == true &&
                   (request.resource.data.proximaCalibracaoPrevista != null ||
                    request.resource.data.proximaManutenccaoPrevista != null ||
                    request.resource.data.status != null ||
                    request.resource.data.hmac != null);
  
  // Calibrações subcollection (immutable after signature)
  match /calibracoes/{calibId} {
    allow read: if request.auth.uid != null;
    
    allow create: if request.auth.uid != null &&
                     request.resource.data.dataRealizacao != null &&
                     request.resource.data.certificado_url != null;
    
    allow update: if false;  // immutable
  }
  
  // Manutenções subcollection
  match /manutencoes/{manutId} {
    allow read: if request.auth.uid != null;
    
    allow create: if request.auth.uid != null &&
                     request.resource.data.dataRealizacao != null &&
                     request.resource.data.tipo != null;
    
    allow update: if false;  // immutable
  }
}
```

## Cloud Functions (ADR 0007)

**1. agendarProximaCalibracacao() — callable**
```
Input: { labId, equipId, dataProxima, observacoes }
Requires: responsavelTecnico role
Output: { success, proximaData, proximoId }

Process:
- Update equipamento.proximaCalibracaoPrevista = dataProxima
- Create Calibracao entry with status='agendada'
- HMAC-sign
- Return scheduled date
```

**2. registrarCalibracacao() — callable**
```
Input: { labId, equipId, dataRealizacao, certificado_url, fornecedorId, status }
Requires: responsavelTecnico role
Output: { success, calibracaoId, proximaDataCalibracao }

Process:
- Create immutable Calibracao subcollection entry
- Update equipamento.ultimaCalibracaoData
- Calculate proximaCalibracaoPrevista (now + standard interval, default 12m)
- HMAC-sign
- Return next calibration date
```

**3. registrarManutencao() — callable**
```
Input: { labId, equipId, tipo, descricao, peças_substituidas[], custo_total, dataProxima }
Requires: responsavelTecnico role
Output: { success, manutencaoId }

Process:
- Create immutable Manutenção entry
- Update equipamento.ultimaManutenccaoData
- Set equipamento.proximaManutenccaoPrevista = dataProxima
- HMAC-sign
- Return maintenance record
```

**4. validarCalibracaoEquipamento() — internal/callable**
```
Input: { labId, equipId }
Output: { allowed: boolean, reason?: string }

Process:
- Query equipamento
- Check if proximaCalibracaoPrevista < now()
- Check status
- Return allowed=false + reason if blocked
```

## 6-Module Integration (CIQ Module Updates)

Each module adds gate check + denormalization:

```typescript
// Before save:
const calibracaoValid = await validarCalibracaoEquipamento(labId, equipId);
if (!calibracaoValid.allowed) {
  throw new HttpsError('failed-precondition', calibracaoValid.reason);
}

// On success, denormalize:
run.equipamentoId = equipId;
run.equipamento = {
  nome: equipment.nome,
  numeroSerie: equipment.numeroSerie,
  ultimaCalibracaoData: equipment.ultimaCalibracaoData,
};
```
