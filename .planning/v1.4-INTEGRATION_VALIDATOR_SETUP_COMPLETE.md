## Escalation Protocol (Ready)

### SLA by Severity

| Severity                                                                | SLA    | Route                                    | CTO              |
| ----------------------------------------------------------------------- | ------ | ---------------------------------------- | ---------------- |
| 🔴 CRITICAL (schema conflict, auditor blocker, security finding)        | 2h     | Wave lead → CTO → Escalation meeting     | YES              |
| 🟡 MEDIUM (callable signature change, test failure, NOTIVISA queue lag) | 24h    | Wave lead → Validator → Wave coordinator | If schedule risk |
| 🟢 LOW (doc update, minor test tweak, performance optimization)         | 1 week | Wave lead → Validator log                | NO               |

### Routing Example

**Scenario: Phase 4 changes `signLaudo` callable signature (breaks Phase 6 PDF)**

1. Validator detects change (daily schema diff check)
2. Escalation created: `BLOCK-20260522-01: signLaudo signature regression`
3. Notification: `#hc-quality-integration` Slack + Wave 1 lead + CTO
4. SLA: Resolve in 2h (🔴 CRITICAL — blocks Phase 6)
5. Resolution options:
   - Option A: Revert Phase 4 change, use original signature
   - Option B: Update Phase 6 PDF callable to match new signature + re-test Scenario 5
   - CTO decision → Implementation + re-test → Scenario 5 E2E ✅ PASS
