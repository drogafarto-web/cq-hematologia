# v1.4 Research Summary — Strategic Foundations

**Period:** 2026-05-06 (research) → v1.4 planning
**Scope:** DICQ compliance gaps, RDC 2025 regulatory landscape, tech stack review, competitive analysis, high-ROI feature opportunities

---

## Executive Summary

v1.3 delivered **25 modules in production** with DICQ compliance climbing from **71.3% → ~78–82%** (honest baseline), closing RDC 978/2025 gaps in CIQ, CEQ, EC, and Gestão Documental. v1.4 has three strategic vectors:

1. **Compliance-2 Phase** (60–120 days) — push DICQ from ~80% → **85%+** by completing Blocks A, C, D, H (governance, personnel, risks, feedback-loop)
2. **Regulatory Hardening** — RDC 986/2025 clarifications + LGPD formalization + audit-ready reporting
3. **Market Differentiation** — OCR imuno (roadmap north star), NOTIVISA integration, multi-equipamento hematologia, mobile-first bancada

**Tech Stack Assessment:** Current stack (React 19, TS 5.8, Vite 6, Zustand 5, Firebase 12, Gemini 2.5 Flash, Node 22) is solid. **No forced upgrades.** Firebase v1→v2 migration complete in v1.3; Node 22 on LTS through 2027. Risk vectors are external (Gemini API latency, Firestore analytical query performance, Tailwind CDN removal is done).

**Market Position:** HC Quality is **category-defining** for small/medium lab quality SaaS (RDC compliance + OCR diff). Competitive set is fragmented (WinLab/Labcat are LIS, not CIQ-focused; PNCQ Gestor is desktop). Gap: lack of **audit-ready dashboards** (5-minute overview for health inspector) and **NOTIVISA automation**.

**v1.4 Recommendation:** Ship **Compliance-2 Phase** (mandatory foundation) + **audit-ready reporting** (market enabler) + **OCR imuno pilot** (north-star differentiator). Defer multi-equipamento + NOTIVISA to v1.5 unless customer demands unlock funding.

### RDC 986/2025 (August 2025 alteradora) — New in May 2026 Operating Window

**Alteradora published 15/08/2025, vigente immediately.** Riopomba has been operating under both 978 + 986 since Aug 15.

**Key clarifications affecting HC Quality:**

1. **Profissional capacitado** now formally defined — system must allow filtering by formal education level (habilitado) vs training (capacitado) in personnel roster. ✅ **Recommend:** Add `profissionalType` enum to personnel module (Habilitado | Capacitado | Estagiário).

2. **Validação** formal definition — any "novo método" must have validation records. ✅ **Recommend:** Audit existing `validacao-metodos` module (if any); if not, build template.

3. **Providores** (controle vs proficiência) now separated in law. ✅ **Recommend:** Audit CEQ module to ensure controle vendor ≠ provedor proficiência are distinct entities.

**Risk vector:** If audit asks "mostre validação de todos os novos métodos usados aqui", system must have proof. **Spot-check required in v1.4 pre-deploy audit.**

## Section 5: v1.4 Strategic Opportunities

### High-ROI Feature Map

Ranked by **compliance impact × market differentiation × effort**.

#### 🎯 Tier 1: Compliance Mandates (must-ship)

**A1. Supervisor Shift Registry (RDC 122 + audit accountability)**

- **What:** UI for RT/supervisor to clock in/out by shift; audit report per auditor query
- **Why:** RDC 978 requires supervisor present during operation. Auditor asks "quem era supervisor em 2026-05-06 14:00-22:00?" System must answer in <5 min.
- **Components:** Personnel shift UI + Cloud Function trigger log + reporting CF
- **Effort:** 1–2 weeks
- **Impact:** RDC compliance 100% (Art. 122)
- **Market:** Audit-ready checkbox; required for any regulated lab

**A2. LGPD Formal Policy Document in SGD**

- **What:** `lgpd-policy` Documento in SGD, versioned, signed by RT + counsel
- **Why:** Art. 77 (RDC 978) explicitly requires LGPD documentada. Audit asks "mostre política LGPD". System must have it in governance folder.
- **Components:** Policy template + SGD doc versioning + assinatura RT
- **Effort:** 1 week
- **Impact:** RDC Art. 77 compliance 100%
- **Market:** Table stake for any customer audit

**A3. Lab Apoio Contracts Module**

- **What:** Formalize supplier types: insumo-vendor (NF only) vs lab-apoio (contrato Art. 82 obrigatório)
- **Why:** RDC Arts. 36–39 require written contracts with 6 minimum clauses. Current system has generic "fornecedores"; need to differentiate.
- **Components:** Supplier type enum, contract template + versioning, cláusula checklist
- **Effort:** 2 weeks
- **Impact:** RDC Arts. 36–39 compliance 100%
- **Market:** Audit-ready; no competitor has this formalized

#### 🚀 Tier 2: DICQ Compliance Blockers (push to 85%+)

**B1. Risk Management Module (DICQ 4.14.6 + RDC 107–109)**

- **What:** FMEA-lite tool (risk matrix 3×3, probability×impact, controls, monitor)
- **Why:** DICQ 4.14.6 requires formal risk plan. RDC 107–109 ask for risk identification. Current system has none.
- **Components:** Risk catalog seed (30 lab common risks), matrix builder, control checklist, annual review workflow
- **Effort:** 3–4 weeks (template + UI + CF aggregation)
- **Impact:** +6–8 DICQ points (Bloco D 85%)
- **Market:** Differentiator vs PNCQ Gestor (which lacks risk mgmt)

**B2. Internal Audit Cycle (DICQ 4.14.5)**

- **What:** Annual audit program with monthly/quarterly checklists, nonconformities linked, follow-up tracking

## Appendix: Glossary & Definitions

- **DICQ** — Diretrizes para Acreditação em Análises Clínicas (SBAC, 8ª ed., based on ISO 15189:2015)
- **RDC 978/2025** — Regulamento federal ANVISA para EAC (Exames de Análises Clínicas)
- **RDC 986/2025** — Alteradora of RDC 978, published 15/08/2025
- **STIII** — Serviço Tipo III (laboratório clínico, com direito a análises autorais)
- **PGQ** — Programa de Garantia da Qualidade (RDC 86)
- **SGD** — Sistema de Gestão Documental
- **EC** — Escrita Controlada (audit chain via LogicalSignature)
- **NC** — Não-conformidade
- **CAPA** — Corrective + Preventive Actions
- **LGPD** — Lei Geral de Proteção de Dados (Brazil's GDPR)
- **NOTIVISA** — Sistema de Notificação de Eventos Adversos (MS health authority)
- **CFU** — Cloud Function Unit (billable invocation)
- **TTI** — Time to Interactive (Core Web Vital)
