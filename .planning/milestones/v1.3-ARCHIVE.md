## Lessons Learned

1. **Functions API migration debt** — v1→v2 should have been Phase 0 dependency. Pre-migration check recommended for v1.4.
2. **Phase parallelism** — works at module level, but cross-cutting concerns need dedicated owner. Bias toward `shared/*` duplication.
3. **Deferral honesty** — ship 70% well-tested vs 100% untested. Phase 10/11 deferrals were the right call.
4. **Riopomba as template** — validates multi-tenant SGD approach for Mercês, Tabuleiro next labs.
