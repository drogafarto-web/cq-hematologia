## Section 8: Resolution Notes (Detailed)

### Why REQUIREMENTS.md said "Phase 1, 2, 3"

REQUIREMENTS.md was written **before** ROADMAP.md decomposed the work into 15 phases. At time of writing, the author used "Phase 1/2/3" to mean **"Wave 1 (early MVP)"**, **"Wave 2 (mid MVP)"**, **"Wave 3 (late MVP / stretch)"** — coarse buckets.

When ROADMAP.md later defined Phases 1–15 as detailed execution units, the labels collided. AUDIT-REPORT.md flagged this as 13 mismatches.

### Why ROADMAP.md is authoritative

1. **More detail:** 15 phases vs 3 buckets. Each phase has explicit success criteria, owner, duration, dependencies.
2. **Dependency graph:** ROADMAP encodes critical path (Phase 3 → Phase 4 → Phase 8 etc.). REQUIREMENTS doesn't.
3. **Execution-ready:** Stream A–D allocations, weekly cadence, artifact list per phase.
4. **VALIDATOR confirmed:** AUDIT-REPORT Section 1.1 says "Phase assignments are incoherent" and recommends reconciliation **toward ROADMAP phase numbers**.

### What this means for execution

- When a developer asks "which phase owns REQ-403?", the answer is **Phase 9**, NOT Phase 1.
- When a planner asks "what closes in Phase 4?", the answer is **REQ-412 + REQ-413 (setup)**, NOT REQ-403/405.
- This matrix is the **single source of truth** for the question "which REQ → which phase".

### Open Questions (Forwarded to CTO)

1. **REQ-401 (KPI Dashboarding):** Currently mapped to Phase 12 (perf audit). Should it instead get a dedicated mid-cycle phase (e.g., Phase 9.5) given its Stream D ownership? — VALIDATOR flagged this as ambiguous.
2. **REQ-403 (Personnel Dossier):** Should there be a Phase 1 "skeleton kickoff" to start data migration early? Currently waits until Phase 9. AUDIT-REPORT R1 risk: HR data migration is risky — earlier start might help.
3. **REQ-411 + TD-401/402 concurrent:** Is Phase 1 (Wk 1–2) realistic for 4 concurrent compliance + DevOps tasks? AUDIT-REPORT flagged Phase 1 as "aggressive (3–4 days)". May need to extend to 5–7 days.

These questions do NOT block execution; this matrix locks the assignment for now and flags them for CTO confirmation in Phase 2 planning kickoff.
