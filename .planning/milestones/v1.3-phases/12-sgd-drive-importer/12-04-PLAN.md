<tasks>

### Task 1: Setup staging environment (0.5 dia)
- Confirmar staging project Firebase ativo
- Limpar dados anteriores (qualquer test)
- Configurar OAuth credentials staging-only

**Acceptance:** Staging clean; OAuth funciona staging.

### Task 2: Selecionar 30 docs piloto (0.5 dia)
- Critério de seleção:
  - MQ-001 (1) — manual canônico
  - PQ-01 a PQ-25 (até 24 PQs prioritários — pular numerados fora de sequência) — core procedimentos
  - 5 ITs principais (rastreabilidade, coleta, limpeza, higiene, equipamentos)
- Documentar lista em PILOT-IMPORT-LOG.md

**Acceptance:** Lista aprovada pelo CTO + RT.

### Task 3: Executar import via wizard (Plan 12-03) (1 dia)
- RT acessa `/sgq/importar-drive` em staging
- 5 steps wizard até Confirm
- Captura screenshots de cada step
- Tempo total medido (target <30min RT)

**Acceptance:** 30 docs criados como `draft` em staging.

### Task 4: Validação manual por amostragem (1.5 dia)
- 5 docs com formatação rica selecionados (MQ-001, PQ que tem tabela, PQ que tem imagem, IT, FR)
- Para cada:
  - Comparar Drive original vs HC Quality preview lado-a-lado
  - Verificar tabelas convertidas
  - Verificar imagens (se existir)
  - Verificar headers/footers
  - Anotar discrepâncias em PILOT-IMPORT-LOG.md
- LD sugerida vs real Riopomba: validar com RT (5 docs amostra)
- Hierarquia gerada vs LM-01: validar 100% (30 docs)

**Acceptance:** Discrepâncias <10%; ajustes documentados.

### Task 5: PilotValidationDashboard (1.5 dia)
- KPIs:
  - Classificação automática accuracy (% confidence ≥0.9)
  - LD coverage (% docs com LD não-vazio)
  - Hierarquia consistency (% docs com parent correto)
  - Tempo médio preview por doc
  - Tempo total batch import
- Lista de issues encontrados
- Status: Pass/Fail por doc

**Acceptance:** Dashboard renderiza; KPIs corretos.

### Task 6: gerarRelatorioPilotMigracao callable (1.5 dia)
- Auth: claim RT ou admin
- Input: `{ labId, jobId }` (do staging job)
- Pipeline:
  - Lê job + 30 docs importados
  - Compila relatório formal: capa + KPIs + lista docs + discrepâncias + ajustes recomendados + RT sign-off space
  - PDF via Puppeteer
  - Upload Storage staging
- Brand neutro

**Acceptance:** PDF gerado; legível; <5MB.

### Task 7: Ajustes pre-prod (1 dia)
- Baseado em PILOT-IMPORT-LOG.md, aplicar ajustes:
  - Corrigir heurísticas classificação se accuracy <85%
  - Ajustar LM-01 parser se gaps detectados
  - Refinar preview rendering se formatação rica falha
  - Atualizar mapping LD/setores se discrepâncias
- Re-run pilot subset (5 docs) para validar ajustes

**Acceptance:** Re-run accuracy ≥95%; sem regressões.

### Task 8: RT sign-off + decisão go/no-go (0.5 dia)
- Reunião RT + CTO (presencial ou call)
- Apresentar PDF auditoria + dashboard
- RT decide: Go (proceder Plan 12-05) | No-go (mais ajustes necessários)
- Documentar decisão em PILOT-IMPORT-LOG.md

**Acceptance:** Decisão registrada; se Go, Plan 12-05 unblocked.

</tasks>

Se RT não aprova piloto: documentar issues; itera com Plan 12-03 antes de Plan 12-05.

</deviation_protocol>
