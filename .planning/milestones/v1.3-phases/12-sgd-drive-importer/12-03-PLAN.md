### Task 1: OAuth client setup (1.5 dia)

  - Google OAuth2Client (googleapis SDK)
  - Refresh token logic (storage no Firestore encrypted)
  - Scopes: `drive.readonly` + `drive.metadata.readonly`
- Setup Google Cloud Console (CTO faz):
  - OAuth consent screen
  - Redirect URI

**Acceptance:** OAuth flow completo end-to-end em staging.

### Task 2: lm01Parser (1.5 dia)

- `functions/src/sgq/_drive/lm01Parser.ts`:
  - Lê Google Sheets LM-01 via Sheets API
  - Parse: 15 tipos documento + 17 setores (CONTEXT.md §11)
  - Mapping: cada linha LM-01 → { codigo, tipo, titulo, versao, listaDistribuicao[], parent? }
  - Validação Zod
- Tests com 5 LM-01 samples (Riopomba real + edge cases)

**Acceptance:** 80 entries LM-01 parseadas corretamente.

### Task 3: listarDocsDrive callable (1.5 dia)

- Internal callable
- Pipeline:
  1. Lê LM-01 (lm01Parser)
  2. Para cada código LM-01: query Drive `name contains '{codigo}'`
  3. Match best file (mimeType priority: gdoc > docx > pdf)
  4. Retorna lista com metadata (driveFileId, name, mimeType, lastModified)
  5. Marca gaps (LM-01 entry sem arquivo Drive correspondente)

**Acceptance:** 80 docs Riopomba listados; gaps logged.

### Task 4: previewDocDrive callable (2 dias)

- Pipeline:
  1. Download Drive content via API (gdoc → export markdown; docx → export PDF inline)
  2. HTML sanitization (XSS protection)
  3. Convert to plain text for preview UI
  4. Audit log
- Edge cases: arquivo grande (>10MB) usa preview parcial; falha download retorna error claro

**Acceptance:** 5 docs reais Riopomba: preview funcional.

### Task 5: classificarDocAuto heurística (1 dia)

- Inputs: codigo LM-01, conteúdo preview
- Heurística:
  - Código `MQ-XXX` → tipo MQ
  - Código `PQ-XX` → tipo PQ
  - Código `IT-XXX` → tipo IT (com sub-classificação ITA/ITE/CCE baseado em conteúdo)
  - Código `FR-XX` → tipo FR
  - Código `DC-XXX` → tipo DC
- Confidence: 1.0 se código bate exato; 0.7 se ambíguo (ex: IT-005 pode ser IT, ITA, ITE)
- Sugere LD baseado em LM-01 listaDistribuicao field

**Acceptance:** 80 docs classificados; ≥85% confidence ≥0.9.

### Task 6: ImporterWizard 5 steps (3 dias)

- Step 1 — OAuthConsentStep: explica scopes + botão "Autorizar"
- Step 2 — DriveListStep: mostra 80 docs listados; gaps em vermelho
- Step 3 — PreviewBatchStep: card grid (1 por doc) com preview + classificacao automática + LD sugerida
- Step 4 — MappingEditor: RT edita classificação/LD/hierarquia individualmente OR batch (apply same to selected)
- Step 5 — Confirm: resumo final + botão "Importar todos"
- Estado: cada step tem validação; back/next; cancela limpa state

**Acceptance:** UX premium; RT navega 80 docs em <30min.

### Task 7: aprovarBatchImport callable (1.5 dia)

- Auth: claim RT
- Input: `{ labId, importJobId, docsToImport: ImportableDoc[] }`
- Pipeline (writeBatch transaction):
  1. Para cada doc: cria em `/sgq/documentos/{id}` com `statusVigencia: 'draft'`
  2. Calcula chainHash sequencial
  3. signature server-side
  4. urlDriveOriginal preservado
  5. importedFromDrive metadata
  6. Audit log do job
- Idempotência: hash driveFileId + labId

**Acceptance:** 80 docs criados em batch <30s; nenhuma duplicata em re-run.

### Task 8: ImportHistory + ImportProgress (1 dia)

- ImportHistory: lista jobs anteriores; status (running/completed/failed); link para preview
- ImportProgress: real-time progress bar durante batch (subscribe job doc)
- Cancel button: pausa job; permite retomar

**Acceptance:** UX clara; RT acompanha progress.

</tasks>
