### Task 3: QR code generator (0.5 dia)

- `functions/src/liberacao/_pdf/qrCode.ts`:

  ```typescript
  import QRCode from 'qrcode';

  export async function generateQRCodeDataUrl(url: string): Promise<string> {
    return QRCode.toDataURL(url, {
      errorCorrectionLevel: 'M',
      type: 'image/png',
      width: 120,
      margin: 1,
      color: { dark: '#000000', light: '#FFFFFF' },
    });
  }
  ```

- URL format: `https://hmatologia2.web.app/api/validar-laudo/{laudoId}/v{version}?h={hashTruncado8chars}`
- Hash truncado serve como CRC visual (não auth) para detecção de tampering óbvio

**Acceptance:** QR escaneia em smartphone Android + iOS; redireciona pro endpoint corretamente.

### Task 4: generateLaudoPDF callable (2.5 dias)

- `functions/src/liberacao/generateLaudoPDF.ts`:
  - Auth: internal (chamado por liberarLaudo) ou `isActiveMemberOfLab` (manual re-render)
  - Input: `{ labId, laudoId, version }`
  - Pipeline:
    1. Lê LaudoVersion (snapshot imutável)
    2. Compila template HTML com dados do snapshot
    3. Gera QR code com URL de validação
    4. Puppeteer launch + setContent + pdf
    5. Validação: PDF size <10MB; se exceder, page break adicional ou fallback "PDF muito grande, contate suporte"
    6. Calcula `pdfHash` = SHA-256(pdfBuffer)
    7. Upload Storage: `gs://hmatologia2.appspot.com/laudos/{labId}/{laudoId}/v{version}.pdf`
    8. Update LaudoVersion: `pdfUrl: gsUrl, pdfHash`
    9. Retorna `{ signedUrl, pdfHash }` (signed URL expira 1h)
  - Region: `southamerica-east1`, memory: `2GiB`, timeout: `120s`
  - Reutiliza puppeteer setup de `auditoria/generatePDF.ts`
- Storage rules:
  ```
  match /laudos/{labId}/{laudoId}/{filename} {
    allow read: if isActiveMemberOfLabFromAuth(labId) || (request.auth.token.medicoSolicitante == true);
    allow write: if false;
  }
  ```

**Acceptance:** PDF gera em <15s p99; tamanho <10MB com até 20 exames; QR escaneia.

### Task 5: validarLaudoPublico endpoint (2 dias)

- `functions/src/liberacao/validarLaudoPublico.ts`:
  - HTTPS endpoint (não callable) — request `/api/validar-laudo/{laudoId}/v{version}`
  - Rate limit: 60 req/h por IP (Firestore counter doc); response 429 se excedido
  - Pipeline:
    1. Parse params (laudoId, version)
    2. Lê LaudoVersion (qualquer lab — query collection group)
    3. Retorna metadata sem PII:
       ```json
       {
         "valid": true,
         "hash": "abc123...",
         "version": 2,
         "isCurrent": false,
         "supersededBy": "ver_xyz789",
         "rt": { "name": "Dr. João Silva", "registro": "CRBM 1234" },
         "lab": { "name": "Lab Clínico Riopomba", "cnes": "1234567" },
         "emissaoEm": "2026-07-15T10:30:00Z",
         "criadoEm": "2026-07-15T10:30:00Z"
       }
       ```
    4. NÃO retornar: paciente, exames, resultados, médico solicitante
  - HTML render simples para acesso direto (não API JSON):
    - Layout dark-first; brand HC Quality
    - "✅ Laudo válido" ou "⚠️ Laudo superado por versão mais nova" se isCurrent=false
    - Link "Ver laudo atual" se superseded
  - JSON response se Accept: application/json
- Error handling: laudo não existe → 404; rate limit → 429; corrupt → 500 + log

**Acceptance:** Cloudflare CDN cache 5min em frente; latência <500ms p99.
