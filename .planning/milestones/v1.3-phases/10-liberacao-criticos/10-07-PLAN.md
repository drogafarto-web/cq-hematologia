### Task 4: ADR 0009 + ADR 0010 (1 dia)
- `docs/adr/0009-liberacao-state-machine-hibrida.md`:
  - Status: Accepted
  - Context: RT manual vs auto-liberação tradeoff
  - Decision: Híbrido por classificação de exame (rotina/revisao-rt/bloqueio-critico)
  - Consequences: + throughput, + segurança, - complexidade UI config
  - Alternatives: sempre RT (rejected: gargalo), sempre auto (rejected: risco)
  - References: RDC 978 Art. 186, DICQ 5.9.1-2
- `docs/adr/0010-portal-medico-auth-externa.md`:
  - Status: Accepted
  - Context: médico solicitante precisa acessar laudos sem ser membro do lab
  - Decision: Firebase Auth com custom claim `medicoSolicitante: true` + `crm: string` + `crmUf`
  - Consequences: + isolation, + multi-tenant transversal, - complexidade onboarding
  - Alternatives: subdomínio separado (defer v1.4), magic link (rejected: UX inferior pra médicos)
  - References: ADR 0001 (audit chain), ADR 0002 (multi-tenant)

**Acceptance:** ADRs aprovados pelo CTO + merged.
