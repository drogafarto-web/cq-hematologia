### Task 1: Convite RT (1.5 dia)

- `functions/src/portal-medico/convidarMedico.ts`:
  - Auth: `isActiveMemberOfLab` + claim `rt` ou `adminLab`
  - Input: `{ labId, email, medicoIdWorklab? }` (medicoIdWorklab opcional para vincular à master Worklab)
  - Pipeline:
    1. Verificar email não tem convite ativo (idempotência)
    2. Gera token único (32 chars random) + expira 7d
    3. Cria doc em `/global-portal-medico/invitations/{invitationId}`:
       ```
       { token, email, labId, medicoIdWorklab?, convidadoPorRtId, expiresAt, status: 'pending' }
       ```
    4. Envia email via Resend com link `https://hmatologia2.web.app/portal-medico/onboarding/{token}`
    5. Audit log
  - Idempotência: hash do email + labId; reenviar = update token

**Acceptance:** Email entrega; token expira em 7d; admin RT vê histórico de convites no `/liberacao/admin/medicos`.

### Task 2: aceitarConvitePortal callable (2 dias)

- `functions/src/portal-medico/aceitarConvitePortal.ts`:
  - Input: `{ token, crm, crmUf, senha }`
  - Pipeline:
    1. Lê convite por token; valida `expiresAt > now && status === 'pending'`
    2. Validar CRM:
       - Soft validation: regex `\d{4,7}/[A-Z]{2}` (no MVP)
       - Cross-check com cache Worklab: lê `/labs/{convite.labId}/medicos-solicitantes` por CRM; se match, vincula medicoIdWorklab
    3. Cria/Update Firebase Auth user:
       - `auth().createUser({ email, password: senha })` ou link com existente
       - `auth().setCustomUserClaims(uid, { medicoSolicitante: true, crm, crmUf })`
    4. Cria/update doc em `/global-portal-medico/medicos/{uid}`:
       ```
       { uid, email, name, crm, crmUf, labsAtuantes: [labId], criadoEm }
       ```
    5. Update convite: `status: 'accepted', acceptedAt`
    6. Audit log
  - Race condition: se CRM já existe em outro UID → erro (sugere recuperar acesso)

**Acceptance:** Custom claim aparece no token JWT em <1min; reload força refresh.

### Task 10: Firestore rules + indexes (1 dia)

- Adicionar bloco para portal médico:
  ```
  // Médico vê laudos só dos seus pacientes (collectionGroup query)
  match /{path=**}/laudos/{laudoId} {
    allow read: if request.auth.token.medicoSolicitante == true
      && resource.data.medicoSolicitanteId == request.auth.uid;
  }
  match /{path=**}/laudo-versions/{versionId} {
    allow read: if request.auth.token.medicoSolicitante == true
      && resource.data.snapshot.medicoSolicitanteId == request.auth.uid;
  }
  // Comunicacoes: read-only para médico
  match /{path=**}/comunicacoes/{comunicacaoId} {
    allow read: if request.auth.token.medicoSolicitante == true
      && resource.data.laudoMedicoSolicitanteId == request.auth.uid;
  }
  // Top-level invitations
  match /global-portal-medico/invitations/{invitationId} {
    allow read: if resource.data.email == request.auth.token.email;
    allow create, update, delete: if false;
  }
  // Médicos top-level
  match /global-portal-medico/medicos/{uid} {
    allow read: if request.auth.uid == uid || isActiveMemberOfLab(resource.data.labsAtuantes[0]);
    allow update: if request.auth.uid == uid && /* limited fields */;
    allow create, delete: if false;
  }
  ```
- Indexes: collection group `(medicoSolicitanteId, criadoEm DESC)`, `(medicoSolicitanteId, criticoFlag, criadoEm DESC)`

**Acceptance:** Rule tests verde: médico vê só seus, cross-uid blocked.
