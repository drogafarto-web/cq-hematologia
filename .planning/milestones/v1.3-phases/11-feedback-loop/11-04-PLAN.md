<tasks>

### Task 1: dispararNPSPosResolucao trigger (1.5 dia)
- Firestore trigger onUpdate em `/labs/{labId}/reclamacoes/{id}`
- Filtra: `before.status !== 'Resolvida' && after.status === 'Resolvida'`
- Pipeline:
  1. Gera token único (32 chars + hmac assinado para validação)
  2. Cria doc em `/labs/{labId}/satisfacao-respostas/{id}` com `pendingResposta: true, token, expiresAt: now+14d`
  3. Envia email via Resend com link `/portal-paciente/nps/{token}`
  4. Update reclamação: `npsTokenSent: true`

**Acceptance:** 100 reclamações resolvidas → 100 emails enviados; tokens únicos.

### Task 2: dispararNPSRecurring cron (2 dias)
- `onSchedule({ schedule: '0 9 15 1,4,7,10 *', timeZone: 'America/Sao_Paulo' })` (15 jan/abr/jul/out 09:00)
- Pipeline:
  1. Para cada lab: query pacientes ativos (laudo nos últimos 12 meses)
  2. Cria docs satisfacao-respostas pending com tokens
  3. Publish Pub/Sub messages `nps-email-batch`
  4. Worker function consome: 1000 emails/min rate limit
  5. Audit log da campanha

**Acceptance:** 5000 emails simulados → todos enviados em <10min; nenhum perdido.

### Task 3: submitNPSResposta callable público (1 dia)
- Endpoint público (sem auth, validação por token)
- Input: `{ token, nota: 0-10, comentario? }`
- Pipeline:
  1. Validar token (HMAC + expiresAt)
  2. Categoriza: 0-6=detrator, 7-8=neutro, 9-10=promotor
  3. Update doc satisfacao-respostas: `pendingResposta: false, nota, categoria, comentario, respondidoEm`
  4. IP hash (anti-spam, não-PII)
  5. Filtro Gemini em comentário: detecta PII → sugere redação (RT aprova depois)
- Idempotência: token só pode submeter 1 vez

**Acceptance:** Token expirado retorna 410; duplicate submit bloqueado.

### Task 4: anonimizarRespostas cron daily (1 dia)
- `onSchedule({ schedule: '0 3 * * *', timeZone: 'America/Sao_Paulo' })`
- Query respostas `respondidoEm < now - 90d && anonimizadoEm == null`
- Update batch:
  - `pacienteId: null`
  - `reclamacaoId: null` (desvincula)
  - `anonimizadoEm: now`
  - Mantém: respostaId UUID, nota, categoria, comentario (já filtrado), origem, trimestre

**Acceptance:** 100 respostas antigas → todas anonimizadas em 1 batch.

### Task 5: NPSScoreCard + useNPSScore (1 dia)
- `useNPSScore(labId, filters)`: calcula `%promotores - %detratores`
- Card visual: número grande NPS + breakdown chart (3 barras) + delta trimestre anterior
- Indústria saúde benchmark: NPS ≥ 40 é bom

**Acceptance:** Memoização correta; recálculo <50ms.

**Acceptance:** LCP <2.5s.
