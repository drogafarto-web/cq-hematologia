<tasks>

### Task 1: criarReclamacao callable (2.5 dias)
- Auth: `isActiveMemberOfLab` (canais internos) OR `recaptchaValid` (canal público)
- Input Zod multi-canal:
  ```typescript
  {
    labId, canalEntrada, descricao, anexos?, reclamante: {nome, cpf, email?, tel?},
    consentimentoLgpd: {aceito, ipAddress, userAgent},
    origemDados: {source, metadata},   // ex: examCode + laudoId
    recaptchaToken?: string,           // canal público
  }
  ```
- Pipeline:
  1. Valida reCAPTCHA (canal público) via Google API
  2. Valida CPF format
  3. Anexos: scan VirusTotal (defer real scan; placeholder mock no MVP)
  4. Cria reclamação em status 'Nova'
  5. Chama `classificarReclamacaoIA` async
  6. Trigger criarNCDraft (Plan 11-01) automaticamente
  7. Envia email confirmação reclamante (Resend)
  8. Retorna `{ reclamacaoId, status, slaPrazo }`
- Idempotência: hash CPF + descricao + recebidoEm hour
- Region/memory padrão

**Acceptance:** 6 canais testados; idempotência verificada; consentimento LGPD persistido.

### Task 2: classificarReclamacaoIA Gemini (2 dias)
- Internal callable (não exposto)
- Input: `{ labId, descricao, contexto?: {examCode, areaSugerida} }`
- Pipeline:
  1. Compila prompt structured (CONTEXT.md §5)
  2. Gemini 2.5 Flash com `responseMimeType: 'application/json'`
  3. Zod parse strict (rejeita parse parcial)
  4. Salva log em `/labs/{labId}/feedback-classificacao-auto/{logId}` (audit IA)
  5. Update reclamação com `classificacaoAuto`
  6. Retorna sugestão
- Fallback: timeout 5s → reclamação fica sem classificacaoAuto (RT classifica manual)
- Confidence < 0.6 → flagga "review needed"

**Acceptance:** 100 reclamações teste → ≥80% confidence ≥0.7; tempo p99 <5s.

### Task 3: Web form interno (1 dia)

  - Campos: canal, reclamante, descricao, anexos

**Acceptance:** A11y AA; navegação teclado.

### Task 4: Web form público + reCAPTCHA (1.5 dia)

### Task 5: Email parser (Resend Inbound) (1.5 dia)
- `parseEmailReclamacao.ts`:
  - HTTPS endpoint: `/api/inbound/reclamacoes`
  - Verificação signature Resend (HMAC)
  - Parse: extrai sender, subject, body (HTML stripping)
  - Heurística: se `body` contém CPF → extrai como reclamante.cpf
  - Cria reclamação com canalEntrada='email'
  - Resposta auto: "Recebemos sua reclamação, código #XXX"

**Acceptance:** Resend webhook verificado; HTML injection blocked; 10 emails reais parseados corretamente.
