## Task 1: TypeScript Types

**Goal:** Type-safe personnel domain with roles and designations.

Create `src/features/personnel/types/index.ts`:

```typescript
export type RoleType =
  | 'tecnico'
  | 'enfermeiro'
  | 'farmaceutico'
  | 'analista'
  | 'gq'
  | 'rt'
  | 'diretor';

export interface Cargo {
  id: string; // role slug: "tecnico", "gq", "rt", etc.
  labId: string;
  titulo: string; // "Técnico de Laboratório", "Gerente de Qualidade", etc.
  descricao: string; // Full description with responsibilities
  responsabilidades: string[]; // Bullet points of duties
  autoridades: string[]; // Authority limits and decisions
  certificacoes?: string[]; // Required certifications
  reportaA?: string; // Role this reports to
  createdAt: Timestamp;
  updatedAt: Timestamp;
  deletedAt?: Timestamp | null;
}

export interface OrgChartNode {
  cargoId: string; // Role ID
  designacaoId?: string; // Current occupant's designation ID
  nome?: string; // Current occupant's name
  titulo: string; // Role title
  reportaA?: string; // Parent node cargoId
  filhos?: OrgChartNode[]; // Child nodes (subordinates)
}

export interface Designacao {
  id: string; // UUID
  labId: string;
  cargoId: string; // FK to Cargo
  pessoaId: string; // FK to staff member (from treinamentos)
  pessoaNome: string; // Denormalized
  dataInicio: Timestamp;
  dataFim?: Timestamp | null; // NULL = still in role
  descricaoAutoridade: string; // What this person is authorized to do
  successorId?: string; // Next person in line
  chainHash: LogicalSignature; // Signed by director/auditor
  certificadoUrl?: string; // Download URL for certificate PDF
  createdAt: Timestamp;
  updatedAt: Timestamp;
  deletedAt?: Timestamp | null;
}

export interface LogicalSignature {
  hash: string; // HMAC-SHA256 (64 chars)
  operatorId: string; // who signed
  ts: Timestamp; // when signed
}
```

**Acceptance:**

- All types exported from index
- No `any` usage
- RoleType enum covers all roles
- Timestamp usage consistent

## Task 2: Cargo Service

**Goal:** Multi-tenant Firestore service for job descriptions.

Create `src/features/personnel/services/cargoService.ts`:

**Functions:**

- `watchCargos(labId, callback)` — onSnapshot all job descriptions
- `getCargos(labId)` — single read, all cargos
- `getCargo(labId, cargoId)` — single job description
- `upsertCargo(labId, cargo)` — create or update (no direct client writes, via CF)
- `softDeleteCargo(labId, cargoId, operatorId)` — soft-delete
- `getCargoHierarchy(labId)` — tree structure for org chart

**Key Requirements:**

- Multi-tenant scoped: `/labs/{labId}/personnel/cargos/{cargoId}`
- Soft-delete only: `deletedAt` field
- Snapshot mapping: Firestore → Cargo
- Denormalize reportaA chain
- No business validation

**Acceptance:**

- Service has zero business logic
- All queries filter `where('deletedAt', '==', null)`
- Hierarchy computed correctly
