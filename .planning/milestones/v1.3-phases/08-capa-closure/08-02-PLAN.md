## Task 1: TypeScript Types

**Goal:** Type-safe calibration domain with certificate + due date tracking.

Create `src/features/calibracao/types/index.ts`:

```typescript
export interface CertificateUpload {
  id: string; // UUID
  calibracaoId: string;
  filename: string; // original filename
  mimeType: 'application/pdf' | 'image/jpeg' | 'image/png';
  storagePath: string; // gs://bucket/calibracao/{labId}/{equipId}/{id}
  hash: string; // chain-hash HMAC-SHA256 (64 chars)
  operatorId: string; // who uploaded
  uploadedAt: Timestamp;
  fileSize: number; // bytes
  chainHash: LogicalSignature; // integrity proof
}

export interface DueDateInfo {
  nextDueDate: Timestamp;
  daysUntilDue: number;
  status: 'no-prazo' | 'em-risco' | 'vencido'; // >30d, 7-30d, <7d
  alertsSent: {
    dias30: boolean;
    dias15: boolean;
    dias7: boolean;
  };
}

export interface CalibracaoRecord {
  id: string; // {equipId}
  labId: string;
  equipId: string; // FK to equipamentos
  equipName: string; // denormalized
  equipSerial?: string;
  lastCalibrationDate: Timestamp;
  nextDueDate: Timestamp;
  vendor?: string; // calibration vendor
  vendorRef?: string; // vendor document number
  certificates: CertificateUpload[];
  dueDateInfo: DueDateInfo;
  notes?: string;
  createdAt: Timestamp;
  updatedAt: Timestamp;
  deletedAt?: Timestamp | null;
}

export type CalibracaoStatus = 'no-prazo' | 'em-risco' | 'vencido';

export interface DueDateAlert {
  id: string;
  labId: string;
  equipId: string;
  calibracaoId: string;
  daysRemaining: number; // 30, 15, 7
  sentAt: Timestamp;
  dismissedAt?: Timestamp | null;
}
```

**Acceptance:**

- All types exported from index
- No `any` usage
- LogicalSignature reused from signature pattern
- Timestamp usage consistent with service layer

## Task 3: Certificate Upload Service

**Goal:** Cloud Storage integration with chain-hash computation.

Create `src/features/calibracao/services/certificateUploadService.ts`:

**Functions:**

- `uploadCertificate(labId, equipId, file, operatorId) → Promise<CertificateUpload>`
  - Validate file: PDF or JPG only, <10MB
  - Compute HMAC-SHA256 chain-hash: `hash(labId + equipId + filename + operatorId + ts)`
  - Upload to Cloud Storage: `/calibracao/{labId}/{equipId}/{uuid}`
  - Store metadata (CertificateUpload) in Firestore subcollection
  - Return CertificateUpload object

- `downloadCertificate(storagePath) → Promise<Blob>`
  - Get download URL from Cloud Storage
  - Validate chain-hash matches stored value
  - Return file blob

- `validateCertificateIntegrity(cert, file) → boolean`
  - Re-compute hash, compare to stored
  - Throw error if tampered

**Key Requirements:**

- File validation (MIME type, size)
- Chain-hash HMAC-SHA256 (not simple SHA, keyed for security)
- Atomic: metadata stored in Firestore + file in Storage (both or nothing)
- Error handling: storage quota, upload timeout
- Progress callbacks for UI (% uploaded)

**Acceptance:**

- Files stored in correct bucket path
- Chain-hash 64 chars, hex-encoded
- Download validates integrity before returning
- Test: integration with Firebase Storage emulator

## Task 7: Firestore Rules

**Goal:** Server-enforce write protection on calibration records.

Update `firestore.rules`:

```
match /labs/{labId}/calibracao/{equipId} {
  allow read: if isMember(labId);

  match /certificates/{certId} {
    allow read: if isMember(labId);

    // Writes ONLY via Cloud Function
    allow create, update, delete: if false;
  }

  // Writes (new calibration, update due date) ONLY via Cloud Function
  allow create, update: if false;
  allow delete: if false;
}
```

**Acceptance:**

- Direct client writes rejected
- Reads allowed for lab members
