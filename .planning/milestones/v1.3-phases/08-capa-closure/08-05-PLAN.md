## Task Breakdown (6 Weeks)

### Week 1-2: Evidence Preparation (2026-05-13 → 2026-05-27)

**Task 1: CAPA Dashboard Review**

- Load `capa-tracking` module (08-01) — review all 12 CAPAs in dashboard
- Verify 2 Critical + 6 High are visible + deadline tracking accurate
- Document current status (Open, In Progress, etc.)

**Task 2: Module Deployment Verification**

- Confirm 08-02 (calibracao) deployed to Firebase
- Confirm 08-03 (personnel/cargos + designacoes) deployed
- Confirm 08-04 (management-review) deployed
- Take deployment screenshots for auditor evidence

**Task 3: Certificate Collection (NC-002)**

- Request calibration certificates from equipment vendors
- Upload to calibracao module (via UI or batch import)
- Generate calibracao module evidence report:
  - Equipment list with calibration records
  - Certificate photos/scans
  - Due date monitoring active

**Task 4: Job Descriptions & Designations (NC-003 + NC-004)**

- Populate all personnel/cargos entries (Técnico, Enfermeiro, etc.)
- Sign designacoes for Quality Manager + RT + Director
- Capture screenshots of org chart + designation signatures
- Export designacao PDF certificates

**Task 5: Management-Review Setup (NC-001)**

- Prepare first management-review ata (2026-05 annual review)
- Complete all 15 mandatory sections
- Director signature + LogicalSignature validation
- Screenshot evidence of review system

**Task 6: Auditor Communication**

- Send Week 1 email to auditor:
  - NC-011 deferral discussion confirmation
  - 4-module deployment status
  - Evidence timeline (Weeks 2-4)
  - Sign-off meeting tentative date (Week 6)

### Week 3-4: RFI Response Documentation (2026-05-27 → 2026-06-10)

**Task 7: RFI Response — NC-001**

- Document: "Auditoria Interna POP not documented"
- Response:
  - Management-review module deployed (DICQ 4.15 compliance)
  - Annual ata prepared with 15 mandatory sections
  - Director signature + LogicalSignature proof
  - POP for quarterly mini-reviews created + signed
- Evidence artifacts:
  - Screenshots of management-review module
  - Signed ata PDF
  - Mini-review POP document
  - Deployment git commits

**Task 8: RFI Response — NC-002**

- Document: "Calibração — Falta em analisadores"
- Response:
  - Calibracao module deployed with certificate upload
  - All lab equipment registered in system
  - Vendor certificates collected + uploaded
  - Due date alerts configured (30/15/7 days)
  - Rastreabilidade metrológica (DICQ 5.3.1.4) implemented
- Evidence artifacts:
  - Equipment list with certificate IDs
  - Photos of certificate uploads
  - Due date monitoring screenshots
  - Audit trail (git commits)

**Task 9: RFI Response — NC-003 + NC-004**

- Document: "Job descriptions not formalized" + "GQ designation not documented"
- Response:
  - Personnel/cargos module deployed with all role descriptions
  - Designacoes module deployed with signature support
  - Quality Manager + RT + Director formally designated + signed
  - Org chart available in system for authority tracking
- Evidence artifacts:
  - Populated cargos list (all roles)
  - Signed designacao certificates (PDF)
  - Org chart visualization
  - Audit trail (git commits)

**Task 10: RFI Response — NC-007–010 (Operational)**

- Document findings per audit report (TBD)
- Responses: Process improvements, training, documentation
- Evidence: Updated procedures, training records, sign-offs

### Week 5: Evidence Upload + Status Update (2026-06-10 → 2026-06-24)

**Task 11: Dashboard Evidence Upload**

- For each of 8 CAPAs in dashboard:
  - Upload RFI response document
  - Upload evidence artifacts (photos, PDFs, screenshots)
  - Link git commits via commit hash
  - Add sign-off checklist (DICQ 4.14 format)
  - Set CAPA status to "Evidence Submitted"
  - Add LogicalSignature metadata

- Evidence upload via CloudFunction callable:
  ```typescript
  submitCAPAEvidence({
    capaId: "NC-001",
    evidenceType: "RFI_RESPONSE",
    documents: [...],
    rfiResponse: {...},
    signOffChecklist: {...},
    auditTrail: {hash, operatorId, ts}
  })
  ```

**Task 12: Closure Summary Report**

- Create closure report (markdown):
  - 8/12 CAPAs closed (2 Critical + 6 High)
  - 92% closure rate (11/12 after NC-011 deferral)
  - Timeline: all deadlines met or deferred with approval
  - Evidence artifacts: X total files, Y MB
  - Auditor sign-off pending (Week 6)
- Commit to git:
  ```
  docs(08-capa-closure): CAPA closure summary — 8/12 closed, 92% compliance
  ```

### Week 6: Auditor Sign-Off (2026-06-24 → 2026-06-30)

**Task 13: Auditor Coordination**

- Schedule video call with auditor (1 hour)
- Present:
  - Dashboard walkthrough (all 8 CAPAs)
  - Module deployments (screenshots + live demo if possible)
  - Evidence summary (RFI responses + artifacts)
  - Closure summary report
- Auditor reviews evidence + marks CAPAs "Closed" in dashboard

**Task 14: Formal Sign-Off**

- Auditor sends closure email:
  - "All 8 CAPAs reviewed and approved for closure"
  - Email archived in evidence storage
- LogicalSignature added to CAPA records:
  - hash: SHA-256 of auditor sign-off email + CAPA metadata
  - operatorId: auditor email / ID
  - ts: sign-off timestamp
- Commit auditor correspondence:
  ```
  docs(08-capa-closure): auditor sign-off — 8 CAPAs closed
  ```
