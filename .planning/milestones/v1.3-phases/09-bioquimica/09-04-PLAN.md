### Task 2: chainHashCalc shared util (0.5 dia)

- Criar `functions/src/bioquimica/_shared/chainHashCalc.ts`:

  ```typescript
  import { createHash } from 'crypto';
  import { canonicalize } from './canonicalize';

  export function calcChainHash(prevHash: string, payload: object): string {
    const canonical = canonicalize(payload);
    const input = `${prevHash}|${canonical}`;
    return createHash('sha256').update(input).digest('hex');
  }
  ```

- Idempotente: mesma sequência → mesmo hash final
- Genesis hash: string fixa `'00...0'` (64 chars de zero)

**Acceptance:** Sequência de 10 runs gera mesma chain hash em 3 execuções independentes.

### Task 3: recordRunBioquimica callable (2 dias)

- Criar `functions/src/bioquimica/recordRunBioquimica.ts`:
  - Auth: `isActiveMemberOfLab(labId, request.auth.uid)`
  - Input Zod: `{ labId, equipmentId, lotId, resultados, operatorId, signature, complianceOverride? }`
  - Pipeline atômico (Firestore transaction):
    1. Lê últimos 30 runs do par (para checkWestgardCLSI)
    2. Lê stats source: bula primeiro, fallback interna se N≥20
    3. Roda `checkWestgardCLSI` server-side por analito × nível
    4. Determina status final (`Aprovada` | `Rejeitada`) baseado em violations + override
    5. Lê último chainHash do lab; calcula novo via `calcChainHash`
    6. Cria run doc com violations + chainHash + signature server-side validada
    7. Audit log entry em `/labs/{labId}/bioquimica/audit/{logId}` (append-only)
  - Retorna: `{ runId, status, violations, chainHash }`
- Idempotência: mesma `signature.hash` → segunda chamada retorna run existente sem duplicar
- Region: `southamerica-east1`, memory: `512MiB`, timeout: `60s`

**Acceptance:** 100 runs concorrentes não geram chainHash duplicado nem perdido (transaction garante).
