## How to Use This Framework

### For Executors (Engineers)

1. **Find your stream:** Ctrl+F "Stream [A/B/C/D]" in `v1.4-ENGINEER-TASK-DETAIL.md`
2. **Read your Phase N:** Get task breakdown, acceptance criteria, code locations
3. **When blocked:** Escalate via `v1.4-ENGINEER-TASK-DETAIL.md` blocker section
4. **When complete:** Use **executor reporting template** in `v1.4-MASTER-EXECUTION-FRAMEWORK.md` to submit completion report

### For CTO

1. **Morning:** Check `v1.4-CTO-EXECUTIVE-SUMMARY.md` (08:00 BRT)
2. **Decisions needed:** See **DECISION 1–3** section; respond by SLA
3. **Risks escalate:** Weekly review on Mondays 09:30 BRT (see risk register)
4. **Pre-phase approval:** Sign off on pre-deploy checklists (DevOps)

### For QA / Auditor

1. **Weekly:** Check `v1.4-AUDITOR-COMPLIANCE-CHECKLIST.md` (Fridays 14:00 BRT)
2. **Phase 4 start:** Confirm pre-alignment call (Week 1, target 2026-05-13)
3. **Phase completion:** Review auditor sign-off section for gate criteria
4. **Aug 31:** Confirm final audit readiness (last section)

### For DevOps

1. **72h before deploy:** Check `v1.4-OPS-DEPLOYMENT-READINESS.md` pre-deploy checklist
2. **Deploy day:** Follow 3-step procedure (Rules → Functions → Hosting)
3. **24h post-deploy:** Hotline duty; monitor Cloud Logs
4. **Next morning:** Submit deployment report (template in OPS doc)
