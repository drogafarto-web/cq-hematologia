## Executive Summary

**ADR 0003 (NC Global):** Consolidate Non-Conformidade spine. Currently scattered across modules (NC temporiários em Controle de Qualidade, aberturas em Insumos, etc). Creates canonical `NaoConformidade` collection with CAPA (Ação Corretiva) workflow. Integrates with ADR 0005 (HMAC audit trail for NC history). Closes **V-001**.

**ADR 0004 (POP Versionado):** Implement Standard Operating Procedures with automatic versioning. Each POP version is RT-signed, immutable, and referenced in all CIQ module runs. Operators can only use POPs they're trained on. Closes **V-004, V-011**.

**Success Criteria:**

- NC Global schema implemented + integrated across 7 modules
- CAPA workflow: open → investigate → action → verify efficacy
- Critical NCs block module operations (auto-close on remediation)
- POP versioning: v1.0, v1.1, etc with RT signature
- All CIQ runs reference popId + popVersaoId
- Zero breaking changes to existing data

**Parallel Execution:** Both ADRs can run in parallel. NC doesn't depend on POP, and POP doesn't depend on NC. Only shared dependency: ADR 0005 (audit trail).

## ADR 0003 — Spine NC Global

### Problem Statement

**V-001:** Non-Conformidade (NC) handling is fragmented:

- Some modules create `NaoConformidadeTemp` in their own collections (e.g., `labs/X/controleQualidade/desvios`)
- No central registry of all NCs across the lab
- CAPA (investigação → ação corretiva → verificação) logic is manual, not enforced
- No audit trail of NC lifecycle (who opened, when, actions taken)
- Critical NCs don't automatically block module operations

**Impact:** Compliance gap. Auditors can't trace desvios → NC → ação → eficácia. RDC 978 requires formal NC handling.

### Solution Design

**Schema: NaoConformidade**

```typescript
interface NaoConformidade {
  id: string; // auto-generated
  labId: string; // FK to lab
  numero: string; // NC-{YYYY}-{seq} for audit trail

  // Origin
  origem: 'insumo' | 'controle' | 'equipamento' | 'pessoas' | 'processo' | 'outro';
  origemId?: string; // FK: insumoId, equipId, userId, popId, etc
  moduloOrigemId: string; // Which module opened: 'insumos', 'qualidade', etc

  // Description
  descricao: string; // What went wrong
  severidade: 'leve' | 'grave' | 'critica'; // Impact level

  // Status machine
  status: 'aberta' | 'investig' | 'correcao' | 'verif_eficacia' | 'fechada' | 'cancelada';
  statusHistory: Array<{
    // HMAC-signed via ADR 0005
    timestamp: Timestamp;
    novoStatus: string;
    mudadoPor: string; // uid
    motivo?: string;
    hmac: string; // signed entry
  }>;

  // CAPA Workflow
  capa: {
    investigacao?: {
      realizada: boolean;
      dataInicio: Timestamp;
      dataFim?: Timestamp;
    };
    acaoCorretiva?: {
      descricao: string;
      dataPrevista: Timestamp;
      dataRealizacao?: Timestamp;
      responsavel: string; // uid
      status: 'planejada' | 'em_exec' | 'concluida';
    };
    verificacaoEficacia?: {
      realizada: boolean;
      resultado: 'eficaz' | 'ineficaz' | 'nao_concluida';
      dataVerificacao?: Timestamp;
      verificadoPor: string; // uid
    };
  };

  // Rastreability
  aberta: {
    timestamp: Timestamp;
    uid: string; // Who opened (operador, supervisor)
    motivo: string;
  };
  fechada?: {
    timestamp: Timestamp;
    uid: string;
    motivo: string;
  };

  // Blocking
  bloqueiaOperacoes: boolean; // If true, module operations should check & block
  operacoesTodasBloqueadas?: string[]; // ['hematologia', 'imunologia'] if needed

  // Audit (ADR 0005 integration)
  hmac: string; // Signature of full NC
  previousHash: string | null; // Chain link
  _ncAuditTrailRef: string; // FK to audit entry
}
```

**Integration Points (7 modules):**

1. **Insumos:** NCs for expired lots, contamination, deviation from lot specs
2. **Equipamento:** Equipment failures, calibration issues, maintenance problems
3. **Controle de Qualidade:** QC failures, EQA issues, CEQ deviations
4. **Pessoas:** Training failures, qualification expirations, competency issues
5. **POPs:** Procedure deviations, non-adherence to workflows
6. **Evoluções (Resultados):** Result quality issues, transcription errors
7. **Auditoria:** Finding from external audits, corrective action tracking

**Flow: Open NC → Auto-block if critical → Investigate → Action → Verify → Close**

```
┌──────────────────────────────────────────────────────────────────┐
│ Módulo detects deviation (e.g., Insumo expired)                 │
└────────────────┬─────────────────────────────────────────────────┘
                 │
                 ▼
        ┌────────────────────┐
        │ openNaoConformidade│  (callable)
        │ (operador/RT)      │
        └────────┬───────────┘
                 │
        ┌────────▼──────────────────────────┐
        │ Create NC doc (status: aberta)    │
        │ Sign w/ HMAC (ADR 0005)           │
        │ Add to audit trail                │
        └────────┬──────────────────────────┘
                 │
        ┌────────▼──────────────────────────┐
        │ IF severidade = critica           │
        │   THEN bloqueiaOperacoes = true   │
        │   Block module from further ops   │
        └────────┬──────────────────────────┘
                 │
        ┌────────▼──────────────────────────┐
        │ investigarNC() [RT-only]           │
        │ Doc → CAPA.investigacao.realizada │
        │ Sign w/ HMAC (ADR 0005)           │
        └────────┬──────────────────────────┘
                 │
        ┌────────▼──────────────────────────┐
        │ executarAcaoCorretiva() [RT]      │
        │ Plan action, date, responsible   │
        │ Change status → 'correcao'       │
        │ Sign w/ HMAC (ADR 0005)          │
        └────────┬──────────────────────────┘
                 │
        ┌────────▼──────────────────────────┐
        │ verificarEficacia() [RT]          │
        │ Test if action fixed issue       │
        │ If OK → status 'fechada'         │
        │ If NOT → reopen investigation    │
        │ Sign w/ HMAC (ADR 0005)          │
        └────────┬──────────────────────────┘
                 │
                 ▼
        ┌────────────────┐
        │ NC FECHADA     │
        │ Full history   │
        │ HMAC-signed    │
        └────────────────┘
```

## ADR 0004 — POP / Documento Vigente

### Problem Statement

**V-004, V-011:** POP (Standard Operating Procedures) handling is not formalized:

- POPs exist in various formats (PDF, docs, shared drives)
- No versioning (v1.0, v1.1) or obsolescence tracking
- RT signature not mandatory
- Operators can use expired/obsolete POPs
- No linkage from CIQ runs to POP version used
- Training on POPs not tracked or enforced

**Impact:** Compliance gap. Can't prove operator was trained on POP X when running procedure on date Y. RDC 978 requires POP traceability.

### Solution Design

**Schema: POP**

```typescript
interface POP {
  id: string; // auto-generated
  labId: string; // FK to lab
  nome: string; // "Hemograma - Sysmex XN-1000"
  codigo: string; // "POP-HEM-001" for audit

  // Content
  conteudo: {
    markdown?: string; // Inline procedure text
    pdfUrl?: string; // GCS link to PDF
    versaoDocumento: string; // "v1.0", "v2.1", etc (auto-increment)
  };

  // Versioning
  versoes: Array<{
    numero: string; // "1.0", "1.1", "2.0"
    dataVigenciaInicio: Timestamp;
    dataVigenciaFim: Timestamp;
    hashConteudo: string; // SHA-256 of content (immutable)
    assinadaPor: {
      uid: string; // RT uid
      nome: string;
      cargo: string;
      timestamp: Timestamp;
      hmac: string; // ADR 0005 signature
    };
    proximaRevisao: Timestamp;
    status: 'ativa' | 'obsoleta' | 'em_revisao';
    motivo_obsolescencia?: string;
  }>;

  // Training Linkage
  treinamentosObrigatorios: Array<{
    modulo: string; // 'hematologia', 'imunologia', etc
    tipoTreinamento: 'inicial' | 'reciclagem';
    periodicidadeMeses: number; // e.g., 24 for biennal
  }>;

  // Audit
  criadoEm: Timestamp;
  criadoPor: string; // uid (admin/RT)

  // Scope
  modulos: string[]; // ['hematologia', 'imunologia'] (which modules use this)
}

interface POPVersao {
  // Denormalized in runs for immutability
  popId: string;
  popVersaoNumero: string; // "1.0", "1.1", etc
  assinadaPor: string; // uid (RT who signed)
  dataAssinatura: Timestamp;
}

// In every CIQ run (Hematologia, Imunologia, etc):
interface RunCIQ {
  // ... existing fields ...
  popReferencia?: POPVersao; // NEW: which POP version was used
  operadorTrainedEmPopId?: boolean; // Denormalized at run-time for audit
}
```

**Training Integration:**

```typescript
// Qualificacao (ADR 0006) extended
interface Qualificacao {
  // ... existing ...
  treinamentosPOP?: Array<{
    popId: string;
    popVersaoNumero: string;
    dataConcluso: Timestamp;
    validoAte: Timestamp;
    certificado_url?: string;
  }>;
}

// Validator before run
async function canOperadorUsarPOP(
  labId: string,
  uid: string,
  popId: string,
  popVersaoAtual: string,
): Promise<{ allowed: boolean; reason?: string }> {
  // Check if operator has active training on THIS POP version
  const qual = await db
    .collection(`labs/${labId}/qualificacoes`)
    .where('uid', '==', uid)
    .where('treinamentosPOP', 'array-contains', { popId, popVersaoNumero: popVersaoAtual })
    .get();

  if (qual.empty) return { allowed: false, reason: 'Operator not trained on this POP' };

  const trainRec = qual.docs[0]
    .data()
    .treinamentosPOP.find((t) => t.popId === popId && t.popVersaoNumero === popVersaoAtual);

  if (new Date(trainRec.validoAte) < new Date()) {
    return { allowed: false, reason: 'Training expired' };
  }

  return { allowed: true };
}
```

**Flow: Create POP → RT signs → Operator trains on version → Uses in runs → Old version auto-obsolete**

```
┌─────────────────────────────────────┐
│ Admin creates/updates POP (markdown)│
└────────────────┬────────────────────┘
                 │
        ┌────────▼─────────────────────────┐
        │ createPOPVersion()                │
        │ - Compute hashConteudo (SHA-256) │
        │ - Set vigenciaInicio = now       │
        │ - Set vigenciaFim = now + 2 years│
        └────────┬─────────────────────────┘
                 │
        ┌────────▼─────────────────────────┐
        │ Send for RT approval (signal)    │
        │ RT reviews, approves             │
        └────────┬─────────────────────────┘
                 │
        ┌────────▼─────────────────────────┐
        │ assinaturaRT()                    │

        │ - Set assinadaPor, hmac          │

        │ - OLD version → status='obsoleta'│
        └────────┬─────────────────────────┘
                 │
        ┌────────▼─────────────────────────┐
        │ TreinamentoPOP (auto-schedule)   │
        │ - Notify operators: new POP v1.1 │
        │ - Training deadline              │
        └────────┬─────────────────────────┘
                 │
        ┌────────▼─────────────────────────┐
        │ Operador trains on POP v1.1      │
        │ recordarTreinamentoPOP()         │
        │ - Add to Qualificacao[]          │
        │ - validoAte = now + 24 meses     │
        │ - Certificate timestamp          │
        └────────┬─────────────────────────┘
                 │
        ┌────────▼─────────────────────────┐
        │ Run CIQ test                      │
        │ checkPOPVersion():                │
        │   canOperadorUsarPOP()?           │

        └────────┬─────────────────────────┘
                 │
                 ▼
        ┌─────────────────────────┐
        │ Run saved w/ POP trace  │
        │ Fully auditable         │
        └─────────────────────────┘
```

## Risks & Mitigations

| Risk                                                    | Likelihood | Impact | Mitigation                                                  |
| ------------------------------------------------------- | ---------- | ------ | ----------------------------------------------------------- |
| NC backfill maps incorrectly (origin module)            | Medium     | High   | Dry-run on test lab first; manual audit sample              |
