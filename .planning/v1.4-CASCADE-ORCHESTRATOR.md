## Execution Strategy

### 1. Polling & Detection

**Trigger mechanism:** Poll `.planning/STATE.md` every 5 minutes for phase completion signals.

**Detection logic:**

```
Every 5 minutes (automated):
  1. Read STATE.md
  2. Check latest phase status line: "**Phase N** | ✅ COMPLETE"
  3. If complete && next phase queued:
       → Dispatch next phase kickoff (Skill: /gsd-plan-phase)
  4. If next phase executing && all subagents alive:
       → Dispatch /gsd-execute-phase (auto-approve, sleep-safe)
  5. Log event to ORCHESTRATOR_LOG.md
  6. Wait 5 min, repeat
```

**Completion markers:**

- Phase 5 complete: `| ✅ COMPLETE | 2026-05-14 |` in STATE.md AND all 05-\*.SUMMARY files exist
- Phase 6 complete: `| ✅ COMPLETE | 2026-05-28 |` in STATE.md AND all 06-\*.SUMMARY files exist
- (etc.)

### 2. Auto-Trigger Sequence

When Phase N completion detected:

```
1. Invoke /gsd-plan-phase phase:N+1 auto-approve all-permissions
   (creates .planning/phases/0N+1-*/0N+1-*.PLAN.md)

2. Poll for PLAN.md creation (max 10 min)

3. Invoke /gsd-execute-phase phase:N+1 waves:15 auto-approve all-permissions sleep-safe
   (dispatches 4 subagents, runs in background)

4. Log to ORCHESTRATOR_LOG.md:
   - Trigger timestamp
   - Phase number
   - Subagent count
   - Expected completion (EST_DATE)

5. Resume polling for Phase N+1 completion
```

### 3. Error Handling & Recovery

**If a phase execution stalls >24 hours:**

1. Check ORCHESTRATOR_LOG.md for last activity
2. Inspect `.planning/phases/0N-*/00-CHECKPOINT.md` (GSD standard)
3. If safe: manually resume with `/gsd-resume-work`
4. If unsafe: escalate to CTO (drogafarto@gmail.com) with incident report

**If a subagent dies:**

1. GSD skill auto-respawns (built into /gsd-execute-phase)
2. Log to ORCHESTRATOR_LOG.md
3. Continue execution (no manual action needed)
