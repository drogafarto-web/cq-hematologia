## RFI Management (Request For Information)

**When Auditor Asks a Question (any time):**

1. **Immediate action (same day):**
   - Log in RFI tracking sheet: `AUDITOR_RFI_TRACKING.md`
   - Assign ID: RFI-XXX (sequential)
   - Flag to relevant module owner (if engineering answer needed)
   - Set due date: [Open date + 5 business days]
   - Status: ⏳ Pending

2. **Coordination (Days 1–3):**
   - Daily: Check with assigned owner on progress
   - Day 3: If no draft answer, escalate to CTO

3. **Draft response (Days 3–4):**
   - Module owner prepares technical answer + evidence links
   - You verify: question restated accurately, evidence compelling, tests pass
   - Forward to CTO for sign-off

4. **Send response (Day 4–5):**
   - Use Template 2 (RFI Response Email)
   - Include: restatement, answer, code references, tests, next steps
   - Attach: evidence PDF (code snippets + test results)
   - Copy to tracking sheet: Status = ✅ Answered [date]
   - Archive email in `.planning/email-archive/`

5. **If > 5 business days:**
   - Day 6: Send escalation email (Template 8)
   - Assign new ETA + owner
   - Copy CTO + Ernani
   - Update tracking sheet with new due date

## Escalation Playbook

**Trigger: Phase delivery >3 days late**

```
Severity: 🟡 Yellow
1. Email CTO immediately (subject: "Phase X Slipping 3+ Days — HC Quality")
2. Propose re-plan: which Phase X+1 tasks can absorb delay?
3. DICQ impact: will new ETA still hit 88% by Aug 4?
4. Notify Ernani in next Friday weekly email (Template 1) — auditor context
```

**Trigger: DICQ trend <+0.5%/week (2-week rolling average)**

```
Severity: 🟡 Yellow
1. Flag in Friday briefing: "DICQ trend slower than expected"
2. Propose acceleration: can we parallelize Phase X?
3. Escalate to CTO if no mitigation found
4. May require Phase 5 kickoff earlier than Jun 9
```

**Trigger: RFI >5 business days without response**

```
Severity: 🟡 Yellow → 🔴 Red (if >7 days)
1. Day 6: Send escalation email (Template 8)
2. Notify CTO in Slack (tag @CTO, "RFI-XXX at risk")
3. Offer: can we respond with partial answer + follow-up?
4. Day 7: Escalate to board/decision-maker (if CTO unavailable)
```

**Trigger: Critical security finding in Cloud Logs**

```
Severity: 🔴 Red (IMMEDIATE)
1. Alert CTO immediately (phone + email)
2. Incident commander activates
3. Notify Ernani within 1h (severity + remediation plan)
4. Daily updates until resolved
5. Post-mortem + RFI-follow-up from auditor expected
```

**Trigger: LGPD data breach detected**

```
Severity: 🔴 Red (IMMEDIATE + LEGAL)
1. Alert CTO + legal immediately (phone + email)
2. Incident commander + legal team lead response
3. Notify Ernani + regulatory authorities per LGPD Art. 34 (within 2h)
4. Daily status updates
5. Compliance remediation plan due within 72h
```
