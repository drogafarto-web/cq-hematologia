# v1.4 Integration Blockers Log

**Purpose:** Track integration blockers discovered during cross-wave execution (Phases 4–15).

**Log entry format:**
- ID: `BLOCK-YYYYMMDD-NN` (date-sequenced)
- Phase: Which phase discovered blocker (or "Discovery" if pre-execution)
- Status: OPEN | IN PROGRESS | RESOLVED
- SLA: 2h (critical) / 24h (medium) / 1 week (low)

---
