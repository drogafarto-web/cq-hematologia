## 2. Functional Gates (Per-Phase)

### Phase 4: Portal Auth + NOTIVISA (Target: 2026-06-02)

**Must-Haves:**
- [ ] RT authentication callable working (3 happy-path tests)
- [ ] NOTIVISA queue processing 100% (0 failures in 24h staging)
- [ ] Portal UI loads <2.5s LCP (Lighthouse CI)
- [ ] E2E: laudo review → sign → archive (8 flows PASS)
- [ ] No regressions: 738/738 baseline tests still green
- [ ] Cloud Logs: 0 errors, <3% warning rate

**Go-No-Go Decision:**
- All must-haves ✅ → Phase 4 APPROVED for production
- Any must-have ❌ → Continue Phase 4 extended window (1 week buffer), re-test
- More than 2 must-haves failing repeatedly → escalate to CTO, consider deferral to v1.4.1

### Phase 5: Critical Escalation + IA (Target: 2026-06-30)

**Must-Haves:**
- [ ] SMS/email delivered <2 min after critical value
- [ ] Critical value dashboard 100% uptime (24h test)
- [ ] IA strip upload: 1,000+ images processed (batch + Gemini Vision latency <5s)
- [ ] Model versioning: v1.0 vs v1.1 A/B framework ready
- [ ] E2E: 8 critical flows PASS
- [ ] No regressions: 738/738 baseline tests + Phase 4 gains

**Go-No-Go Decision:**
- All must-haves ✅ → Phase 5 APPROVED for production
- SMS delivery >2 min or IA latency >10s → extended tuning window (1 week)
- Gemini Vision quota exhaustion → Phase 5 deferred to v1.4.1

### Phases 6–7: Liberação Completion + Reclamações Polish (Target: 2026-07-28)

**Must-Haves:**
- [ ] PDF generation + QR validation callable working
- [ ] Portal médico SSO authentication complete
- [ ] Portal paciente feedback submission working
- [ ] Trending dashboard filters functional
- [ ] E2E: 16 flows PASS (8 liberacao + 8 feedback)
- [ ] No regressions: all previous tests + Phase 5 gains

**Go-No-Go Decision:**
- All must-haves ✅ → Phases 6–7 APPROVED for production
- Portal issues → defer to v1.4.1 (keep state machine + foundation)
- PDF generation latency >30s → batch processing queue required

### Phase 8: CAPA Closure (Target: 2026-08-05, Mandatory for External Audit)

**Must-Haves:**
- [ ] F-01 to F-04 eng findings fully resolved + verified
- [ ] F-05 to F-07 CTO + Auditor review complete
- [ ] Auditor sign-off on all 7 findings (documentation in `/docs/audits/`)
- [ ] Management review evidence collected + archived
- [ ] Zero open "needs verification" items

**Go-No-Go Decision:**
- All findings closed ✅ → CAPA APPROVED for auditor ceremony
- ≥1 finding still open ❌ → CAPA NOT READY (halt external audit)
  - If single finding fixable in <3 days → extended window (2026-08-12)
  - If structural issue → escalate to auditor + reschedule audit (post-2026-08-31)

## 7. Escalation Matrix

### Severity: RED (Stop Production, Escalate Immediately)

| Issue | Owner | Response Time |
|-------|-------|----------------|
| Data loss or chain-hash break | CTO + DevOps | <30 min |
| Rules lockout (users blocked) | Security + DevOps | <30 min |
| Cloud Function cascading failure (>10% 5xx) | Eng lead + DevOps | <1 hour |
| Auditor finding indicates compliance gap | CTO | <4 hours |

**Action:** Rollback + incident postmortem + no further deploys until root cause fixed.

### Severity: YELLOW (Extended Window, Re-Plan)

| Issue | Owner | Response Time |
|-------|-------|----------------|
| Performance regression (LCP >3s) | Eng A | <8 hours |
| IA latency >10s or quota exhaustion | Eng B | <8 hours |
| Portal auth >2 sec cold-start | Eng C | <8 hours |
| Regressions in baseline tests (>1% new failures) | QA | <24 hours |

**Action:** Extended tuning window (1 week) or deferral to v1.4.1. Continue Phase execution.

### Severity: GREEN (Monitor, Log, Plan for v1.4.1)

| Issue | Owner | Action |
|-------|-------|--------|
| Minor UI polish needed | Eng A | Note for v1.4.1 backlog |
| Lighthouse CI score 88–89 (target 90) | Eng A | Plan optimization for Phase 6 |
| Test coverage 95–97% (target 98%) | QA | Plan additional E2E for Phase 6 |

**Action:** Continue phase. No blocking.
