# v1.4 Consolidation — Automated Update Schedule

**Purpose:** Define when each document updates, who updates it, what triggers updates, and escalation paths when updates miss SLA.

---

## Master Status Matrix Update Schedule

### Primary Update (Every 4 Hours)

**Document:** `.planning/v1.4-MASTER-EXECUTION-FRAMEWORK.md`

**Times:** 08:00, 12:00, 16:00, 20:00 BRT  
**Owner:** Documentation Consolidator  
**Trigger:** Automatic (cron) + on-demand (manual trigger in Slack)

**Actions per update:**
1. Check Slack `#v1-4-exec` for phase completion reports (last 4 hours)
2. Pull any executor reports submitted to consolidator@v1.4
3. Update "Master Status Matrix" section (columns: Owner, Status, % Done, Blockers, Last Update)
4. Cascade updates to role-specific views (see below)
5. If P0/P1 blocker detected: notify via Slack `#v1-4-risks` immediately

**Format:** Timestamp + summary line (e.g., "2026-05-20 12:00 BRT: Phase 4 Task 4.1 75% complete, no new blockers")

## Risk Register Update Schedule

### Weekly (Mondays 09:30 BRT)

**Document:** `.planning/milestones/v1.4-RISK-REGISTER.md`

**Owner:** CTO (final authority) + Stream leads (input)  
**Trigger:** Weekly risk review meeting + escalations from previous week

**Actions:**
1. Review all active risks (RISK-401 through RISK-419)
2. For each risk: update "Status" column (Active / Mitigated / Materialized / Closed)
3. If mitigation status changed: update "Mitigation" column + timestamp
4. If new risk surfaced (Phase completion): add RISK-[new] entry (score, probability, impact)
5. Identify any risks crossing threshold (score ≥7 or triggered escalation event)

## Deployment & Hotline Reporting

### Pre-Deploy (72h Before)

**Document:** `.planning/v1.4-OPS-DEPLOYMENT-READINESS.md`

**Time:** Phase N deploy date - 72h (typically Thursday for Friday deploy)  
**Owner:** DevOps Lead  
**Trigger:** Phase N completion + CTO sign-off

1. Review pre-deploy checklist (tests, bundle, secrets, rules)
2. Schedule staging deploy (Monday)
3. Schedule production deploy (Friday 10:00 BRT)
4. Notify all teams of deploy schedule via Slack

---

### Post-Deploy (24h Hotline)

**Time:** Immediately after production deploy (Friday 10:00 BRT)  
**Owner:** DevOps on-call + Stream lead (standby)  
**Trigger:** Production deploy completion

**Actions:**
1. Monitor Cloud Logs (errors, latency, quota)
2. Run automated smoke tests (3 flows)
3. If error rate >1%: escalate to P0, consider rollback
4. Hourly status updates (hours 1–4)
5. Daily monitoring (hours 5–24)
6. Submit deployment report (next morning)

**Hourly report template:**
```

Latency p95: [Yms]
Quota usage: [Z]%
New issues: [None / list]

Status: ✓ GREEN | ⚠️ YELLOW (investigating) | 🔴 RED (rollback considered)
```

## Escalation SLA (If Update Misses Schedule)

### P0 — Master Matrix Update Missed (>4h overdue)

**Trigger:** Last update timestamp >4h old AND executor reports in Slack awaiting consolidation

**Action:**
1. Consolidator checks Slack for reports (manually, if auto-fail)
2. If reports present: immediate manual update
3. If reports absent: contact stream leads (ping @stream-lead in Slack)
4. Escalate to CTO if no response within 30 min

**Recovery:** Update published within 1h of escalation

---

### P1 — CTO Summary Missed (>30 min overdue by 08:00 BRT)

**Trigger:** 08:30 BRT and CTO summary not updated

**Action:**
1. Consolidator checks master matrix (is it updated?)
2. If yes: regenerate CTO summary immediately
3. If no: escalate P0 (see above)
4. Notify CTO via email "Summary delayed, published by 09:00"

**Recovery:** CTO summary published by 09:00 BRT (latest)

---

### P2 — Engineer Task Breakdown Missed (>30 min overdue by 09:00 BRT)

**Trigger:** 09:30 BRT and standdup not collected

**Action:**
1. Post in Slack `#v1-4-exec`: "Daily standup: [waiting for reports from Stream A, B, C]"
2. @mention stream leads (2-min reminder)
3. If no response by 10:00: mark sections as "pending" + publish anyway

**Recovery:** Updated doc by 10:00 BRT with best-available data

---

### P2 — Auditor Checklist Missed (>1h overdue by 14:30 Friday)

**Trigger:** 15:30 Friday and auditor doc not emailed

**Action:**
1. Consolidator generates snapshot from master matrix
2. Email to auditor + QA Lead + CTO by 16:00 Friday
3. Note: "Weekly briefing delayed due to phase activity"

**Recovery:** Email sent by 16:00 Friday (latest), standing call proceeds as scheduled

## Manual Overrides & Emergency Updates

### Real-Time Escalation (P0/P1 Blocker)

**Trigger:** Slack message in `#v1-4-risks` tagged with 🔴 P0 or 🟠 P1

**Action (Consolidator):**
1. Immediate update to master framework (blocker section)
2. Notify CTO + relevant stream lead (mention in Slack)
3. Escalate email if P0 (subject: `[BLOCKER] Phase X - Title`)
4. Track resolution timestamp

**Example:**
```
[BLOCKER] Phase 4 Task 4.1 — Firestore index build stuck
Blocker: CAPA collection index "status_createdAt" still building (12h, expected 4h)
Impact: Phase 4 cannot proceed until index READY
Owner: DevOps, escalation: CTO
SLA: Response within 30 min, resolution within 2h
```
