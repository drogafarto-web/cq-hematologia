## Trust Boundaries

| Boundary                           | Description                                                                      |
| ---------------------------------- | -------------------------------------------------------------------------------- |
| User Input → Date/Email Fields     | User enters arbitrary strings; validated client-side before submission           |
| Export Initiation → Cloud Function | Wizard submits valid request via httpsCallable; job creation happens server-side |

## STRIDE Threat Register

| Threat ID | Category               | Component                                             | Disposition | Mitigation Plan                                                                                   |
| --------- | ---------------------- | ----------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------- |
| T-07-01   | Information Disclosure | Wizard stores format + date range in Zustand store    | mitigate    | Store never persisted to localStorage; only lives in memory during session                        |
| T-07-02   | Tampering              | User modifies startDate > endDate before submit       | mitigate    | Client-side validation rejects (startDate must be ≤ endDate). Duplicate check in Cloud Callable.  |
| T-07-03   | Denial of Service      | User opens wizard 100 times, each creates state slice | mitigate    | Zustand store is singleton; only one wizard instance in DOM. Each `open()` resets to clean state. |

</threat_model>
