<tasks>

### Task 1: convidarPaciente callable (1 dia)
- Auth: `isActiveMemberOfLab` + claim RT/recepção
- Input: `{ labId, email, cpf, nome, motivo }`
- Pipeline: gera token + cria invitation doc + envia email via Resend
- Idempotência por email + labId

**Acceptance:** Email entrega; token expira 7d.

### Task 2: aceitarConvitePaciente callable (2 dias)
- Input: `{ token, cpf, senha }`
- Pipeline:
  1. Valida token + expiresAt
  2. Confirma CPF match com invitation
  3. Cria/link Firebase Auth user
  4. setCustomUserClaims: `{ paciente: true, cpf, cpfHash: SHA256(cpf+salt) }`
  5. Cria doc em `/global-portal-paciente/pacientes/{uid}` com `labsAtuantes: [labId]`
  6. Update invitation `status: 'accepted'`

**Acceptance:** Custom claim em <1min; race conditions handled.

### Task 3: Self-signup alternativo (1 dia)
- Sem convite: paciente acessa `/portal-paciente/cadastro` direto
- Valida CPF format + email confirmation (Firebase Auth)
- Cria conta sem invitation; claim `paciente: true` (sem labsAtuantes inicialmente)
- Lab autoriza acesso (RT vê em `/liberacao/admin/pacientes` e aprova)
- Pendente até aprovação: dashboard mostra apenas "Aguardando aprovação"

**Acceptance:** Self-signup → aprovação RT → acesso liberado.

### Task 4: OnboardingFlow UI (1.5 dia)
- 4 steps animados:
  1. Token validation
  2. CPF input + format validation
  3. Senha (requisitos visíveis)
  4. Confirmação + redirect
- Dark-first, mobile-first; brand neutro

**Acceptance:** UX premium em 5 dispositivos.

### Task 5: PortalShell + routing (1 dia)
- AppBar: logo neutro + LabSwitcher + perfil dropdown
- Sidebar mobile = drawer
- Wrapper auth: redirect para login se sem claim

**Acceptance:** App-shell ≤30KB; lazy routes.

### Task 6: Dashboard sections (2 dias)
- `PacienteDashboard.tsx`: 4 cards (Reclamações ativas, Sugestões enviadas, Laudos recentes, Notificações não lidas)
- `MinhasReclamacoes.tsx`: lista paginada com status badges + SLA tracker
- `MinhasSugestoes.tsx`: lista com status + votos
- `MeusLaudos.tsx`: read-only via collectionGroup query (`pacienteId == uid`)
- `Notificacoes.tsx`: timeline de eventos (status changes, novos laudos, NPS pendentes)

**Acceptance:** LCP <2.5s em mobile; INP <200ms.

### Task 7: MeusDadosLgpd (LGPD Art. 18) (1.5 dia)
- `exportarMeusDadosLgpd` callable:
  - Query todas as reclamações, sugestões, satisfacao-respostas, laudos do paciente
  - Gera JSON estruturado + PDF formatado (Puppeteer pattern)
  - Upload Storage temporário (signed URL 24h)
  - Email com link
- `solicitarExclusaoLgpd` callable:
  - Cria doc em `/labs/{labId}/lgpd-requests/{id}` com `tipo: 'exclusao', status: 'pending'`
  - Notifica admin
  - Admin tem 15 dias úteis para processar (LGPD Art. 19)

**Acceptance:** Export funciona; request exclusão notifica admin.

### Task 8: Firestore rules + collectionGroup (1 dia)
- Bloco para portal-paciente:
  ```
  match /{path=**}/reclamacoes/{id} {
    allow read: if request.auth.token.paciente == true
      && resource.data.reclamante.cpfHash == request.auth.token.cpfHash;
  }
  match /{path=**}/sugestoes/{id} {
    allow read: if request.auth.token.paciente == true
      && resource.data.autorId == request.auth.uid;
  }
  // MeusLaudos via Plan 10 (collectionGroup laudos)
  match /global-portal-paciente/pacientes/{uid} {
    allow read: if request.auth.uid == uid;
    allow update: if request.auth.uid == uid;
    allow create, delete: if false;
  }
  ```
- Indexes collectionGroup `(cpfHash, criadoEm DESC)`

**Acceptance:** Rule tests verde; cross-uid blocked.
