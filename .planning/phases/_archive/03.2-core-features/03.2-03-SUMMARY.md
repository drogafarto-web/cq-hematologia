# Phase 03.2 Plan 03: Export Wizard + Job Queue Summary

4-step async export wizard (format → date range → email → review), real-time queue view, job status polling, signed URL downloads, plus batch/scheduled export extensions and full Cloud Functions backend.

## What Was Built

### Task 1: Types, Schema, Wizard State

- `src/features/export/types/index.ts` — core types: `ExportFormat` ('xlsx' | 'pdf' | 'csv'), `ExportJobStatus` ('queued' | 'processing' | 'completed' | 'failed'), `ExportJob`, `ExportRequest`, `ExportInitiateResponse`
- `src/features/export/types/Exports.ts` — wizard-specific `ExportWizardState` (4-step, includes `submittedJobId`)
- `src/features/export/services/exportSchema.ts` — Zod validation for Firestore job records using `z.coerce.date()` to handle Firestore Timestamps
- `src/features/export/services/jobStatusCodes.ts` — `JOB_STATUS_META` (label, bgClass, textClass, ringClass, animate per status) + `FORMAT_LABELS` + `FORMAT_VARIANT_LABELS`
- `src/features/export/hooks/useExportWizardState.ts` — Zustand store with `useExportWizardStore`, actions: open/close/setStep/nextStep/prevStep/setFormat/setStartDate/setEndDate/setSubmittedJobId/reset

### Task 2: ExportWizard Modal (4 Steps)

- `ExportWizard.tsx` — 4-step modal (expanded from planned 3). Includes keyboard trap (Escape/Tab), body-scroll lock, a11y `role="dialog"` + `aria-modal`, focus management via `dialogRef`, success view showing `jobId` after submission
- `ExportStep1.tsx` — format card selector (xlsx/csv), violet/emerald accent per card, badge labels, controlled via props (`selectedFormat` + `onSelect`)
- `ExportStep2.tsx` — date range pickers (ISO strings), defaulted to last 30 days, start ≤ end validation
- `EmailDeliveryStep.tsx` — step 3 (beyond plan), optional email delivery with toggle switch, regex validation, LGPD notice, `role="alert"` on errors
- `ExportStep3.tsx` — review step (step 4), shows format + date range, confirm triggers Cloud Callable
- `StepIndicator` — inline sub-component with progress line and checkmark SVGs (no emoji, per guidelines)

### Task 3: Export Callables + Job Polling Hooks

- `src/features/export/services/exportCallables.ts` — typed `HttpsCallable<ExportRequest, ExportInitiateResponse>` via `httpsCallable(functions, 'initiateExport')`
- `src/features/export/services/exportService.ts` — `ExportService` object with `initiateExport` method (alternative client for mobile)
- `src/features/export/hooks/useExportInitiate.ts` — `submit()` wrapper with loading/error state + `clearError()`
- `src/features/export/hooks/useExportJobs.ts` — named exports: `useExportJob(labId, jobId)` (onSnapshot single-job), helpers `isDownloadExpiringsome`, `formatFileSize`, `formatDuration`
- `src/features/export/hooks/useExportPolling.ts` — `useExportPolling({ labId, jobId, intervalMs, disabled })` with interval polling, auto-stop on terminal state ('completed' | 'failed'), `mountedRef` guard, immediate `refresh()` escape hatch

### Task 4: ExportQueueView + Download UI

- `ExportQueueView.tsx` — inline onSnapshot subscription (no separate hook), Firestore path `/labs/{labId}/export-jobs` ordered by `createdAt` desc, responsive: desktop table + mobile cards, `coerceJob()` handles Timestamp → Date conversion, empty/error/loading states
- `ExportStatusPoller.tsx` — headless component (returns null), delegates to `useExportPolling`, surfaces updates via `onUpdate`/`onError` callbacks
- `DownloadButton.tsx` — `<a href download>` with emerald/amber accent depending on expiry proximity, renders expired label when `new Date() > job.expiresAt`, file size sub-label

### Task 5: Cloud Functions Backend

All functions in `functions/src/modules/export/`:

| Function | Type | Purpose |
|---|---|---|
| `initiateExport.ts` | Cloud Callable | Auth check → validate format/dates → Firestore job doc → Pub/Sub publish → return jobId + estimatedMinutes |
| `exportWorker.ts` | (Pub/Sub worker stub) | Job pickup + XLSX generation dispatch |
| `backgroundWorker.ts` | Pub/Sub subscriber | Full worker: queued → processing → completed/failed; invokes generators, uploads to GCS, signs URL, emails if `emailRecipient` set |
| `batchExport.ts` | Cloud Callable | Creates multiple jobs in one call (batch by format list) |
| `scheduledExport.ts` | Scheduled CF | Weekly Pub/Sub trigger reading `/labs/{labId}/exportSchedule` |
| `emailDelivery.ts` | Module | `sendExportEmail()` — non-fatal, used by backgroundWorker |
| `xlsxGenerator.ts` | Module | SheetJS-based CIQ run export |
| `excelFormatter.ts` | Module | Conditional formatting (red=invalid, amber=pending, green=valid) |
| `csvGenerator.ts` | Module | RFC 4180, UTF-8 BOM CSV audit log |
| `pdfCompressor.ts` | Module | PDF compression helper |

### Beyond-Plan Extensions

- `BatchFormatSelector.tsx` + `useBatchExport.ts` — multi-format batch export checkbox UI with per-job onSnapshot status tracking
- `ScheduledExportConfig.tsx` + `useScheduledExport.ts` — weekly scheduled export config stored at `/labs/{labId}/exportSchedule`

## Implementation Decisions

| Decision | Rationale |
|---|---|
| Feature folder `export/` not `exports/` | Consistent with all other module naming in codebase |
| 4 steps instead of planned 3 | EmailDeliveryStep added for download link delivery — step is optional/skippable |
| `ExportFormat` as `'xlsx' | 'pdf' | 'csv'` | Generic type; UI cards in ExportStep1 express variant distinction; avoids type proliferation |
| `'queued'` not `'pending'` for initial status | Aligns with CF-side job lifecycle (`queued → processing → completed/failed`) |
| `ExportStatusPoller` is headless | ExportQueueView handles its own onSnapshot; ExportStatusPoller serves as a reusable interval-poller bridge for consumers that need a callback-based API |
| `DownloadButton` uses `<a>` not `document.createElement` | Native browser download UX without JS-side synthetic events; better a11y |
| Zod schema uses `z.coerce.date()` | Firestore SDK returns Timestamp objects; coercion avoids manual `.toDate()` calls at each read site |
| Signed URL expiry check in DownloadButton | Client-side check (`Date.now() > expiresAt`) is acceptable for UX; authoritative enforcement is Cloud Storage HMAC signature |
