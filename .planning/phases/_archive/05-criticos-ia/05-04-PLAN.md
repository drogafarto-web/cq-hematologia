### 1. Training Dataset Version Management

**Firestore collection:** `labs/{labId}/imuno-ias-dev/training-versions/{versionId}`

```typescript
interface TrainingVersion {
  id: string;                           // auto-generated, format: "v1.0", "v1.1"
  labId: string;
  version: string;                      // "1.0", "1.1", "1.2", etc.

  // Metadata
  createdAt: timestamp;
  createdBy: string;                    // user ID who initiated collection/versioning
  description: string;                  // e.g., "Baseline with 50 golden samples"

  // Sample composition
  goldenSampleCount: number;            // number of validated samples
  stripIds: string[];                   // references to imuno-strips/{stripId}

  // Data quality
  dataQualityMetrics: {
    totalSamples: number;
    uniqueEquipment: number;
    uniqueAnalytes: number;
    outlierCount: number;               // samples with >2σ from mean
    missingValueCount: number;
  };

  // Model performance (if tested)
  modelMetrics?: {
    accuracy: number;                   // % of correct classifications
    precision: number;                  // TP / (TP + FP)
    recall: number;                     // TP / (TP + FN)
    f1Score: number;                    // harmonic mean
    auc: number;                        // area under ROC curve
    validationDataSetSize: number;
    crossValidationFolds: number;       // typically 5
  };

  // Status & deployment
  status: "COLLECTING" | "READY" | "DEPLOYED" | "ARCHIVED";
  deployedAt?: timestamp;
  deployedBy?: string;
  notes: string;                        // audit notes, version change rationale

  // Compliance
  assinatura: LogicalSignature;
  criadoEm: timestamp;
  deletadoEm?: timestamp;
}

// Example: v1.0 baseline
{
  id: "training-v1-0",
  version: "1.0",
  createdAt: 2026-05-21T10:00:00Z,
  createdBy: "lab-admin-uid",
  goldenSampleCount: 50,
  stripIds: ["strip-001", "strip-002", ..., "strip-050"],
  modelMetrics: {
    accuracy: 0.91,
    precision: 0.89,
    recall: 0.93,
    f1Score: 0.91,
  },
  status: "DEPLOYED",
  deployedAt: 2026-05-21T11:00:00Z,
}
```

**Workflow for creating a new version:**

1. **Collection phase** (manual, ongoing):
   - Operator uploads strip images (Task 05-03)
   - Expert validates (golden sample) — mark `iGoldenSample: true` in `imuno-strips/{stripId}`
   - Collect N samples (e.g., 100 for v1.1, 300 for v1.2)
   - Total time: 1–4 weeks depending on lab volume

2. **Preparation phase** (semi-automated):
   - Call `prepareTrainingDataset(labId, targetVersion, goldenSampleIds[])`
   - Validate all samples have OCR results + reconciliation
   - Filter out poor quality (confidence <70%, anomaly flags)
   - Generate summary stats (sample count, analytes, equipment coverage)
   - Create `TrainingVersion` doc with status = "READY"

3. **Validation phase** (manual, expert):
   - Download training dataset export (CSV or JSON)
   - Run offline validation (cross-validation, confusion matrix)
   - Compute model metrics (accuracy, precision, recall, F1, AUC)
   - Update `TrainingVersion.modelMetrics`
   - Sign-off: `validationBy`, `validatedAt`

4. **Deployment phase** (gated):
   - Promote version: status "READY" → "DEPLOYED"
   - Set as active version for new OCR parsing
   - Log change in audit trail
   - Optionally run A/B test (Task 05-04-B) before full rollout

5. **Archive phase** (on demand):
   - Old versions: status "DEPLOYED" → "ARCHIVED"
   - Keep for historical reference + audit trail

export async function parseImmunologyStripWithVersioning(
  labId: string,
  imageUrl: string,
  equipment: 'yumizen-h550',
): Promise<ParsedStripResult> {
  // Get active model version for this lab
  const activeModelVersion = await getActiveModelVersion(labId);

  // Check A/B test assignment
  const userId = context.auth.uid;
  const isControlGroup = shouldAssignToControl(
    userId,
    activeModelVersion.abTestConfig?.controlGroupPercent || 0,
  );

  // Call parseImmunologyStrip (single Gemini call)
  const result = await parseImmunologyStrip(labId, imageUrl, equipment);

  // Add version tracking to result
  result.modelVersionId = isControlGroup
    ? activeModelVersion.previousVersionId
    : activeModelVersion.trainingVersionId;
  result.experimentId = activeModelVersion.abTestConfig?.experimentId;

  return result;
}

### 3. A/B Testing Framework

**Experiment tracking:** `imuno-ias-dev/{labId}/experiments/{experimentId}`

```typescript
interface ABTestExperiment {
  id: string;
  labId: string;
  name: string; // e.g., "v1.0 vs v1.1 baseline"
  trainingVersionIdControl: string; // old version
  trainingVersionIdTreatment: string; // new version

  // Experiment config
  startedAt: timestamp;
  endedAt?: timestamp;
  status: 'RUNNING' | 'COMPLETED' | 'ABORTED';
  controlGroupPercent: number; // e.g., 50
  treatmentGroupPercent: number; // e.g., 50

  // Results (aggregated daily)
  dailyMetrics?: {
    date: string; // "2026-05-22"
    control: {
      sampleCount: number;
      avgConfidence: number;
      avgAccuracy: number; // vs analyzer reference
      errorCount: number;
    };
    treatment: {
      sampleCount: number;
      avgConfidence: number;
      avgAccuracy: number;
      errorCount: number;
    };
  }[];

  // Final results
  winner?: 'CONTROL' | 'TREATMENT' | 'INCONCLUSIVE';
  confidenceLevel?: number; // statistical significance (0–1)
  summary: string;

  notes: string;
  assinatura: LogicalSignature;
}
```

**Daily metrics aggregation** (Cloud Function, scheduled nightly):

```typescript
export async function aggregateDailyABTestMetrics(
  labId: string,
  experimentId: string,
  date: string, // "2026-05-22"
) {
  // Query imuno-strips created on this date, filtered by experimentId
  const strips = await queryStripsForExperiment(labId, experimentId, date);

  const control = {
    sampleCount: 0,
    avgConfidence: 0,
    avgAccuracy: 0,
    errorCount: 0,
  };

  const treatment = {
    sampleCount: 0,
    avgConfidence: 0,
    avgAccuracy: 0,
    errorCount: 0,
  };

  for (const strip of strips) {
    const isControl = strip.modelVersionId === experiment.trainingVersionIdControl;
    const target = isControl ? control : treatment;

    target.sampleCount++;
    target.avgConfidence += strip.parseResult.overallConfidence;

    // Accuracy: compare OCR results to analyzer reference (if exists)
    if (strip.reconciliation) {
      target.avgAccuracy += strip.reconciliation.reconciliationScore / 100;
      if (!strip.reconciliation.isAcceptable) {
        target.errorCount++;
      }
    }
  }

  // Normalize averages
  control.avgConfidence /= control.sampleCount;
  control.avgAccuracy /= control.sampleCount;
  treatment.avgConfidence /= treatment.sampleCount;
  treatment.avgAccuracy /= treatment.sampleCount;

  // Store daily metrics
  await updateABTestDailyMetrics(labId, experimentId, date, { control, treatment });

  // Check for statistical significance (optional, v1.5)
  const significance = calculateSignificance(control, treatment);
  if (significance > 0.95) {
    console.log(
      `Experiment ${experimentId} has statistical winner: ${getWinner(control, treatment)}`,
    );
  }
}
```

### 5. Confidence Threshold Tuning

**Per-analyte confidence tuning:** `imuno-ias-dev/{labId}/config/confidence-thresholds/{analitoId}`

```typescript
interface ConfidenceThreshold {
  analitoId: string;
  labId: string;

  // Threshold values
  minimumConfidence: number; // e.g., 0.80 (80%)
  highConfidenceThreshold: number; // e.g., 0.95 (95%)

  // Decision logic
  // confidence >= highConfidenceThreshold → auto-accept OCR
  // minimumConfidence <= confidence < highConfidenceThreshold → review required
  // confidence < minimumConfidence → reject OCR, operator manual entry

  createdAt: timestamp;
  updatedAt: timestamp;
  updatedBy: string;

  // Metrics
  appliedToSampleCount: number;
  acceptanceRate: number; // % of results above minimum
  reviewRate: number; // % in gray zone

  notes: string;
}
```

**Threshold tuning workflow (Phase 5-04-C, optional):**

1. Run analysis on recent samples: per analyte, what confidence threshold minimizes error?
2. Recommendation: set `minimumConfidence` such that <5% error rate
3. Set `highConfidenceThreshold` such that >95% auto-acceptance without review
4. Operator can override per analyte (e.g., stricter for critical analytes)
