# Task 03-01 — Firestore Schema v1.4 Extensions

**Objective:** Create 5 new Firestore collections + indexes to support Phases 4–12 (portals, NOTIVISA, critical escalation, IA strip, laudo drafts).

**Owner:** Stream D (DB engineer)  
**Duration:** 2 days  
**Wave:** 1 (can start immediately, no dependencies)

---

## Collections to Create

### 1. `labs/{labId}/portal-configuracao/{docId}`
**Purpose:** Patient portal branding per lab.

**Fields:**
```
{
  logoCdnUrl: string (URL),
  primaryColor: string (hex: #RRGGBB),
  secondaryColor: string (hex: #RRGGBB),
  labelLaudo: string (default: "Resultado"),
  labelPaciente: string (default: "Paciente"),
  termsHTML: string (rich text),
  privacyHTML: string (rich text),
  updatedAt: timestamp,
  updatedBy: string (uid)
}
```

**Indexes:** None needed (labId already indexed at root)

---

### 2. `labs/{labId}/notivisa-outbox/events/{docId}`
**Purpose:** NOTIVISA queue for regulatory notifications (Art. 6º §1).

**Fields:**
```
{
  laudo_id: string (ref to laudos),
  patient_cpf: string (required, masked before sending),
  payload: object (Art. 6º §1 JSON structure),
  status: string (enum: PENDING | SENT | FAILED | DELIVERED),
  attempts: number (retry count, max 5),
  nextRetry: timestamp,
  createdAt: timestamp,
  sentAt: timestamp (nullable),
  error: string (nullable)
}
```

**Indexes:**
- Composite: `(labId, status, createdAt)` — for polling PENDING events
- Single: `createdAt` — for audit trail

---

### 3. `labs/{labId}/criticos-escalacoes/escalacoes/{docId}`
**Purpose:** Critical value escalation log (SMS/email tracking).

**Fields:**
```
{
  resultado_id: string (ref to run.resultados),
  threshold_config_id: string (ref to criticos config),
  analito: string (e.g., "potassium"),
  valor: number,
  limite_inferior: number,
  limite_superior: number,
  sms_sent_to: array of strings (phone numbers),
  email_sent_to: array of strings (emails),
  sla_minutes: number (target resolution time),
  resolved_at: timestamp (nullable),
  resolution_notes: string (nullable),
  createdAt: timestamp
}
```

**Indexes:**
- Composite: `(labId, createdAt)` — for trending dashboard
- Single: `resolved_at` — for SLA tracking

---

### 4. `labs/{labId}/imuno-ias-dev/images/{docId}`
**Purpose:** Strip image metadata for IA OCR training (Phase 9).

**Fields:**
```
{
  imageUrl: string (CDN URL to strip image),
  imageDim: object {
    width: number,
    height: number
  },
  classesDetected: array of strings (enum: IgG | IgM | IgA | Anti-TPO | ... ),
  confidence: number (0.0 to 1.0),
  model_version: string (e.g., "1.0-base", "1.1-tuned"),
  feedback: object (optional human label) {
    classes: array,
    correctedBy: string (uid),
    correctedAt: timestamp
  },
  createdAt: timestamp,
  batch_id: string (nullable, for training batches)
}
```

**Indexes:**
- Composite: `(labId, model_version, createdAt)` — for IA research pipeline
- Single: `batch_id` — for training batch queries

---

### 5. `labs/{labId}/laudos-draft/rascunhos/{docId}`
**Purpose:** Laudo edit state machine for RT portal (Phase 5).

**Fields:**
```
{
  laudo_id: string (ref to laudos),
  edited_by: string (uid, the RT user),
  content_json: object (laudo content snapshot),
  locked_until_ts: timestamp (pessimistic lock expiry),
  version: number (for conflict detection),
  status: string (enum: EDITING | LOCKED | PUBLISHED),
  updatedAt: timestamp,
  publishedAt: timestamp (nullable),
  draft_notes: string (internal RT notes, not visible to patient)
}
```

**Indexes:**
- Composite: `(labId, laudo_id)` — for draft lookup per laudo
- Single: `locked_until_ts` — for cleanup cron (expired locks)

## Firestore Indexes to Create

**Instructions:**
1. Go to Firebase Console → Firestore → Indexes
2. Create the following composite indexes manually or via `firebase deploy`:

| Collection | Fields | Order | Status |
|------------|--------|-------|--------|
| notivisa-outbox | (labId, status, createdAt) | ASC, ASC, DESC | Create |
| criticos-escalacoes | (labId, createdAt) | ASC, DESC | Create |
| imuno-ias-dev | (labId, model_version, createdAt) | ASC, ASC, DESC | Create |
| laudos-draft | (labId, laudo_id) | ASC, ASC | Create |
| laudos-draft | (labId, locked_until_ts) | ASC, ASC | Create |

**Expected build time:** <5 minutes per index (typically instant, unless large collections exist).
