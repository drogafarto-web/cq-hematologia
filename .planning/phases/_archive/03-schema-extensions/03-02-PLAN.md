## 5 Rules Blocks to Add

All blocks use existing role helpers: `isMemberOfLab(labId)`, `isAdminOrRT(labId)`, `isPatient(labId)`, `isServer()`.

### 1. Portal Access Rules

**Match:** `/labs/{labId}/portal-configuracao/{docId}`

```firestore-rules
match /portal-configuracao/{docId} {
  // Patient: read lab portal config (for branding)
  allow read: if isPatient(labId) || isMemberOfLab(labId);
  // RT/Admin: write config (rebranding)
  allow write: if isAdminOrRT(labId) && request.resource.data.updatedBy == request.auth.uid;
}
```

**Match:** `/labs/{labId}/laudos/{laudoId}`

```firestore-rules
// Update: allow patient to read published laudos
allow read: if isPatient(labId)
  && resource.data.paciente_id == request.auth.uid
  && resource.data.publicado == true;
```

### 2. NOTIVISA Outbox Rules

**Match:** `/labs/{labId}/notivisa-outbox/events/{docId}`

```firestore-rules
match /notivisa-outbox/events/{docId} {
  // Admin/RT: create events (trigger NOTIVISA send)
  allow create: if isAdminOrRT(labId)
    && validateNotivisaPayload(request.resource.data);

  // Server only: read and update status (polling, retry)
  allow read, update: if isServer();

  // Audit: allow read for RT (view send history)
  allow read: if isMemberOfLab(labId);
}
```

**Helper function to add:**

```firestore-rules
function validateNotivisaPayload(payload) {
  return payload.laudo_id != null
    && payload.patient_cpf != null
    && payload.payload != null
    && payload.status in ['PENDING', 'SENT', 'FAILED'];
}
```

### 3. Critical Escalations Rules

**Match:** `/labs/{labId}/criticos-escalacoes/escalacoes/{docId}`

```firestore-rules
match /criticos-escalacoes/escalacoes/{docId} {
  // Admin/RT: create escalation events
  allow create: if isAdminOrRT(labId);

  // Member: read (for trending dashboard)
  allow read: if isMemberOfLab(labId);

  // Admin/RT: update resolution
  allow update: if isAdminOrRT(labId)
    && request.resource.data.resolved_at != null;
}
```

### 4. IA Strip Dev Collection Rules

**Match:** `/labs/{labId}/imuno-ias-dev/images/{docId}`

```firestore-rules
match /imuno-ias-dev/images/{docId} {
  // Server/Admin only: all access (IA training pipeline)
  allow read, write: if isServer() || isAdmin(labId);
}
```

### 5. Laudo Draft Rules

**Match:** `/labs/{labId}/laudos-draft/rascunhos/{docId}`

```firestore-rules
match /laudos-draft/rascunhos/{docId} {
  // Admin/RT: create, read, write (edit laudo)
  allow create, write: if isAdminOrRT(labId)
    && validateDraftLock(request);

  // Admin/RT: read drafts
  allow read: if isMemberOfLab(labId);

  // Patient: read (patient portal can show draft status)
  allow read: if isPatient(labId);
}
```

**Helper function to add:**

```firestore-rules
function validateDraftLock(request) {
  // Pessimistic lock: check if locked_until_ts > now
  return request.resource.data.locked_until_ts > request.time
    || request.resource.data.locked_by == request.auth.uid;
}
```
