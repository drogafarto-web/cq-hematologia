### Key Definitions & Constraints

**Multi-tenant paths (RN-MULTI-01):**

```
labs/{labId}/
├── auditorias-internas/{auditoriaId}
│   └── sessoes/{sessaoId}
│       └── checklist-items/{itemId}
│       └── achados/{achadoId}
```

**Type hierarchy:**

- Auditoria: annual audit plan + metadata
- Sessao: single audit execution (can have multiple sessions per annual plan)
- ChecklistItem: individual DICQ requirement (immutable after creation, just responses)
- Achado: finding from a checklist item (references back to item + session)
- LogicalSignature: { hash, operatorId, ts } for audit trail

**Soft-delete only (RN-06):** Never use `deleteDoc`. All entities have `deletadoEm: null | Timestamp`.

**Callable boundary:** Writes to auditorias-internas go via `createAuditoria`, `registerAchado` callables (already in Phase 0b functions). Client service is read-only subscribe + soft-delete via callable.

**From Obsidian checklist:** Load ~115 items from `Obsidian_Brain/01_Projetos/HC_Quality_Checklist_Auditoria.md` as seed data.

### Existing Spine References

- `/users/{userId}` — Pessoa (auditor, responsavelTecnico)
- `/labs/{labId}/naoConformidades/{ncId}` — NC spine (achados maiores link here)
- Callable `registerAchado` + `createAuditoria` already exist in functions/src/modules/auditoria/auditoria.ts

### Design System

Dark-first, Tailwind + design tokens. See `.claude/rules/performance.md` + root CLAUDE.md for tablet-optimized UX.

### Security Rules Pattern

From root CLAUDE.md + `.claude/rules/firestore-security.md`:

- `allow create/write: if isActiveMemberOfLab(labId) && hasClaim('auditor' or 'responsavelTecnico')`
- Hard delete forbidden (`allow delete: if false`)
- LogicalSignature validation on write: `validSignature(d)` — hash 64 chars, operatorId matches auth.uid, ts is timestamp
- `labIdMatches(d)` — document labId == path labId

Create `src/features/auditoria-interna/services/auditoriaService.ts` following "thin service, fat hooks" pattern (audit logic goes to hooks; service is mapping + subscription).

**Service exports:**

1. **subscribeAuditorias(labId, callback, onError)**: Real-time listener for all auditorias in lab (excludes deletedoEm != null)
   - Returns Unsubscribe function
   - Callback receives Auditoria[]
   - Uses onSnapshot with filter for deletadoEm == null client-side

2. **subscribeSessoes(labId, auditoriaId, callback, onError)**: Real-time listener for all sessões of an auditoria
   - Returns Unsubscribe function
   - Filter deletadoEm == null

3. **subscribeChecklistItems(labId, sessaoId, callback, onError)**: Real-time listener for all checklist items in a session
   - Returns Unsubscribe function
   - Ordered by numeroDICQ (e.g., "4.1.1.2" lexicographic)

4. **subscribeAchados(labId, sessaoId, callback, onError)**: Real-time listener for all findings in a session
   - Returns Unsubscribe function

5. **softDeleteAuditoria(labId, auditoriaId, motivo?: string)**: Mark auditoria as deleted (callable-backed)
   - Calls `functions.httpsCallable('deleteAuditoria')({labId, auditoriaId, motivo})`
   - Soft-delete only (never deleteDoc)

6. **softDeleteSessao(labId, sessaoId, motivo?: string)**: Mark sessão as deleted

7. **softDeleteAchado(labId, achadoId, motivo?: string)**: Mark achado as deleted

**Multi-tenant guard:**

```typescript
async function ensureLabRoot(labId: string) {
  const ref = doc(db, `labs/${labId}`);
  await setDoc(ref, {}, { merge: true });
}
```

Call this once per operation to ensure `/labs/{labId}` doc exists (idempotent).

**Firestore paths (strict):**

- Auditorias: `collection('labs').doc(labId).collection('auditorias-internas')`
- Sessões: `collection('labs').doc(labId).collection('auditorias-internas').doc(auditoriaId).collection('sessoes')`

**Collection:** `labs/{labId}/auditorias-internas`

**Rules logic:**

```
match /labs/{labId}/auditorias-internas/{auditoriaId} {
  allow read: if isActiveMemberOfLab(labId);
  allow create: if false;  // Phase 0b: via callable only
  allow update: if false;  // Phase 0b: via callable only
  allow delete: if false;  // RN-06: soft-delete only

  // Nested: sessoes, checklist-items, achados (same pattern)
  match /sessoes/{sessaoId} {
    allow read: if isActiveMemberOfLab(labId);
    allow create: if false;
    allow update: if false;
    allow delete: if false;

    match /checklist-items/{itemId} {
      allow read: if isActiveMemberOfLab(labId);
      allow create: if false;
      allow delete: if false;
    }

    match /achados/{achadoId} {
      allow read: if isActiveMemberOfLab(labId);
      allow create: if false;
      allow delete: if false;
    }
  }
}
```

**Notes:**

- All writes via callable (Phase 0b). Client cannot create/update directly.
- Read permission: any active member of the lab
- Soft-delete: `deletadoEm != null` is filtered client-side; no need for server-side rule
- No `validSignature()` check here (callables validate server-side with Admin SDK)

Per `.claude/rules/firestore-security.md`: audit trail is immutable (soft-delete only), reads allowed to all members (filtering by role happens in UI), writes go through callables (Admin SDK signs with server timestamp).

Test rules before deploy: `firebase emulators:exec --only firestore "npm test"`

<threat_model>

## Trust Boundaries

| Boundary                                                | Description                                                | Risk                                                                                              |
| ------------------------------------------------------- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| Client → Callable (`registerAchado`, `createAuditoria`) | Client sends audit findings; callable validates and signs  | Caller could forge achado; mitigated by `operatorId == request.auth.uid` in callable + chain-hash |
| Auditoria → NC spine                                    | Achado `grave`/`crítica` auto-creates NC; NC is regulatory | Achado chain broken; NC never links; mitigated by callable + atomic batch write                   |
| Client → Firestore (read)                               | Auditor reads checklist items in-loco; offline possible    | Stale data; mitigated by onSnapshot + localStorage sync on online                                 |

## STRIDE Threat Register

| ID      | Category               | Component                 | Disposition | Mitigation                                                                                         |
| ------- | ---------------------- | ------------------------- | ----------- | -------------------------------------------------------------------------------------------------- |
| T-05-01 | Spoofing               | `registerAchado` callable | mitigate    | `request.auth.uid` must match `operatorId` in payload; rules validate                              |
| T-05-02 | Tampering              | Achado + LogicalSignature | mitigate    | SHA-256 hash immutable after creation; soft-delete only (never hard delete)                        |
| T-05-03 | Repudiation            | Audit trail               | mitigate    | `operadorId + ts` on every write; auditLogs collection append-only                                 |
| T-05-04 | Information Disclosure | Checklist items (public)  | accept      | DICQ items are regulatory, not PII; read available to all lab members                              |
| T-05-05 | Denial of Service      | Callable spam             | mitigate    | Cloud Functions rate-limiting (project-level); callable validation (fail fast on invalid input)    |
| T-05-06 | Elevation of Privilege | Callable access           | mitigate    | `isActiveMemberOfLab(labId)` check; role-based perms in UI (enforce in Cloud Function, not client) |

</threat_model>
