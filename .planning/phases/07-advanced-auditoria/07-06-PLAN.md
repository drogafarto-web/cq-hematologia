## SA-19 — `functions/src/modules/qualidade/scheduledReportJob.ts`

**Modelo:** Haiku  
**Dependências:** `reportGenerator.ts` (SA-14)  
**Contrato:** Cloud Function cron que dispara geração de relatório diário/semanal/mensal para cada lab.

**Interface a implementar:**

```typescript
import { onSchedule } from 'firebase-functions/v2/scheduler';
import { generateAuditReport } from './reportGenerator';

// Relatório diário — todo dia às 06:00 BRT (09:00 UTC)
export const dailyAuditReport = onSchedule(
  { schedule: '0 9 * * *', region: 'southamerica-east1', timeZone: 'America/Sao_Paulo' },
  async () => { ... }
);

// Relatório semanal — toda segunda às 07:00 BRT
export const weeklyAuditReport = onSchedule(
  { schedule: '0 10 * * 1', region: 'southamerica-east1', timeZone: 'America/Sao_Paulo' },
  async () => { ... }
);

// Relatório mensal — dia 1 de cada mês às 08:00 BRT
export const monthlyAuditReport = onSchedule(
  { schedule: '0 11 1 * *', region: 'southamerica-east1', timeZone: 'America/Sao_Paulo' },
  async () => { ... }
);
```

**Lógica de cada handler:**
1. Buscar todos os labs ativos: `db.collection('labs').where('status', '==', 'active')`
2. Para cada lab, verificar se `labs/{labId}/config/reports.scheduledReports.{period}` é `true`
3. Calcular `startDate` e `endDate` baseado no período
4. Chamar `generateAuditReport` via admin SDK (não via callable HTTP)
5. Logar sucesso/falha por lab (não quebrar se 1 lab falhar)
6. Try/catch por lab — falha de um não cancela os outros

**Regras:**
- Usar `firebase-functions/v2/scheduler` (não v1)
- `timeZone: 'America/Sao_Paulo'`
- Sem email nesta versão (downloadUrl no report é suficiente para V1)
- ≤100 LOC

**Done:**
- [ ] `scheduledReportJob.ts` exporta `dailyAuditReport`, `weeklyAuditReport`, `monthlyAuditReport`
- [ ] Cron strings válidas (formato cron `* * * * *`)
- [ ] `npx tsc --noEmit` sem erros
