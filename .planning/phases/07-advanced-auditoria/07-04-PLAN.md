## Report Structure

**Page 1: Executive Summary**
- Lab name, report period, generated date
- Key metrics: total audit entries, anomalies flagged, critical alerts, dismissal rate
- NLP-generated summary (150–200 words): "During the period {date1} to {date2}, the lab conducted {X} operations across {Y} modules. Anomaly detection identified {Z} suspicious activities, of which {A} were dismissed as false positives and {B} remain under investigation. Key findings: [1–3 bullet points from Gemini]"
- Compliance status: RDC 978 Article alignment

**Page 2–N: Detailed Audit Trail**
- Table: Timestamp, Operator, Module, Operation, Result, Anomaly Score (if flagged), Dismissal Status
- Grouped by module (collapsible sections)
- Color-coded anomaly severity (red/orange/yellow)

**Page N+1: Anomaly Analysis**
- Summary: "Detected {X} anomalies. {A} high-confidence, {B} medium-confidence, {C} low-confidence"
- Table: Anomaly ID, Operator, Score, Dimensions breakdown, AI Insight summary, Status (flagged/dismissed)

**Page N+2: RDC 978 Compliance Mapping**
- Table: Article → Findings (linked audit entries)
- Example: "Art. 107 — All regulatory writes audited. {X} audit entries across {Y} operators with signature validation."

## Report Export Formats

**PDF**: Single-page summary + multi-page details (via Cloud Function using a PDF library — check if project has puppeteer or similar)
**CSV**: Raw audit data (Excel-compatible)

## Compliance Mapping

- **RDC 978 5.3** — Report must include audit trail summary + signature validation
- **RDC 978 Art. 107** — Regular audit reviews with documented findings
- **DICQ 4.4** — Trilha de auditoria with comprehensive reporting

Create a Cloud Function callable that generates complete audit reports with user-specified filters and exports to PDF/CSV.

1. Export function (Cloud Function callable): `generateAuditReport(filters: ReportFilters): Promise<{ reportId: string; summary: SummaryResult; dataCount: number; pdfUrl: string; csvUrl: string }>`

2. Input type:
   ```typescript
   interface ReportFilters {
     labId: string;
     startDate: Date;
     endDate: Date;
     modules?: string[];  // filter by module (optional, null = all modules)
     operators?: string[];  // filter by operator (optional, null = all operators)
     severities?: ('critical' | 'high' | 'medium')[];  // include only anomalies with these severities
     includeDismissed?: boolean;  // default false (only flagged anomalies)
   }
   ```

3. Report generation steps:
   a. Query audit trail: `labs/{labId}/audit-trail` filtered by dateRange + modules + operators
   b. Query anomaly flags: `labs/{labId}/anomaly-flags` filtered by severity + dismissal status
   c. Query anomaly insights: join with anomaly-flags to get AI insights
   d. Compute metrics: total entries, anomalies, critical count, dismissal count
   e. Extract key data: top operators, affected modules, common operations
   f. Generate NLP summary: call `summarizeAuditFindings()`
   g. Create PDF report: use a PDF library (check if project has puppeteer, @react-pdf/renderer, or similar) or Cloud Function HTML-to-PDF service
   h. Create CSV export: stream audit data to CSV file
   i. Upload both to Cloud Storage: `gs://hmatologia2-audit-reports/{labId}/{reportId}/report.pdf` and `.csv`
   j. Return signed URLs (1-hour expiry) for direct download

4. Storage of report metadata:
   ```typescript
   interface AuditReport {
     id: string;  // report ID
     labId: string;
     filters: ReportFilters;
     summary: SummaryResult;
     dataCount: number;
     pdfUrl: string;  // signed URL
     csvUrl: string;  // signed URL
     generatedBy: string;  // operator ID
     generatedAt: Timestamp;
     expiresAt: Timestamp;  // 1 hour from now
   }
   ```
   Store in: `labs/{labId}/audit-reports/{reportId}`

5. Scheduled reports:
   - Export function: `scheduleReport(config: ScheduleConfig): Promise<{ scheduleId: string }>`
   - Config: `{ labId, frequency: 'daily'|'weekly'|'monthly', filters: ReportFilters, emailRecipients: string[], timezone: string }`
   - Use Cloud Scheduler (Cloud Functions scheduled) to invoke `generateAuditReport()` at specified intervals
   - Auto-email PDF to recipients upon completion

6. Performance optimization:
   - Query limits: max 5000 audit entries per report (if >5000, paginate with cursor)
   - Caching: if same filters run within 1 hour, reuse PDF/CSV (don't regenerate)
   - Parallel queries: fetch audit-trail, anomaly-flags, anomaly-insights in parallel (Promise.all)

Implementation notes:
- PDF generation: if project lacks a PDF library, use Cloud Function that renders HTML template and calls a PDF service (e.g., Wkhtmltopdf or Google Docs API)
- CSV: use a simple array-to-CSV converter (no large library needed)
- Cloud Storage: ensure proper access control (only lab members + AUDITOR role can read)
- Email: use Cloud Functions email integration or SendGrid (Phase 5 pattern)

Per performance.md: report generation must complete in <10 seconds for 1000 entries.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cloud Function → Report Generation | Trusted: CF queries audit data, generates report server-side |
| User → Cloud Storage | Untrusted: user cannot forge signed URLs; storage enforces access control |
| Gemini API → Summary | Untrusted: summary is advisory; no sensitive data included |
| Auditor → Schedule Config | Partially trusted: auditor specifies email recipients (validate email format) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-19 | Spoofing | Non-auditor generates report | Mitigate | Cloud Function checks `isActiveMemberOfLab() && role == 'AUDITOR'` |
| T-07-20 | Tampering | User modifies report data before export | Mitigate | Report generated server-side by CF; client cannot modify |
| T-07-21 | Repudiation | "I didn't request this report" | Accept | Report metadata includes `generatedBy` + `generatedAt` timestamp (logged for audit) |
| T-07-22 | Info Disclosure | Signed URL shared externally | Mitigate | URLs expire after 1 hour; access control in Cloud Storage (only lab members + AUDITOR) |
| T-07-23 | Denial of Service | Report generation overload | Mitigate | CF timeout: 5 min. Limit 1 report per user per 5 minutes (rate limit). Monitor Cloud Storage quota. |
| T-07-24 | Elevation | Non-AUDITOR schedules email | Mitigate | Cloud Function checks role before creating schedule. Email recipients must be lab members (validated). |

</threat_model>
