# Phase 2 Batch 2: RH + Infraestrutura

**Scope:** 5 modules, >80% test coverage, parallel execution with Batch 1  
**Inputs:** ADR 0006 (Qualificacao), ADR 0004 (POPs), ADR 0007 (Equipamento)  
**Outputs:** 5 modules live, training system + biosafety + compliance dashboard

---

## Tasks

### Task 1: Treinamentos Module

**Duration:** 2 weeks  
Build training registry per POP version with expiry validation.

**Components:**

- TreinamentoService: registrarTreinamento, checkTreinamentoValido
- TrainingList: per operator, per POP version, expiry tracking
- EvidenceUpload: upload training certificates

**Integration:** Extends ADR 0006 validation (qualificacoes → treinamentosPOP)

**Tests:** >80% (expiry logic, validation gate)

**Commit:** "Batch 2 Task 1: Treinamentos Module complete"

---

### Task 2: Biossegurança Module

**Duration:** 2 weeks  
Area mapping with biosafety level (NB1-NB3) and PPE tracking.

**Components:**

- BiossegurancaService: mapearArea, validarBioseguracaArea
- AreaMapping: sala, fluxo diagram, NB level, EPE matrix
- EPECheck: daily sign-off for operators

**Tests:** >80% (NB validation, EPE requirements)

**Commit:** "Batch 2 Task 2: Biossegurança Module complete"

---

### Task 3: PGRSS Module

**Duration:** 2 weeks  
Waste management registry with segregation tracking.

**Components:**

- PGRSSService: registrarGeracaoResiduo, registrarColeta
- WasteRegistry: generation log, segregation checklist, collection tracking
- ComplianceReport: monthly waste report

**Tests:** >80% (waste types, segregation validation)

**Commit:** "Batch 2 Task 3: PGRSS Module complete"

---

### Task 4: KPIs Dashboard

**Duration:** 2-3 weeks  
Real-time metrics aggregation from all modules.

**Metrics:**

- Turnaround (run creation → result release)
- Rework % (repeat runs per equipment/method)
- NC origins (which modules generate NCs)
- Compliance % (all runs with popId + equipId + operadorId)

**Components:**

- KPIDashboard: charts (line, pie, bar)
- ScheduledComputeKPIs: daily aggregation Cloud Function
- AlertThresholds: warning if turnaround > SLA

**Tests:** >80% (aggregation logic, threshold alerts)

**Commit:** "Batch 2 Task 4: KPIs Dashboard complete"

---

### Task 5: LGPD + Data Deletion

**Duration:** 2-3 weeks  
Policy portal + automated deletion processor.

**Components:**

- LGPDPolicyPortal: rights explanation, deletion request form
- DeletionProcessor: anonymize + archive (scheduled Cloud Function)
- DPIATemplate: data impact assessment
- AuditTrail: log all deletions for compliance

**Tests:** >80% (anonymization verification, audit logging)

**Commit:** "Batch 2 Task 5: LGPD Module complete"

---

### Task 6: Rules Deployment + Integration

**Duration:** 1 week  
Deploy all 5 modules' rules, run cross-module tests.

**Smoke Scenarios:**

- Create audit → KPIs dashboard updates
- Operator training expires → blocked from running
- LGPD deletion → user anonymized in reports

**Commit:** "Batch 2 Rules deployed + integration tests pass"

---

### Task 7: Production Deployment

**Duration:** 3-5 days  
Final build, deploy, verification.

**Output:** BATCH2-COMPLETION.md with metrics

**Commit:** "Batch 2 Complete: RH + Infra live (5 modules, >80% coverage)"
