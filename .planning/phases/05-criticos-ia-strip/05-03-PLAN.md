**Gemini integration:**
```typescript
import { GoogleGenerativeAI } from '@google/generative-ai';

const genAI = new GoogleGenerativeAI(process.env.GOOGLE_API_KEY);
const model = genAI.getGenerativeModel({ model: 'gemini-2.5-flash' });

const response = await model.generateContent([
  {
    inlineData: {
      mimeType: 'image/jpeg',
      data: base64ImageData,
    },
  },
  {
    text: `Classify this rapid test strip result for ${testKit}. 
    Respond ONLY with JSON: { "classification": "Positive" | "Negative" | "Invalid" | "Grey-Zone", "confidence": 0.0-1.0, "reasoning": "..." }`,
  },
]);
```

**Response schema:**
```typescript
interface GeminiClassificationResponse {

  confidence: number; // 0.0 to 1.0
```

### REQ-03.3: Confidence Validation Pipeline

**Flow:**
```
Gemini returns classification + confidence
  ↓
Check confidence >= 0.85 (configurable threshold)
  ├─ If YES:
  │   ├─ Populate imuno-run result field
  │   ├─ Mark as "IA-CLASSIFIED" in audit
  │   └─ Return { success: true; result }
  └─ If NO (confidence < 0.85):
      ├─ Flag for RT manual review
      ├─ Return { success: false; confidence; reason: "LOW_CONFIDENCE" }
      ├─ Send in-app notification to RT
      └─ User retake image or override manually
```

**Confidence threshold config:**
```
/labs/{labId}/criticos-thresholds (or new /labs/{labId}/iaConfig)
  ├─ iaMinConfidence: number (default 0.85, range 0.7-0.99)
  └─ ativo: boolean
```

**Manual override:**
- RT can always override Gemini result
- Override logged in audit trail (operadorId, timestamp, reasoning)
- Feedback collected for model improvement (Task 05-04)

**Acceptance criteria:**
- [ ] Confidence threshold enforced (≥0.85 → auto-accept)
- [ ] Low confidence flagged for RT review
- [ ] Manual override always allowed (button on result)
- [ ] Override logged in audit trail
- [ ] E2E test: <0.85 confidence → RT notification + manual review flow

### REQ-03.4: Dataset Collection + Labeling

**Schema:**
```
/labs/{labId}/imuno-ias-dev/{runId}
  ├─ imageUrl: string
  ├─ testKit: 'HIV' | 'Dengue' | 'Syphilis' | 'COVID' | 'HCG'
  ├─ uploadedAt: Timestamp
  ├─ uploadedBy: string
  ├─ geminiClassification: {
  │   ├─ classification: string
  │   ├─ confidence: number
  │   ├─ reasoning: string
  │   └─ timestamp: Timestamp
  │ }
  ├─ manualVerdict?: {
  │   ├─ classification: string (RT override)
  │   ├─ verifiedBy: string (operadorId)
  │   ├─ verifiedAt: Timestamp
  │   └─ reasoning: string
  │ }
  ├─ metadata: {
  │   ├─ lightingCondition: string
  │   ├─ testKitBatch: string
  │   └─ deviceInfo: object
  │ }
  └─ deletadoEm?: Timestamp (soft-delete)
```

**Collection strategy:**
- Target: 500+ labeled images per test kit (HIV, Dengue, syphilis, COVID, HCG)
- Diverse conditions: lighting (bright/dim), device (iPhone/Android/tablet), batch (multiple suppliers)
- Label rate: Every image goes through Gemini; RT provides feedback only if low confidence or override

**Export format (Task 05-04):**
```json
{
  "dataset_version": "1.0",
  "export_date": "2026-05-31",
  "lab_id": "riopomba",
  "test_kit": "HIV",
  "total_images": 150,
  "accuracy_baseline": 0.88,
  "samples": [
    {
      "image_id": "abc123",
      "image_url": "gs://...",
      "gemini_classification": "Positive",
      "gemini_confidence": 0.92,
      "manual_verdict": "Positive",
      "match": true,
      "metadata": {...}
    }
  ]
}
```

**Acceptance criteria:**
- [ ] Dataset stored in Firestore (500+ images by phase end)
- [ ] All images labeled (Gemini + RT feedback)
- [ ] Export format matches ML team requirements
- [ ] Privacy: no patient PII in images or metadata
