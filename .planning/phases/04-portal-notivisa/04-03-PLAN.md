## Scope

### 1. NOTIVISA Regulatory Context

**RDC 978 Article 167** — Resultado de exame → **notificação obrigatória ao paciente**
- Within 24h of publication (deadline ~2026, enforcement TBD)
- Patient notification = evidence of access right (LGPD Art. 18)
- Methods: email + SMS + patient portal (v1.4: email + portal)

**RDC 978 Article 6º §1** — NOTIVISA sandbox integration (não-obrigatório em v1.4, prep para produção)
- "Portaria nº 204/2023" defines notification to gov regulatory system
- Scope: abnormal results + critical values + specific conditions (e.g., positive serology)
- Format: XML/JSON submission to `https://notivisa-sandbox.anvisa.gov.br/` (test environment)
- v1.4: queue + API client ready, **do not** submit to gov (sandbox test only)
- v1.5+: production API integration + auditor approval

**Payload structure (Portaria 204):**
```json
{
  "event_id": "uuid",
  "lab_cnpj": "xx.xxx.xxx/xxxx-xx",
  "result_id": "laudo.id",
  "patient_cpf": "xxx.xxx.xxx-xx",
  "analyte": "examename",
  "value": "xx",
  "ref_range": "xx-xx",
  "result_status": "abnormal|critical|normal",
  "timestamp": "2026-05-07T14:30:00Z",
  "signature": "HMAC-SHA256(payload, key)",
  "version": "1.0"
}
```

### 2. Queue Architecture & State Machine

**Collection: `notivisa-outbox`** (from Phase 3 schema)

```
/labs/{labId}/notivisa-outbox/
  ├── metadata/ {
        avgProcessingTimeMs: number,
        totalProcessed: number,
        totalFailed: number,
        lastProcessedAt: timestamp,
        sandboxApiUrl: string,
        productionApiUrl: string (null in v1.4),
        sandboxApiKey: string (from Firebase secrets)
      }
  └── events/
        ├── [uuid1] {
              laudo_id: string,
              patient_cpf_hash: string (HMAC-SHA256),
              payload: { /* Portaria 204 JSON */ },
              status: "PENDING" | "PROCESSING" | "SENT" | "FAILED" | "DELIVERED",
              attempts: number (0–5),
              nextRetry: timestamp (null if delivered),
              createdAt: timestamp,
              sentAt: timestamp (nullable),
              deliveredAt: timestamp (nullable),
              error: string (nullable, last error message),
              response_code: number (nullable, HTTP status),
              response_body: string (nullable, truncated to 1KB),
              sandboxTest: boolean (true if v1.4),
              operator_signature: { hash, operatorId, ts }
            }
        ├── [uuid2] { ... }
```

**State transitions:**

```
PENDING → PROCESSING → SENT → DELIVERED
   ↓                    ↓
 (error) ────────────→ FAILED
   ↑
   └─ retry every 5min (attempts < 5)
   └─ after 5 attempts, stop (manual intervention required)
```

**Trigger:** When laudo status → "published" + `visibilidadePaciente === true`:
1. Cloud Function `onLaudoPublished` fires
2. Create event doc in `notivisa-outbox/events/{newId}`
3. Set `status: PENDING`, `createdAt: now`
4. Enqueue task for processor cron

### 3. Queue Processor Cron

**Function:** `processNotiVisaQueue` (Cloud Function, scheduled hourly via Cloud Scheduler)

**Logic:**

```typescript
// functions/src/callables/notivisa/processNotiVisaQueue.ts

interface NotiVisaEvent {
  laudo_id: string;
  patient_cpf_hash: string;
  payload: NotiVisaPayload;
  status: 'PENDING' | 'PROCESSING' | 'SENT' | 'FAILED' | 'DELIVERED';
  attempts: number;
  nextRetry?: Timestamp;
  createdAt: Timestamp;
  error?: string;
}

export async function processNotiVisaQueue(
  req: functions.https.Request,
  res: functions.Response
) {
  const labId = req.query.labId as string;
  if (!labId) {
    return res.status(400).json({ error: 'Missing labId' });
  }

  const db = admin.firestore();
  const outboxRef = db.collection('notivisa-outbox')
    .doc('events')
    .collection('queue');

  try {
    // 1. Fetch all PENDING events (batch)
    const pendingSnap = await outboxRef
      .where('status', '==', 'PENDING')
      .where('nextRetry', '<=', admin.firestore.Timestamp.now())
      .limit(10) // batch size
      .get();

    const results = {
      totalProcessed: 0,
      totalSent: 0,
      totalFailed: 0,
      errors: [] as Array<{ eventId: string; error: string }>,
    };

    // 2. Process each event
    for (const eventDoc of pendingSnap.docs) {
      const event = eventDoc.data() as NotiVisaEvent;
      const eventId = eventDoc.id;

      try {
        // Update status to PROCESSING
        await eventDoc.ref.update({
          status: 'PROCESSING',
          attempts: (event.attempts || 0) + 1,
        });

        // 3. Call NOTIVISA sandbox API
        const apiResponse = await callNotiVisaSandboxAPI(event.payload, labId);

        // 4. Check response
        if (apiResponse.statusCode === 202) {
          // 202 Accepted = successfully queued by gov system
          await eventDoc.ref.update({
            status: 'DELIVERED',
            sentAt: admin.firestore.Timestamp.now(),
            deliveredAt: admin.firestore.Timestamp.now(),
            response_code: apiResponse.statusCode,
            response_body: apiResponse.body?.substring(0, 1024),
          });
          results.totalSent++;
        } else if (apiResponse.statusCode >= 400 && apiResponse.statusCode < 500) {
          // 4xx = client error, don't retry
          await eventDoc.ref.update({
            status: 'FAILED',
            error: `API returned ${apiResponse.statusCode}: ${apiResponse.body}`,
            response_code: apiResponse.statusCode,
          });
          results.totalFailed++;
        } else {
          // 5xx = server error, retry
          const nextRetry = new Date();
          nextRetry.setMinutes(nextRetry.getMinutes() + 5);
          await eventDoc.ref.update({
            status: 'PENDING',
            nextRetry: admin.firestore.Timestamp.fromDate(nextRetry),
            error: `API returned ${apiResponse.statusCode}`,
          });
        }
      } catch (err) {
        // Network error, retry
        const nextRetry = new Date();
        nextRetry.setMinutes(nextRetry.getMinutes() + 5);
        await eventDoc.ref.update({
          status: 'PENDING',
          nextRetry: admin.firestore.Timestamp.fromDate(nextRetry),
          error: `Error: ${err instanceof Error ? err.message : String(err)}`,
        });
        results.errors.push({
          eventId,
          error: err instanceof Error ? err.message : String(err),
        });
      }

      results.totalProcessed++;
    }

    // 5. Log summary + send alert if failures
    console.log(`[NOTIVISA] Processed ${results.totalProcessed} events: ` +
      `${results.totalSent} sent, ${results.totalFailed} failed`);

    if (results.totalFailed > 0) {
      // Alert ops (email / Slack webhook)
      await alertOps(
        `NOTIVISA: ${results.totalFailed}/${results.totalProcessed} failed`,
        results.errors
      );
    }

    return res.json(results);
  } catch (err) {
    console.error('[NOTIVISA] Queue processor failed:', err);
    return res.status(500).json({ error: String(err) });
  }
}
```

**Retry strategy:**
- Exponential backoff: 5min → 10min → 20min → 40min → 60min
- Max 5 attempts (if still failing after 5 attempts, manual intervention required)
- Log every retry (audit trail)

**Scheduled deployment:**
- Cloud Scheduler trigger: `0 * * * *` (hourly, on the hour)
- Region: `southamerica-east1` (consistent with other functions)
- Timeout: 120s (should complete <10s for batch of 10)
- Retries: 2 on failure (Cloud Scheduler auto-retry)

### 4. NOTIVISA Sandbox API Client

**Function:** `callNotiVisaSandboxAPI` (helper, `functions/src/shared/notivisa.ts`)

```typescript
// functions/src/shared/notivisa.ts

interface NotiVisaPayload {
  event_id: string;
  lab_cnpj: string;
  result_id: string;
  patient_cpf: string;  // hashed in queue, plaintext in payload
  analyte: string;
  value: string;
  ref_range: string;
  result_status: 'abnormal' | 'critical' | 'normal';
  timestamp: string;  // ISO 8601
  signature: string;  // HMAC-SHA256
  version: string;
}

export async function callNotiVisaSandboxAPI(
  payload: NotiVisaPayload,
  labId: string
): Promise<{
  statusCode: number;
  body?: string;
  headers: Record<string, string>;
}> {
  const apiKey = process.env.NOTIVISA_SANDBOX_API_KEY;
  if (!apiKey) {
    throw new Error('NOTIVISA_SANDBOX_API_KEY not configured');
  }

  const sandboxUrl = process.env.NOTIVISA_SANDBOX_URL ||
    'https://notivisa-sandbox.anvisa.gov.br/api/v1/notifications';

  try {
    const response = await fetch(sandboxUrl, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${apiKey}`,
        'X-Request-ID': payload.event_id,  // idempotency key
      },
      body: JSON.stringify(payload),
      timeout: 10000,  // 10s timeout
    });

    const body = await response.text();
    return {
      statusCode: response.status,
      body,
      headers: Object.fromEntries(response.headers),
    };
  } catch (err) {
    console.error('[NOTIVISA] API call failed:', err);
    throw err;
  }
}

export function generateNotiVisaPayload(
  laudo: Laudo,
  runResult: RunResult,
  labConfig: LabConfig
): NotiVisaPayload {
  // Map laudo → Portaria 204 format
  const resultStatus = runResult.valor > runResult.refMax ? 'abnormal'
    : runResult.valor < runResult.refMin ? 'abnormal'
    : 'normal';

  return {
    event_id: generateUUID(),
    lab_cnpj: labConfig.cnpj,
    result_id: laudo.id,
    patient_cpf: laudo.paciente.cpf,
    analyte: runResult.analito,
    value: String(runResult.valor),
    ref_range: `${runResult.refMin}-${runResult.refMax}`,
    result_status: resultStatus,
    timestamp: new Date().toISOString(),
    signature: generateNotiVisaSignature(laudo, runResult),
    version: '1.0',
  };
}

function generateNotiVisaSignature(laudo: Laudo, result: RunResult): string {
  const message = `${laudo.id}${laudo.paciente.cpf}${result.analito}${result.valor}`;
  const key = process.env.NOTIVISA_SIGNATURE_KEY || 'dev-sandbox-key';
  return crypto
    .createHmac('sha256', key)
    .update(message)
    .digest('hex');
}
```

### 5. Triggers & Event Hooks

**Trigger 1: On laudo publication** (in existing `laudoService.publishLaudo` or via Firestore trigger)

```typescript
// functions/src/triggers/onLaudoPublished.ts

export const onLaudoPublished = functions.firestore
  .document('labs/{labId}/laudos/{laudoId}')
  .onUpdate(async (change, context) => {
    const before = change.before.data();
    const after = change.after.data();
    const { labId } = context.params;

    // Only trigger if status changed to 'published'
    if (before.status !== 'published' && after.status === 'published') {
      // Enqueue NOTIVISA event for each result
      const db = admin.firestore();
      const outbox = db.collection('notivisa-outbox').doc('events');

      for (const result of after.resultados) {
        if (result.status === 'released') {
          const payload = generateNotiVisaPayload(after, result, /* labConfig */);
          await outbox.collection('queue').add({
            laudo_id: laudoId,
            patient_cpf_hash: hashCPF(after.paciente.cpf),
            payload,
            status: 'PENDING',
            attempts: 0,
            createdAt: admin.firestore.Timestamp.now(),
            operator_signature: {
              hash: computeHash(JSON.stringify(payload)),
              operatorId: context.auth?.uid || 'system',
              ts: admin.firestore.Timestamp.now(),
            },
          });
        }
      }
    }
  });
```

**Trigger 2: Manual re-queue** (admin action, optional)

```typescript
// functions/src/callables/notivisa/requeueNotiVisaEvent.ts

export const requeueNotiVisaEvent = functions.https.onCall(
  async (data: { eventId: string; labId: string }, context) => {
    if (!context.auth) throw new functions.https.HttpsError('unauthenticated', 'Not authenticated');

    const { eventId, labId } = data;
    const db = admin.firestore();
    const eventRef = db
      .collection(`labs/${labId}/notivisa-outbox`)
      .doc('events')
      .collection('queue')
      .doc(eventId);

    const snap = await eventRef.get();
    if (!snap.exists) {
      throw new functions.https.HttpsError('not-found', 'Event not found');
    }

    const event = snap.data()!;
    if (event.attempts >= 5) {
      throw new functions.https.HttpsError(
        'failed-precondition',
        'Max retries exceeded; manual intervention required'
      );
    }

    await eventRef.update({
      status: 'PENDING',
      nextRetry: admin.firestore.Timestamp.now(),
      attempts: 0,
    });

    // Log action
    await logAction(labId, context.auth.uid, 'notivisa-event-requeued', {
      eventId,
      previousAttempts: event.attempts,
    });

    return { success: true };
  }
);
```

### 6. Monitoring & Alerting

**Cloud Logs monitoring:**
- Filter: `resource.type="cloud_function" AND function_name="processNotiVisaQueue"`
- Alert triggers:
  - `ERROR` or `CRITICAL` severity
  - `totalFailed > 0` (check logs summary)
  - Function timeout (>120s)
  - Failed API responses (5xx persistent)

**Metrics to track (logged per run):**
- `totalProcessed` — events in batch
- `totalSent` — successfully delivered (202)
- `totalFailed` — client errors (4xx, unrecoverable)
- `avgProcessingTimeMs` — per event
- `failureRate` — `totalFailed / totalProcessed`

**Dashboard (optional):**
- Pub/Sub topic: `notivisa-events` (publish `processNotiVisaQueue` summary)
- Looker/Data Studio query: `SELECT timestamp, totalSent, totalFailed FROM notivisa_metrics`
- Alert: if `failureRate > 0.1` (>10% fail rate) for 3 consecutive runs

**Manual intervention:**
- Ops checks Cloud Logs for 4xx errors (bad payload format, API auth fail, etc.)
- If API key expired: rotate via Firebase Secrets Manager
- If persistent 5xx: escalate to NOTIVISA support (gov team)

### 7. Firestore Rules for NOTIVISA Collection

**Rules block (wired in Phase 4-03, tested in 04-04):**

```firestore-rules
// NOTIVISA outbox — append-only for events, metadata read-only for members
match /labs/{labId}/notivisa-outbox/metadata {
  allow read: if isMemberOfLab(labId);
  allow write: if false;  // System-only (Cloud Function writes)
}

match /labs/{labId}/notivisa-outbox/events/{eventId=**} {
  allow create: if false;  // Cloud Function only
  allow read: if isMemberOfLab(labId) && hasRole(labId, 'quality_manager');
  allow update: if false;  // Function only
  allow delete: if false;  // Immutable audit trail
}
```
