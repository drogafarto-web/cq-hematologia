## SA-15 — `functions/src/modules/bioquimica/geminiOCRParser.ts`

Cloud Function fallback server-side para parsing Gemini (fallback quando client rate-limited).

**Export:**

```typescript
// onRequest (não callable) — aceita POST com imageUrl ou imageBase64
export const parseOCRImage: functions.HttpsFunction;
```

**Contrato da função:**

- Region: `southamerica-east1`
- Memory: `256MB`, timeout: `10s`
- Validate: `req.headers.authorization` presente (não nulo)
- Validate body: `{ imageUrl?: string; imageBase64?: string; labId: string }`
- Se nem `imageUrl` nem `imageBase64`: 400 `{ error: "Missing imageUrl or imageBase64" }`
- API key de `process.env.GEMINI_API_KEY`; se ausente: 500 `{ error: "Gemini API key not configured" }`
- Chama `parseImageWithGemini(imageBase64 || imageUrl, apiKey)` do shared service
- Retorna `res.json(result)` no sucesso
- Catch: `res.status(500).json({ error: message })`

**Estrutura do import:**

```typescript
import { onRequest } from 'firebase-functions/v2/https';
// parseImageWithGemini pode ser duplicado ou importado de shared
// NÃO importar de src/features (client-only)
```

## SA-16 — `docs/BIOQUIMICA_OCR_INTEGRATION.md`

Documentação técnica end-to-end do pipeline OCR. Haiku escreve — não executa código.

**Seções obrigatórias:**

### 1. Arquitetura (diagrama ASCII)

```
Operador → OCRUploadModal → useGeminiVision → geminiVisionService
  → Gemini 2.5 Flash API → OCRParseResult
  → ocrFuzzyMatch (Levenshtein + semantic) → MatchResult[]
  → Operador revisa → useOCRValidation.submitDecision
  → ocrValidationService.recordOCRaudit → Firestore audit (imutável)
  → recordRunBioquimica callable (Phase 9.4+)
```

### 2. Prompt Gemini

Texto literal do `GEMINI_OCR_PROMPT` do SA-09.

### 3. Modelo de Custo

| Volume          | Custo/imagem | Custo/dia | Custo/mês |
| --------------- | ------------ | --------- | --------- |
| 10 imagens/dia  | ~$0.0005     | ~$0.005   | ~$0.15    |
| 50 imagens/dia  | ~$0.0005     | ~$0.025   | ~$0.75    |
| 200 imagens/dia | ~$0.0005     | ~$0.10    | ~$3.00    |

### 4. Thresholds de Confiança

| Range  | Cor      | Ação                      |
| ------ | -------- | ------------------------- |
| > 95%  | Verde    | Auto-aceito               |
| 85–95% | Amarelo  | Revisão sugerida          |
| 70–85% | Laranja  | Revisão obrigatória       |
| < 70%  | Vermelho | Rejeitar / inserir manual |

### 5. Cenários de Erro e Tratamento

- Timeout (5s): botão de retry + fallback manual
- API 429 (rate limit): usar CF server-side fallback (SA-15)
- JSON inválido: `OCRParseResult` com `errors`, operador vê aviso
- Imagem corrompida: `overallConfidence: 0`, operador re-faz upload

### 6. Rollback

Se Gemini API indisponível: desabilitar botão OCR, manter entrada manual funcional.

### 7. Compliance

- RDC 978 Art. 161: operadorId + timestamp em cada decisão OCR
- DICQ 4.4: trilha completa image→parse→match→approve em `labs/{labId}/bioquimica/audit`
