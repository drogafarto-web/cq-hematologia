## SA-05 — `src/features/bioquimica/utils/westgardRulesCLSI.ts`

Implementar engine Westgard client-side. Função pura: mesma entrada → mesma saída. Usa tipos de W0.

**Export principal:**

```typescript
export function evaluateRules(
  points: number[], // valores de controle em ordem
  config: WestgardConfig,
  sigma: number, // desvio padrão do analito
): RuleViolation[] | null;
```

**Regras a implementar (em ordem):**

- `1-2s`: qualquer ponto > 2σ → `severity: "warn"`
- `1-3s`: qualquer ponto > 3σ → `severity: "reject"`
- `2-2s`: 2 pontos consecutivos > 2σ no mesmo lado → `severity: "reject"`
- `R-4s`: range entre 2 pontos consecutivos > 4σ → `severity: "reject"`
- `4-1s` (se enabled): 4 consecutivos > 1σ mesmo lado → reject
- `10x` (se enabled): 10 consecutivos mesmo lado da média → reject
- `6T` (se enabled): 6 consecutivos crescentes ou decrescentes → reject
- `6X` (se enabled): 6 alternando positivo/negativo → reject

**Contrato:**

- Retornar `null` se `!config.enabled` ou `points.length === 0`
- Retornar `null` se nenhuma violação encontrada
- Array com todas as violações se houver alguma
- Cada `RuleViolation` inclui `affectedPoints: number[]` (índices)

**Min lines:** 200  
**Commit:** `feat(bioquimica): implement client-side Westgard engine (Phase 9 W1)`  
**Verify:** `npx tsc --noEmit src/features/bioquimica/utils/westgardRulesCLSI.ts`

## SA-08 — `src/features/bioquimica/utils/ocrFuzzyMatch.ts`

Fuzzy matching: nome parseado → analito semeado. Função pura, sem side effects.

**Exports:**

```typescript
// Função principal
export function fuzzyMatchAnalyte(
  parsedName: string,
  seededAnalitos: Analito[],
  threshold?: number, // default 0.7
): MatchResult;

// Utilitários (exportados para teste)
export function levenshteinDistance(a: string, b: string): number;
export function normalizeAnalyteName(str: string): string;
export function semanticSimilarity(a: string, b: string): number;
```

**Algoritmo:**

1. `normalizeAnalyteName`: lowercase + remove diacríticos + trim
2. Verificar match exato (normalizado) → `matchReason: "exact"`, confidence 1.0
3. Verificar `semanticSimilarity` (lista de equivalências PT/EN hardcoded) → se > bestScore
4. Calcular `levenshteinDistance` / maxLen → confidence normalizada
5. Retornar melhor match se ≥ threshold, senão `matchReason: "no-match"`

**Equivalências semânticas mínimas:**

