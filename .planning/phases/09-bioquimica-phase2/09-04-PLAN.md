## SA-12 — `src/features/bioquimica/hooks/useGeminiVision.ts`

Hook de estado para o pipeline OCR completo: idle → uploading → parsing → matching → validating.

**Export:**

```typescript
export type OCRProgress = 'idle' | 'uploading' | 'parsing' | 'matching' | 'validating';

export interface UseGeminiVisionReturn {
  parsing: boolean;
  result: OCRParseResult | null;
  validationResult: OCRValidationResult | null;
  error: string | null;
  progress: OCRProgress;
  costEstimate: number;
  parseImage: (file: File, analitos: Analito[]) => Promise<OCRValidationResult | null>;
  reset: () => void;
}

export function useGeminiVision(): UseGeminiVisionReturn;
```

**Fluxo `parseImage`:**

1. Setar `progress: "uploading"`, `parsing: true`
2. Ler file como ArrayBuffer
3. Setar `progress: "parsing"` → chamar `parseImageWithGemini(buffer, VITE_GEMINI_API_KEY, file.type)` com timeout 5000ms via `Promise.race`
4. Setar `progress: "matching"` → mapear `result.analytes` com `fuzzyMatchAnalyte(p.name, analitos)`
5. Setar `progress: "validating"` → montar `OCRValidationResult`
6. Setar `parsing: false`, `progress: "idle"`, atualizar `validationResult`

**Timeout:** `new Promise((_, reject) => setTimeout(() => reject(new Error("Parsing timeout (5s)")), 5000))`  
**Erro:** setar `error: errorMsg`, `parsing: false`, retornar `null`

**Min lines:** 100

**Verify:** `npx tsc --noEmit src/features/bioquimica/hooks/useGeminiVision.ts`

## SA-14 — `src/features/bioquimica/components/OCRUploadModal.tsx`

Modal dark-first para upload de imagem H550 → parsing → revisão → aprovação.

**Export:**

```typescript
interface OCRUploadModalProps {
  open: boolean;
  onClose: () => void;
  onApprove: (validationResult: OCRValidationResult) => void;
  analitos: Analito[]; // seed passado via props (evita re-fetch)
}

export function OCRUploadModal(props: OCRUploadModalProps): JSX.Element | null;
```

**3 estados de render:**

1. **Upload state** — input file + botão "Analisar com IA"; desabilitado se parsing
2. **Progress state** — durante parsing: barra de progresso + texto do `progress` state
3. **Results state** — lista de matches com color-coded confidence:
   - > 95%: verde (`emerald-500`)
   - 85–95%: amarelo (`yellow-500`)
   - < 85%: laranja (`orange-500`)
   - no-match: vermelho (`red-500`)
   - Botão "Aprovar" desabilitado se `!validationResult.readyForWestgard`

**Design tokens (CLAUDE.md dark-first):**

- Background: `bg-[#141417]` (modal) + `bg-black/70` (overlay)
- Texto: `text-[#f5f5f7]` + `text-white/60` (secondary)
- Border: `border border-white/10`
- Accent: `bg-violet-500` (botão primário)
- Radius: `rounded-xl` (modal), `rounded-lg` (itens)
- Transições: `transition-all duration-150`

**Acessibilidade:**

- `role="dialog"`, `aria-modal="true"`, `aria-label="Upload de imagem H550"`
- Fechar no backdrop click + Escape key (`useEffect` + `e.key === "Escape"`)
- Focus trap: primeiro `<button>` recebe foco ao abrir
