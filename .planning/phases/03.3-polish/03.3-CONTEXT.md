## Key Architectural Decisions (Locked for 3.3)

### 1. Detox for Mobile E2E
**Decision:** Detox (by Wix) over Appium or Maestro.

**Why:**
- Native synchronization with React Native ‚Äî no flaky wait-for-element timeouts.
- First-class iOS + Android support with same test codebase.
- Graybox testing: interacts via JS bridge, inspects actual component tree.
- Active maintenance, largest RN E2E community.
- Appium: too much driver setup overhead. Maestro: less mature TypeScript API.

**Constraints:**
- Requires physical device or simulator running (not pure CI-headless without extra config).
- Android emulator must have hardware acceleration enabled.
- Detox config in `hc-quality-mobile/.detoxrc.js`.

### 2. Puppeteer / html-pdf for Dashboard PDF Export
**Decision:** Puppeteer (headless Chrome) in Cloud Function for PDF snapshot generation.

**Why:**
- Dashboard is rendered HTML ‚Äî screenshot-to-PDF is pixel-perfect, no layout rework.
- Puppeteer Node.js API is reliable in Cloud Function (Functions Gen 2 supports up to 4GB RAM).
- Alternatives: `@react-pdf/renderer` (good for structured docs, not dashboards); `jsPDF` (canvas-based, lossy for charts).
- For large labs, PDF is generated async (same job queue as existing XLSX exports).

**Constraints:**
- Puppeteer binary must be bundled or pulled from Puppeteer managed CDN in Function env.
- Max PDF size target: <10MB for 12-month lab history.
- Compression step: `gs` (Ghostscript) or `pdflib` post-render.

### 3. Nodemailer for Email Delivery
**Decision:** Nodemailer with SMTP relay (SendGrid or AWS SES) for export download link delivery.

**Why:**
- Cloud Functions cannot send email directly ‚Äî must go via SMTP relay.
- SendGrid free tier: 100 emails/day (sufficient for lab export volumes).
- AWS SES: lower per-email cost at scale, harder initial setup.
- Nodemailer wraps SMTP uniformly ‚Äî swap provider by changing env vars.

**Constraints:**
- `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS` stored in Firebase Secret Manager (not `.env`).
- Email contains signed URL only (not attachment) ‚Äî avoids email attachment size limits.
- Template: plain HTML with HC Quality branding, lab name, export type, expiry date.
- Delivery confirmation: job record updated with `emailSentAt` timestamp.

### 4. Nativewind v4 for Dark Mode Theming
**Decision:** Nativewind v4 (`tailwindcss` for React Native) for systematic dark mode.

**Why:**
- Nativewind v4 supports `dark:` variant natively (mirrors web Tailwind).
- Eliminates inline `StyleSheet` dark/light branching ‚Äî one class handles both.
- Aligns with `CLAUDE.md` brand tokens (`bg-[#141417]`, `violet-500`, `emerald-500`).
- Avoids maintaining parallel StyleSheet objects per color scheme.

**Constraints:**
- Nativewind v4 requires Expo SDK 50+ or React Native 0.73+ (verify project version).
- Tailwind config must extend brand palette matching `DESIGN_SYSTEM.md`.
- All existing `StyleSheet.create` screens must be migrated to Nativewind class props.

## Risk Register

| Risk ID | Domain | Risk | Impact | Likelihood | Mitigation |
|---------|--------|------|--------|------------|-----------|
| R-3.3-01 | Mobile | App Store review delay (Apple) | Launch delay 1-3 weeks | Medium | Submit TestFlight build early (start of Week 5), not at end. Have PWA fallback ready. |
| R-3.3-02 | Mobile | Biometric auth not available on test devices | Feature untestable pre-launch | Low | Implement gracefully degrading ‚Äî falls back to PIN if no biometric enrolled. Test on physical device only. |
| R-3.3-03 | Mobile | Detox E2E flakiness on Android emulator | CI fails intermittently | Medium | Use `waitFor` with explicit sync points; retry config `testRunner.retries: 2`; Android emulator with hardware acceleration. |
| R-3.3-04 | Analytics | PDF generation timeout for large dashboards | Puppeteer times out in Cloud Function | Medium | Set `page.setDefaultTimeout(60000)`; increase CF memory to 2GB; compress after render. |
| R-3.3-05 | Analytics | 30s polling causes excessive Firestore reads | Cost spike for large labs | Medium | Poll `analytics-meta` (1 small doc) not full aggregates; guard with `lastUpdated` diff check. |
| R-3.3-06 | Export | Email deliverability (spam filters) | Emails not received by lab operators | Low | Use verified domain in SMTP relay; include unsubscribe link (LGPD compliance); test with major providers (Gmail, Outlook). |
| R-3.3-07 | Export | Batch export memory exhaustion | Cloud Function OOM for 3+ large formats | Medium | Process formats sequentially (not parallel) in batch job; limit batch to 5 formats max; fail-fast with partial results written. |
| R-3.3-08 | Export | PDF compression quality loss | Charts unreadable after Ghostscript compress | Low | Test compression with `/ebook` profile first; fall back to `/screen` only if <10MB not achievable with `/ebook`. |
| R-3.3-09 | Mobile | Nativewind v4 migration breaks existing StyleSheet screens | Visual regressions across all screens | High | Migrate screens one-by-one with screenshot comparison; keep StyleSheet fallback until verified; start with lowest-risk screen (OfflineIndicator). |
| R-3.3-10 | Cross | TestFlight + Play Store simultaneous beta | Reviewer catches compliance issue in both platforms at once | Low | Submit iOS first (faster review historically for betas); submit Android after iOS beta confirmed. |
