<context>
**From Phase 3.2-03:** Export system uses job queue pattern: `initiateExport` callable → Pub/Sub topic → `backgroundWorker` Cloud Function → Cloud Storage → signed URL. Job records live in `/labs/{labId}/export-jobs/{jobId}` with schema: `{ status, format, gsPath, signedUrl, expiresAt, createdAt, operatorId, labId }`.

**What does NOT exist yet:**

- CSV format (auditLogs export) — no formatter
- Excel conditional formatting — SheetJS creates plain cells; no color rules applied
- PDF compression — PDFs written directly without size optimization
- Email delivery — signedUrl returned to client but not emailed
- Batch export — one callable call = one format only
- Scheduled export — no Cloud Scheduler trigger

**Architecture for each new feature:**

**CSV audit log:**

- Source: `/labs/{labId}/auditLogs` collection
- Fields: `timestamp, operatorId, action, resourceType, resourceId, previousHash, newHash, labId`
- Format: RFC 4180 compliant CSV, UTF-8 BOM (Excel compatibility), header row, ISO 8601 timestamps
- Ordering: `timestamp` ascending (chronological forensics trace)
- Size concern: 100k audit records × ~200 bytes/row = ~20MB raw → compress with gzip to ~5MB

**Excel conditional formatting:**

- SheetJS Pro supports `!cflrules`; SheetJS Community does not
- Use SheetJS community workaround: loop over cells post-generation, set `s.fill` manually for out-of-range cells
- Out-of-range definition: CIQ run `status === 'invalid'` → entire row red fill (`{ fgColor: { rgb: 'FFEF4444' } }`)
- Status `'pending'` → amber fill (`{ fgColor: { rgb: 'FFF59E0B' } }`)
- Requires `write` with `{ bookType: 'xlsx', cellStyles: true }` option

**PDF compression:**

- Use `pdf-lib` (pure JS, works in Node Cloud Function without system deps)
- Strategy: re-save with compression options (`compress: true`)
- Alternative if >10MB persists: use Ghostscript via `child_process.execFile` in CF (requires gs installed in container — use custom Docker base image or check if available in Gen 2 runtime)
- Primary: `pdf-lib` approach (no Docker dependency); fallback note documented

**Email delivery:**

- `nodemailer` with SMTP transport
- SMTP config from Firebase Secret Manager: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`
- Template: HTML email with lab name, export type, download button (CTA), expiry date, HC Quality footer
- Delivery: called by `backgroundWorker` after signed URL generated; if `emailRecipient` field present in job record
- LGPD note: include "Este email foi enviado a pedido do operador {operatorId}" + unsubscribe note

**Batch export:**

- Single callable `batchExport({ labId, formats, dateRange, emailRecipient? })`
- For each format in `formats` array: call `initiateExport()` and collect jobIds
- Use `writeBatch` for atomic creation of all job records
- Return `{ jobIds: string[] }` — client polls each separately
- Max batch size: 5 formats (guard against abuse)

**Scheduled export:**

- Cloud Scheduler: cron `0 2 * * 0` (Sunday 02:00 UTC)
- Reads `/labs/{labId}/exportSchedule` config document (schedule enabled, formats, recipient email)
- For each lab with active schedule: calls `batchExport` with configured formats + recipient
- Firebase Scheduler trigger (not Pub/Sub — simpler for weekly frequency)
  </context>

```typescript
import * as XLSX from 'xlsx';

export const OUT_OF_RANGE_STYLE = {
  fill: { patternType: 'solid', fgColor: { rgb: 'FFEF4444' } }, // red
  font: { color: { rgb: 'FFFFFFFF' }, bold: true },
};

export const PENDING_STYLE = {
  fill: { patternType: 'solid', fgColor: { rgb: 'FFF59E0B' } }, // amber
  font: { color: { rgb: 'FF000000' } },
};

export const VALID_STYLE = {
  fill: { patternType: 'solid', fgColor: { rgb: 'FF10B981' } }, // emerald
  font: { color: { rgb: 'FFFFFFFF' } },
};

/**
 * Applies conditional formatting to CIQ worksheet.
 * Must be called after worksheet is built but before workbook write.
 * Requires XLSX.write with { cellStyles: true }.
 */
export function applyConditionalFormatting(
  ws: XLSX.WorkSheet,
  statusColIndex: number, // 0-based column index of 'status' column
  headerRowCount = 1,
): XLSX.WorkSheet {
  const range = XLSX.utils.decode_range(ws['!ref'] ?? 'A1');

  for (let R = headerRowCount; R <= range.e.r; R++) {
    const statusCell = ws[XLSX.utils.encode_cell({ r: R, c: statusColIndex })];
    if (!statusCell) continue;

    const style =
      statusCell.v === 'invalid'
        ? OUT_OF_RANGE_STYLE
        : statusCell.v === 'pending'
          ? PENDING_STYLE
          : VALID_STYLE;

    // Apply style to all cells in the row
    for (let C = range.s.c; C <= range.e.c; C++) {
      const cellAddr = XLSX.utils.encode_cell({ r: R, c: C });
      if (!ws[cellAddr]) ws[cellAddr] = { t: 's', v: '' };
      ws[cellAddr].s = style;
    }
  }

  return ws;
}

export function getCIQOutOfRangeStyle() {
  return OUT_OF_RANGE_STYLE;
}
```

```typescript
import { PDFDocument } from 'pdf-lib';

export interface CompressionResult {
  buffer: Buffer;
  originalSizeBytes: number;
  compressedSizeBytes: number;
  compressionRatio: number;
}

/**
 * Compresses PDF buffer using pdf-lib re-serialization.
 * Reduces file size by removing redundant cross-references and compressing streams.
 * Target: <10MB for 12-month lab history.
 */
export async function compressPDF(inputBuffer: Buffer): Promise<CompressionResult> {
  const originalSizeBytes = inputBuffer.byteLength;

  const pdfDoc = await PDFDocument.load(inputBuffer, {
    ignoreEncryption: true,
  });

  // Re-serialize with object streams (compresses cross-reference table)
  const compressedBuffer = Buffer.from(
    await pdfDoc.save({ useObjectStreams: true, addDefaultPage: false }),
  );

  const compressedSizeBytes = compressedBuffer.byteLength;
  const compressionRatio = originalSizeBytes / compressedSizeBytes;

  // Log warning if still over 10MB after compression
  if (compressedSizeBytes > 10 * 1024 * 1024) {
    console.warn(
      `[PDFCompressor] File still ${(compressedSizeBytes / 1024 / 1024).toFixed(1)}MB after compression. ` +
        `Original: ${(originalSizeBytes / 1024 / 1024).toFixed(1)}MB. Consider date range reduction.`,
    );
  }

  return { buffer: compressedBuffer, originalSizeBytes, compressedSizeBytes, compressionRatio };
}
```

Firebase Cloud Scheduler trigger (v2 API):

```typescript
import { onSchedule } from 'firebase-functions/v2/scheduler';
import { getFirestore, collection, query, where, getDocs } from 'firebase-admin/firestore';

export const scheduledWeeklyExport = onSchedule(
  { schedule: '0 2 * * 0', timeZone: 'UTC', region: 'southamerica-east1' },
  async (event) => {
    const db = getFirestore();

    // Fetch all labs with active export schedule
    const schedulesSnap = await db
      .collectionGroup('exportSchedule')
      .where('enabled', '==', true)
      .where('frequency', '==', 'weekly')
      .get();

    for (const schedDoc of schedulesSnap.docs) {
      const config = schedDoc.data();
      const labId = config.labId as string;

      try {
        // Call batchExport logic inline (not via HTTP callable — direct function call)
        await triggerBatchExportForLab({
          labId,
          formats: config.formats,
          dateRange: buildLastWeekRange(),
          emailRecipient: config.emailRecipient,
        });

        await schedDoc.ref.update({ lastRunAt: FieldValue.serverTimestamp() });
      } catch (err) {
        console.error(`[ScheduledExport] Failed for lab ${labId}:`, err);
        // Non-fatal for other labs — continue loop
      }
    }
  },
);

function buildLastWeekRange() {
  const end = new Date();
  const start = new Date(end.getTime() - 7 * 24 * 60 * 60 * 1000);
  return { start: start.toISOString(), end: end.toISOString() };
}
```

## Trust Boundaries

| Boundary                             | Description                                                                                                      |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------- |
| Batch export callable → Firestore    | `writeBatch` ensures atomicity; partial batch failure rolls back all job records                                 |
| Email delivery → SMTP relay          | SMTP credentials in Secret Manager; never in code or Firestore                                                   |
| Scheduled export → Lab data          | Cloud Scheduler runs as service account; must have `firestore.read` permission for the lab's subcollections only |
| CSV audit log → auditLogs collection | auditLogs are write-only by operators; export read requires `labAdmin` or `RT` role                              |

## STRIDE Threat Register

| Threat ID | Category               | Component                                                  | Disposition | Mitigation Plan                                                                                                                                                 |
| --------- | ---------------------- | ---------------------------------------------------------- | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-03.3-09 | Information Disclosure | CSV audit log exposes all operator actions                 | mitigate    | CSV export requires `labAdmin` or RT role; Firestore security rule + callable validates before generating                                                       |
| T-03.3-10 | Spoofing               | Scheduled export runs as service account — no per-lab auth | mitigate    | Scheduled function reads only `/exportSchedule` docs with `enabled: true`; data access is scoped to lab's own collections                                       |
| T-03.3-11 | Denial of Service      | Batch of 5 formats × large dataset                         | mitigate    | Max 5 formats hardcoded; each job runs in separate CF invocation with 540s timeout; Pub/Sub handles backpressure                                                |
| T-03.3-12 | Information Disclosure | SMTP credentials in function env                           | mitigate    | SMTP credentials stored only in Firebase Secret Manager (`defineSecret`); never logged, never in function env vars or Firestore                                 |
| T-03.3-13 | Tampering              | Email recipient spoofed in batch payload                   | mitigate    | Email recipient validated via Zod `.email()` schema; email is informational (contains only signed URL, not data itself); signed URL validates labId on download |
| T-03.3-14 | Repudiation            | Scheduled export not traceable to operator                 | mitigate    | Scheduled export logs `{ triggeredBy: 'scheduler', scheduleId, labId, timestamp }` in auditLogs; `operatorId` field = `'system/scheduler'`                      |
