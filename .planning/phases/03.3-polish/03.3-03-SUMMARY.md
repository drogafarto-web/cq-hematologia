## Decisions Made

- **SheetJS Community cell-level styles:** Community Edition does not support `!cflrules` (Pro feature). Workaround: iterate rows post-generation, set `ws[cellAddr].s` for each cell. Works in Excel 365/Google Sheets. **Does NOT render in LibreOffice Calc** — documented as known limitation.
- **pdf-lib over Ghostscript:** pdf-lib is pure JS (no system deps, no custom Docker needed for Gen 2 CFs). Achieves 10-40% size reduction for programmatically generated PDFs. Ghostscript documented as deferred item if image-heavy PDFs still exceed 10MB.
- **randomUUID() not nanoid:** `nanoid` is used in `initiateExport.ts` but isn't in `functions/package.json`. Chose `crypto.randomUUID()` (built into Node 22) to avoid adding a dependency.
- **batchExport calls Cloud Callable, not internal function:** Each batch job enqueues to Pub/Sub independently — consistent with existing architecture. WriteBatch ensures atomic job record creation before any enqueue.
- **scheduledWeeklyExport uses `collectionGroup('exportSchedule')`:** Each lab stores its schedule at `/labs/{labId}/exportSchedule/config`. CollectionGroup query finds all enabled schedules in a single read.

## Known Limitations

- **LibreOffice Excel Formatting:** SheetJS Community cell styles (`s` property) render in Microsoft Excel 365 and Google Sheets but NOT in LibreOffice Calc. LibreOffice ignores the OOXML `xf`/`fill` extension used by SheetJS Community. Files open correctly but appear without fill colors in LibreOffice.
- **PDF compression ratio:** `pdf-lib` achieves 10-40% for programmatically generated PDFs. Image-heavy PDFs (scanned documents) may still exceed 10MB — Ghostscript-based approach with custom Docker would be needed.
- **SMTP not configured:** SMTP secrets (SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS) must be configured in Firebase Secret Manager before email delivery works.

## Deferred Items

- Ghostscript-based PDF compression for image-heavy PDFs (requires custom Docker runtime image for Gen 2 CFs)
- Firestore rules for `/labs/{labId}/exportSchedule` — needs `allow create: if isActiveMemberOfLab(labId)` + `allow read: if isActiveMemberOfLab(labId)` rules
- `nanoid` added to `functions/package.json` (or replace in `initiateExport.ts` with `crypto.randomUUID()` for consistency)
- ExportStep1 component: update format options to include 'xlsx-ciq', 'xlsx-nc', 'pdf-compliance', 'csv-audit' (current Phase 3.3 formats) — currently shows 'xlsx' / 'pdf' / 'csv'
