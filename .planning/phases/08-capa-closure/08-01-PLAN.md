## Task 1: Define TypeScript Types

**Goal:** Type-safe CAPA tracking domain.

Create `src/features/capa-tracking/types/index.ts`:

```typescript
export type CAPAStatus =
  | 'aberto'
  | 'em-andamento'
  | 'evidencia-submetida'
  | 'auditor-revisando'
  | 'fechado';

export type CAPAPriority = 'critica' | 'alta' | 'media' | 'estendida';

export interface CAPATransition {
  from: CAPAStatus;
  to: CAPAStatus;
  operatorId: string;       // request.auth.uid
  signature: LogicalSignature;
  ts: Timestamp;
  notes?: string;
}

export interface CAPAEvidenceRef {
  type: 'foto' | 'documento' | 'certificado' | 'pop' | 'treinamento';
  storagePath: string;       // /labs/{labId}/auditoria-evidencia/capa-{ncId}/...
  uploadedAt: Timestamp;
  uploadedBy: string;
  hash: string;              // chain-hash for integrity
}

export interface CAPA {
  id: string;                // matches NC id (1:1)
  labId: string;
  ncId: string;
  finding: string;            // descriptive finding from audit
  dicqRef: string;            // e.g., "DICQ 4.14.5"
  priority: CAPAPriority;
  deadline: Timestamp;
  owner: string;              // userId
  ownerName: string;          // denormalized for display
  status: CAPAStatus;
  transitions: CAPATransition[];
  evidence: CAPAEvidenceRef[];
  effectivenessCriteria: string;
  closedAt?: Timestamp;
  closedBy?: string;
  closureSignature?: LogicalSignature;
  createdAt: Timestamp;
  deletedAt?: Timestamp | null;
}
```

**Acceptance:**
- All types exported from index
- No `any` usage
- LogicalSignature reused from `src/core/types/signature.ts`

## Task 5: Cloud Function for Status Transitions

**Goal:** Server-validated status transitions with LogicalSignature.

Create `functions/src/capa/updateCAPAStatus.ts`:

**Callable signature:**
```typescript
export const updateCAPAStatus = onCall<
  { labId: string; capaId: string; toStatus: CAPAStatus; notes?: string },
  { success: boolean; signature: LogicalSignature }
>(async (request) => {
  // 1. Auth check (uid required)
  // 2. Permission check (member of labId)
  // 3. Read current CAPA, validate transition is allowed
  // 4. Compute LogicalSignature: hash(capaId + toStatus + uid + ts)
  // 5. Atomic update: status + transitions[] + signature
  // 6. Return signature for UI
});
```

**Valid Transitions (state machine):**
- `aberto` → `em-andamento`
- `em-andamento` → `evidencia-submetida`
- `evidencia-submetida` → `auditor-revisando` | `em-andamento` (rejected)
- `auditor-revisando` → `fechado` | `evidencia-submetida` (re-work)
- `fechado` → terminal (no transitions)

**Acceptance:**
- Invalid transitions throw `failed-precondition`
- LogicalSignature hash is 64 chars (SHA-256)
- All transitions logged in `transitions[]` array
- Test: 5+ test cases (happy path, invalid transition, no auth, no permission, terminal state)
