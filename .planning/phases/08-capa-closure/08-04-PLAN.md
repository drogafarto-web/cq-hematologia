## SA-20 — `functions/src/callables/capa/createCapa.ts`

Callable v2 para criar CAPA a partir de achado de auditoria.

**Input:** `{ labId: string; finding: CapaFinding; rootCause: string; correctiveAction: string; deadlineDate: number }`  
**Output:** `{ capaId: string }`

**Invariantes:**

- Validar: `rootCause.length >= 50`, `correctiveAction.length >= 50`, `deadlineDate > Date.now()`
- Verificar auth + `isActiveMemberOfLab(labId)` (RT/admin/auditor)
- Criar doc em `labs/{labId}/capa-tracking/{uuid}` com `state: 'open'`, `evidence: []`, `rfiLog: []`
- `stateHistory: [{ from: null, to: 'open', transitionedAt: now, transitionedBy: uid }]`
- Logar em Cloud Logs: `console.log(JSON.stringify({ event: 'capa_created', capaId, labId }))`
- Região: `southamerica-east1`

**Verify:** `cd functions && npm run build 2>&1 | grep -c error | grep -E '^0$'`

## SA-26 — `functions/src/callables/management-review/aggregateManagementReviewData.ts`

Callable v2 para agregar as 15 entradas da revisão anual.

**Input:** `{ labId: string; startDate: number; endDate: number }`  
**Output:** `{ entries: ManagementReviewEntry[] }` (sempre 15 entradas)

**Entradas (1–15) + fonte de dados:**

1. NC trends → `labs/{labId}/nc` (count by severity)
2. CAPA status → `labs/{labId}/capa-tracking` (count by state)
3. Training hours → `labs/{labId}/educacao-execucoes` (sum horas)
4. CEQ results → `labs/{labId}/ceq-resultados` (count + % acceptable)
5. Audit findings → `labs/{labId}/auditoria-achados` (count)
6. KPI trends → `labs/{labId}/kpi-snapshots` (último snapshot)
7. Complaints → `labs/{labId}/reclamacoes` (count + resolution %)
8. Suppliers → `labs/{labId}/fornecedores` (count active)
9. Improvements → `labs/{labId}/sugestoes` (count + implemented %)
10. Personnel changes → `labs/{labId}/personnel/designacoes` (mudanças no período)
11. Equipment calibration → `labs/{labId}/calibracao` (count overdue)
12. Incidents → `labs/{labId}/incidents` (count by severity)
13. Risk management → `labs/{labId}/risks` (count high NPR)
14. PGRSS → retornar `{ source: 'manual', data: {} }` (preenchimento manual)
15. Compliance gaps → retornar `{ source: 'manual', data: {} }` (preenchimento manual)

**Invariantes:**

- Retornar sempre 15 entradas — entradas sem dados retornam `{ data: {}, source: 'auto-aggregated' }`
- Erros em queries individuais NÃO abortam — logar warning e continuar com `data: { error: string }`
- Timeout max 30s (functions v2 default)

**Verify:** `cd functions && npm run build 2>&1 | grep -c error | grep -E '^0$'`
