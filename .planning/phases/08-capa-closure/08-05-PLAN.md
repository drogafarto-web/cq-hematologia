## SA-29 — `src/features/capa-tracking/components/CapaEvidenceUpload.tsx`

Modal de upload de evidência com hash SHA-256. ~120 LOC.

**Props:** `{ labId: string; capaId: string; onSuccess: () => void; onClose: () => void }`

**Fluxo:**
1. Drag-drop ou file picker (max 10MB)
2. Selecionar arquivo → computar hash via `crypto.subtle.digest('SHA-256', buffer)`
3. Exibir hash (primeiros 32 chars) + tamanho
4. Textarea: descrição da evidência
5. Upload: `uploadBytes` no Storage → `uploadCapaEvidenceCallable`
6. Estados: idle | hashing | uploading | success | error

**Invariantes:**
- Hash computado client-side antes do upload
- `signature.operatorId` vem de `useAuthStore().user.uid`
- Storage path: `capa-evidence/{labId}/{capaId}/{Date.now()}_{fileName}`

**Verify:** `npx tsc --noEmit 2>&1 | grep -c error | grep -E '^0$'`
