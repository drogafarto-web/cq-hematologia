# Phase 03.1 Plan 02: Analytics Module Foundation Summary

CIQ compliance metrics aggregation (hourly Cloud Function) with client-side React hook for cached dashboard reads.

## What Was Built

### 1. Analytics Type System
- **File:** `src/features/analytics/types/index.ts`
- **Exports:**
  - `CIQComplianceMetrics` — 11 fields (counts, percentages, timing)
  - `AnalyticsMetadata` — freshness tracking (lastRefreshAt, isCached)
  - `AnalyticsQueryFilter` — ready for Phase 3.2 filters
  - `AggregationResult` — Cloud Function return type
  - `AnalyticsState` — React hook state shape
- **Usage:** Imported by hooks, services, and Cloud Functions

### 2. Firestore Aggregation Query (Server-Side)
- **File:** `functions/src/modules/analytics/queryCIQCompliance.ts`
- **Function:** `queryCIQCompliance(labId: string): Promise<CIQComplianceMetrics>`
- **Metrics computed:**
  - `totalRuns` — count of all runs (not soft-deleted)
  - `validRuns` — count where status === 'valid'
  - `invalidRuns` — derived (totalRuns - validRuns)
  - `openNCs` — count where status === 'aberto'
  - `closedNCs` — count where status !== 'aberto'
  - `compliancePercent` — (validRuns / totalRuns) * 100 [0-100]
  - `ncResolutionRate` — (closedNCs / (openNCs + closedNCs)) * 100 [0-100]
  - `avgResolutionDays` — mean time from NC creation to closure [1 decimal]
  - `avgProcessingHours` — mean time from run creation to signing [1 decimal]
  - `computedAt`, `dataAsOf` — timestamps
- **Implementation:**
  - Uses Firestore aggregation queries (`.count().get()`) for scalability
  - Graceful fallback to manual snapshot count on timeout (>60s)
  - Respects RN-06 soft-delete (filters `deletadoEm === null`)
  - Samples first 1000 runs for avg processing (avoids full scan)
  - Execution time <10s target (verified in local testing)

### 3. Scheduled Cloud Function (Hourly Aggregation)
- **File:** `functions/src/modules/analytics/aggregateDaily.ts`
- **Export:** `aggregateAnalytics` (onSchedule)
- **Schedule:** `every 1 hours` via Cloud Scheduler
- **Region:** southamerica-east1 (matches web/functions region)
- **Timeout:** 300s (5 min) for all labs combined
- **maxInstances:** 1 (prevents parallel aggregation)
- **Workflow:**
  1. Fetches all labs from `labs` collection
  2. For each lab, calls `queryCIQCompliance(labId)`
  3. Caches metrics in `/labs/{labId}/analytics/cache/metrics/ciqCompliance`
  4. Updates metadata in `/labs/{labId}/analytics/meta` with `lastRefreshAt`
  5. Logs summary (X/Y labs succeeded, duration)
- **Output cache structure:**
  ```
  /labs/{labId}/analytics/
    ├─ cache/
    │  └─ metrics/
    │     └─ ciqCompliance (CIQComplianceMetrics document)
    └─ meta (AnalyticsMetadata document)
  ```

### 4. React Hook for Client-Side Reads
- **File:** `src/features/analytics/hooks/useAnalytics.ts`
- **Export:** `useCIQAnalytics(): AnalyticsState`
- **Behavior:**
  - Subscribes to `/labs/{labId}/analytics/cache/metrics/ciqCompliance` via `onSnapshot`
  - Also subscribes to `/labs/{labId}/analytics/meta` for freshness metadata
  - Converts Firestore Timestamps to JS Dates automatically
  - Returns `{ ciqCompliance, metadata, loading, error }`
  - Guards on `activeLabId` (returns empty state if no lab selected)
  - Handles Firestore errors gracefully (returns error state, doesn't throw)
  - Works offline (Firebase SDK caches reads)
  - Cleanup unsubscribe on unmount
- **Helper functions exported:**
  - `formatAnalyticsMetrics(metrics)` — returns labeled strings ("87% conformidade")
  - `isAnalyticsStale(metadata)` — detects >30min staleness
- **Usage example:**
  ```tsx
  const { ciqCompliance, metadata, loading, error } = useCIQAnalytics();
  if (loading) return <Spinner />;
  if (error) return <ErrorMessage />;
  return <MetricsCard metrics={ciqCompliance} isStale={isAnalyticsStale(metadata)} />;
  ```

## Known Limitations & Blockers for Wave 2

1. **Cloud Function deployment pending**
   - Functions code is complete and type-safe
   - Not yet deployed to production
   - Awaits CTO decision to deploy Phase 3.1 functions
   - Firestore indices for runs/ncs queries must exist (from Phase 2 CT-05)

2. **No manual refresh callable yet**
   - AnalyticsService.refreshNow() is a Phase 3.2 feature
   - Currently only hourly scheduled aggregation available
   - UI can't force immediate refresh (designed for compliance compliance — predictable schedule)

3. **No Firestore security rules**
   - Analytics cache paths (/labs/{labId}/analytics/*) not yet protected by firestore.rules
   - Phase 3.2 will add read-only rules: `allow read: if isActiveMemberOfLab(labId)`

4. **Mobile hook not implemented**
   - useAnalytics.ts is web-only (uses firebase/firestore)
   - Phase 3.1 Plan 03 will port to React Native (firebase/firestore RN adapter)
   - Same hook signature, different Firebase SDK version

5. **No export integration**
   - AnalyticsService.exportAnalytics() awaits Phase 03.2 and Phase 03.1 Plan 03 (export infrastructure)
   - Cross-module integration with stream module required

6. **Dashboard UI not implemented**
   - useAnalytics hook complete
   - UI components awaiting Phase 3.1 Plan 03 (mobile dashboard) and Phase 3.2 (web dashboard)
