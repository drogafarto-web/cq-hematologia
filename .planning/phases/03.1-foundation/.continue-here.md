## Decisions Made

| Decision                                                            | Rationale                                                                                                                                                                                                    | Phase |
| ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----- |
| **Mobile: React Native CLI (not Expo)**                             | World-class requires full control over iOS/Android build pipeline. Expo limitations (module restrictions, app size, push notifications) unacceptable. Investment in GitHub Actions CI/CD pays off for years. | 3.1   |
| **Analytics: Scheduled hourly aggregation (not real-time)**         | Predictable costs (large labs 100k+ docs spike under listeners). RDC 978: aggregated data reproducible + auditable. Hourly lag acceptable for compliance dashboards.                                         | 3.1   |
| **Export: Server-side Cloud Function + Cloud Storage (not client)** | Scales to any dataset (no browser memory limits). Async UX honest. Each export auditable. CF 540s timeout sufficient for 100k+ docs.                                                                         | 3.1   |
| **Offline: Read-only cache in Phase 3.1 (Phase 3.2 writes)**        | Phase 3.1 foundation: avoid merge conflict complexity. CIQ workflows on-site where Wi-Fi available. Read-only covers 80% of use case. Phase 3.2 adds smart conflict resolution if needed.                    | 3.1   |
| **Test framework: Jest for mobile + functions, Vitest for web**     | Project standard. Jest preset for Expo. Mobile tests with `@testing-library/react-native`.                                                                                                                   | 3.1   |
| **Cache: Zustand (not Redux)**                                      | Lightweight, aligned with Phase 2 patterns, sufficient for Phase 3.1 (Redis deferred to 3.2).                                                                                                                | 3.1   |
