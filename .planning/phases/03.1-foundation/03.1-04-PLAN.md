## Lessons Learned

1. **Type-first development:** Defining types (ExportJob, CIQComplianceMetrics) before implementation prevented mismatches between mobile and Cloud Functions.

2. **Fallback strategies matter:** Firestore aggregation query fallback to manual count (if timeout) made the system resilient to large datasets.

3. **Emulator setup essential:** Firebase emulator (firestore, functions, pubsub) enabled local testing without deploying to GCP.

4. **Multi-tenant discipline:** Consistent filtering by labId across all queries prevented accidental cross-lab data access.

5. **Scheduled functions > real-time:** Hourly aggregation + cache pattern scales better than real-time listeners for large datasets.
