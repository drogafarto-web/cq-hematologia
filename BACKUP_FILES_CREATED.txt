BASH SCRIPTS (Primary - Linux/macOS):

1. C:/hc quality/scripts/backup-full.sh
   - Purpose: On-demand full Firestore backup to GCS
   - Usage: bash scripts/backup-full.sh [project-id]
   - Example: bash scripts/backup-full.sh hmatologia2
   - Verifies GCS bucket exists
   - Creates output directory
   - Triggers async Firestore export
   - Monitors progress with 5-minute timeout
   - Executable, ready to use

2. C:/hc quality/scripts/restore-pitr.sh
   - Purpose: Point-in-time recovery from GCP managed backups
   - Usage: bash scripts/restore-pitr.sh <timestamp> [target-env]
   - Example: bash scripts/restore-pitr.sh "2026-05-07T10:30:00Z" staging
   - Validates timestamp format (RFC 3339)
   - Verifies target project accessible
   - Lists available backups
   - Finds closest matching backup
   - User confirmation before restore
   - Monitors async restore operation
   - Shows progress updates

3. C:/hc quality/scripts/backup-test.sh
   - Purpose: Monthly restore validation test (automated)
   - Usage: bash scripts/backup-test.sh
   - Selects backup from 2-3 weeks ago
   - Restores to staging project
   - Waits for completion (up to 2 hours)
   - Verifies document counts
   - Validates chain-hash integrity (50 samples)
   - Runs smoke tests
   - Generates report: docs/DR_RESTORE_TEST_YYYY-MM-DD.md
   - Reports pass/fail status

4. C:/hc quality/scripts/backup-status.sh
   - Purpose: Current backup status & health check
   - Usage: bash scripts/backup-status.sh [project-id]
   - Example: bash scripts/backup-status.sh hmatologia2
   - Shows GCS backup bucket contents
   - Lists recent daily/weekly/monthly backups
   - Displays GCS bucket sizes
   - Shows GCP managed backups
   - Reports Firestore database statistics
   - Lists Cloud Scheduler jobs
   - Shows recent operations
   - Provides health recommendations
   - Multi-section report format

5. C:/hc quality/scripts/verify-chain-hash.sh
   - Purpose: Chain-hash integrity verification (post-restore)
   - Usage: bash scripts/verify-chain-hash.sh [project] [sample-size]
   - Example: bash scripts/verify-chain-hash.sh hmatologia2-staging 100
   - Samples documents from "laudos" collection
   - Verifies hash format (64-char hex string)
   - Reports valid/missing/invalid counts
   - Calculates success percentage
   - Checks other CIQ collections
   - Pass/fail threshold: 95% valid hashes
   - Detailed verification report
