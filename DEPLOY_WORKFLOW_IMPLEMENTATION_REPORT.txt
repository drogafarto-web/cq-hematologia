DEPLOYMENT ORDERING
================================================================================

Sequential (must happen in order):
1. Pre-Deploy Gate (blocks everything until passes)
2. Firestore Rules
3. Firestore Indexes (can run parallel with Rules)
4. Cloud Functions (depends on Rules)
5. Firebase Hosting (depends on Functions) — MANUAL ONLY
6. Post-Deploy Verification (smoke tests + Cloud Logs)

Rationale:
• Rules before Functions: Functions read via rule-gated paths
• Functions before Hosting: Hosting calls Cloud Function callables
• Hosting never auto: UI changes need human review + timing control

Parallel Jobs:
• Rules and Indexes can run simultaneously (both depend on pre-gate)
• Merge protection (PR checks) runs independently

SECRETS MANAGEMENT (ADR-0018)
================================================================================

Issue: HMAC signing key was PENDING_SET placeholder for 15 days
Root Cause: Secret never provisioned in Firebase
Impact: All signatures computed with placeholder string → wrong hash
Solution: Block deployment if any defineSecret() unprovisioned

Secrets Gate Script:
  File: scripts/preflight-secrets-check.sh (145 lines)
  Scope: Discovers all defineSecret('NAME') in functions/src/
  Action: firebase functions:secrets:access NAME (verify real value)
  Blocks If: Value is PENDING_SET_* or empty
  Fix: firebase functions:secrets:set NAME --project hmatologia2

Secret Names Currently Protected:

  • [Others as discovered in functions/src/]

ROLLBACK FRAMEWORK
================================================================================

Automatic Detection:
  • Pre-gate failure → BLOCK (fix locally + commit)
  • Deploy failure → LOG + report (manual intervention)
  • Post-deploy errors → FLAG in Cloud Logs artifact
  • Error spike → WARN in summary (decision tree required)

Manual Rollback Procedures:

Option A: Git-Based (Recommended)
  git revert HEAD~1
  git push
  # Auto-triggers deploy of reverted code

Option B: Rules-Only Rollback
  git checkout HEAD~1 firestore.rules
  firebase deploy --only firestore:rules --project hmatologia2 --token $FIREBASE_TOKEN

Option C: Functions-Only Rollback
  git checkout HEAD~1 functions/
  cd functions && npm run build && cd ..
  firebase deploy --only functions --project hmatologia2 --token $FIREBASE_TOKEN

Option D: Hosting-Only Rollback
  git checkout HEAD~1 -- src/
  npm run build
  firebase deploy --only hosting --project hmatologia2 --token $FACEBOOK_TOKEN

Decision Tree:
  ├─ Deploy failed in pre-gate? → Fix code + retry
  ├─ Deploy failed in Rules? → Check API logs → fix + retry
  ├─ Deploy failed in Functions? → Check Cloud Functions logs → fix + retry
  ├─ Deploy succeeded but Cloud Logs shows new errors? → ROLLBACK
  └─ Deploy succeeded, Cloud Logs clean? → Live ✅
