WHAT GETS CREATED (Per Lab)
===========================

Each migration creates 5 new collections under labs/{labId}/:

1. portal-configuracao/config
   - Branding (logo, colors)
   - Terms of service (termos)
   - Privacy policy (privacidade)
   - RDC 978 + LGPD compliant defaults

2. notivisa-outbox/_init (marker)
   - NOTIVISA event queue for Phase 4
   - Initialized empty (Phase 4 will populate)

3. criticos-escalacoes/_init (marker)
   - Critical result escalation rules
   - Initialized empty (to be configured)

4. imuno-ias-dev/_init (marker)
   - Immunology AI experiment collection
   - Initialized empty (for Phase 5+ AI features)

5. laudos-draft/_init (marker)
   - Report draft management (pessimistic locking)
   - Initialized empty (for phase 5+ drafts)

All documents include:
- criadoEm: ISO 8601 timestamp
- deletadoEm: null (soft-delete ready)
- versao: "1.4"

QUICK START
===========

Bash (macOS/Linux/WSL):
  1. export LABS_LIST=$(node scripts/list-labs.js --project hmatologia2)
  2. bash scripts/migrate-v1.4.sh --dry-run
  3. bash scripts/migrate-v1.4.sh --execute
  4. bash scripts/validate-migration-v1.4.sh

PowerShell (Windows):
  1. $env:LABS_LIST = $(node scripts/list-labs.js --project hmatologia2)
  2. .\scripts\migrate-v1.4.ps1
  3. .\scripts\migrate-v1.4.ps1 -Execute
  4. bash scripts/validate-migration-v1.4.sh
