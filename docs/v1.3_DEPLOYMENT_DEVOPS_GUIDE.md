# v1.3 Deployment — DevOps Execution Checklist

**Your job:** Execute deployment in strict order. 3 sequential steps (Rules → Functions → Hosting). No skips, no parallelization.

---

## Pre-Deployment Checklist (30 min)

Execute these in order. **Do not proceed until all PASS.**

### 1. Verify Git + Code State

- [ ] Run: `git status`
- [ ] **Expected:** Clean (no untracked files, no uncommitted changes)
- [ ] **If fail:** Commit or stash, then retry

### 2. Verify Firebase Project

- [ ] Run: `firebase projects:list`
- [ ] **Expected:** `hmatologia2` listed with status "ACTIVE"
- [ ] **If fail:** Run `firebase login` and re-authenticate

### 3. Verify Credentials

- [ ] Run: `firebase auth:login`
- [ ] **Expected:** Login succeeds, you are `drogafarto@gmail.com`
- [ ] **If fail:** Clear cache (`firebase logout`) and re-login

### 4. Backup Firestore Rules

- [ ] Run: `firebase rules:get firestore --project=hmatologia2 > ./backup-rules-$(date +%Y%m%d-%H%M%S).txt`
- [ ] **Expected:** File created with current rules
- [ ] **If fail:** Contact security team; don't proceed without backup

### 5. Verify Deploy Directory Structure

- [ ] Confirm directories exist:
  - [ ] `C:\hc quality\functions` (has package.json, tsconfig.json)
  - [ ] `C:\hc quality\src` (has package.json, vite.config.ts)
  - [ ] `C:\hc quality\firebase.json` (deploy config)
- [ ] **If any missing:** Stop, investigate setup

## Step 1: Deploy Firestore Rules + Indexes (15 min)

**Why first:** Rules protect all data at read/write time. Must be live before functions write.

### Deploy

```bash
firebase deploy --only firestore:rules,firestore:indexes --project=hmatologia2
```

### Verify in Firebase Console

1. Open **[Firebase Console](https://console.firebase.google.com)** → Project `hmatologia2`
2. Navigate to **Firestore** → **Rules** tab
3. Check: **"Last published"** timestamp should be ≤ 5 min ago
4. Click **"Test rules"** button (optional validation)

### Verify Indexes

1. Navigate to **Firestore** → **Indexes** tab
2. Check: All ~25 composite indexes show status **"Ready"** or **"Building"** (not "Error")
3. Note: Some indexes may take 5–60 min to build; that's OK if freshly deployed

### Criteria: PASS

- ✅ Rules deployed (timestamp recent)
- ✅ No errors in deploy output
- ✅ Indexes: 0 failures

**If FAIL:**

```
Run: firebase deploy --only firestore:rules --project=hmatologia2 --force
If still failing: Contact security team + save error log
```

### Status: ✅ Step 1 LIVE

## Step 2: Deploy Cloud Functions (45 min)

**Why second:** Functions read from Firestore (which is now protected by rules) and write auth-validated data.

### Build Functions

```bash
cd C:\hc quality\functions
npm run build
```

**Expected:** Zero TypeScript errors. Output:

```
Successfully compiled 32 functions...
```

**If FAIL:**

```
Check: npm run typecheck (detailed errors)
If errors found: git diff, understand changes, fix, rebuild
```

### Deploy Functions

```bash
firebase deploy --only functions --project=hmatologia2 --force
```

**What to expect:**

```
Deploy started...
Function [functionName] deployed successfully
...
Deploy complete! Resources created / updated...
```

### Verify Functions Deployed

1. Open **[Firebase Console](https://console.firebase.google.com)** → **Cloud Functions**
2. Check: All 32 functions present, statuses all **ACTIVE** (green checkmark)
3. Look for these key functions (spot-check 5):
   - ✅ `parseBulaBioquimica` (bioquímica module)
   - ✅ `liberarLaudo` (liberacao module)
   - ✅ `criarReclamacao` (reclamacoes module)
   - ✅ `listarDocsDrive` (SGD importer)
   - ✅ `registrarCalibracao` (controle-temperatura)

### Verify Environment Variables

1. Select any function (e.g., `parseBulaBioquimica`)
2. Click **Runtime settings** tab
3. Verify these **Environment variables** all populated:
   - [ ] `GEMINI_API_KEY` (non-empty)
   - [ ] `RESEND_API_KEY` (non-empty)
   - [ ] `DRIVE_OAUTH_CLIENT_ID` (non-empty)
   - [ ] `DRIVE_OAUTH_CLIENT_SECRET` (non-empty)
   - [ ] `STRIPE_API_KEY` (non-empty)

**If any missing:**

```
STOP. Functions will fail at runtime.
Contact DevOps lead to re-set environment variables in Cloud Functions console.
Then retry Step 2 deploy.
```

### Criteria: PASS

- ✅ All 32 functions deployed
- ✅ All statuses ACTIVE
- ✅ All 5+ env vars populated
- ✅ No errors in deploy output

**If FAIL:**

```
Check Cloud Logs:
firebase functions:log --limit 50 --project=hmatologia2

If timeout errors: Increase function timeout (Runtime settings)
If permission denied: Verify IAM roles (check Firebase console)
If missing dependency: npm install + npm run build + re-deploy
```

### Status: ✅ Step 2 LIVE

## Post-Deployment (30 min)

### 1. Verify Data Integrity

- [ ] Firestore Console → Collections → `/labs/{riopombaLabId}/bioquimica-analitos`
- [ ] **Expected:** 16 seed documents visible (glucose, creatinine, etc.)
- [ ] **If missing:** Investigate seed script; may need manual re-seed

### 2. Spot-Check 3 Data Paths

Execute in Cloud Logs or browser console:

**Check 1: Read CIQ data**

```javascript
// In browser console
db.collection('labs').doc('riopomba').collection('bioquimica-analitos').limit(1).get();
// Expected: 1 doc returned, no permission denied
```

**Check 2: Verify audit trail**

```javascript
// Check that audit subcollection exists
db.collection('labs')
  .doc('riopomba')
  .collection('bioquimica-runs')
  .limit(1)
  .collection('audit')
  .get();
// Expected: docs with `action`, `ts`, `operatorId`
```

**Check 3: Verify soft-delete flag**

```javascript
// Check deletadoEm field (soft-delete pattern)

// Expected: docs with `deletadoEm` field (null or timestamp)
```

### 3. Verify Indexes Built

- [ ] Firebase Console → Firestore → Indexes
- [ ] **Expected:** All indexes show **"Ready"** (may take 5–60 min)
- [ ] **If any "Error":** Contact support; critical for query performance

### 4. Final Status Check

```bash
firebase functions:log --limit 100 --project=hmatologia2 | grep -i error

```

## Rollback Procedure (if needed)

**Trigger:** QA reports "NO-GO" or production has critical errors not fixable in 15 min.

### 1. Decide Rollback

- Gather error logs (screenshot Cloud Logs)
- File issue with root cause
- Get CTO approval: "Roll back to [previous version]"

### 2. Rollback Hosting (fastest path)

```bash
# Option A: Deploy previous commit
git log --oneline -5
git checkout [previous-commit-hash]
npm run build
firebase deploy --only hosting --project=hmatologia2

# Option B: Use Firebase hosting rollback UI
# Firebase Console → Hosting → Rollback button (if available)
```

### 3. Rollback Functions (if needed)

```bash
git checkout [previous-commit-hash]
cd functions && npm run build
firebase deploy --only functions --project=hmatologia2 --force
```

### 4. Rollback Rules (if needed)

```bash
# Restore from backup you created in pre-deployment
firebase rules:set ./backup-rules-[timestamp].txt --project=hmatologia2
```

### 5. Verify Rollback

- [ ] https://hmatologia2.web.app loads
- [ ] Functions present in Firebase console
- [ ] Cloud Logs show no new errors
- [ ] Notify CTO: "Rolled back to [version], resuming investigation"
