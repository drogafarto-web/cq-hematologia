## CLOUD LOGS QUICK VIEW (In Firebase Console)

**Path:** Cloud Logging → Logs Explorer

**Filter for errors:**

```
resource.type="cloud_function"
severity="ERROR"
timestamps: last 24 hours
```

**Filter by function name:**

```
resource.labels.function_name="rtPortalLogin"
severity >= "WARNING"
```

**Common errors to watch for:**

- `Exceeded quota` → Contact Google Cloud
- `UNAUTHENTICATED` → Auth token issue
- `DEADLINE_EXCEEDED` → Timeout / slow Firestore
- `UNAVAILABLE` → Gemini API down

## ROLLBACK DECISION (CTO Only)

**Ask yourself:**

1. Root cause known & fixable in <15 min?
   - YES → Deploy fix now
   - NO → Proceed to #2

2. Previous version tested in prod 24h+ ago?
   - YES → Rollback approved
   - NO → Investigate further (may roll back earlier version)

**If rolling back:**


## NOTIVISA INTEGRATION MONITORING (First 48h)

**Every 2 hours, check in Firebase Console:**

1. Open `Firestore` → Collection `notivisa-outbox/{labId}/events`
2. Count documents with `status="pending"` → Should be 0–2
3. Count documents with `status="sent"` → Should grow steadily
4. Check `lastError` field → Should be null

**If **pending** count > 10 for >30 min:**

- [ ] Declare P1
- [ ] Investigate `notivisaQueueProcessor` logs
- [ ] Check NOTIVISA API status (may be down)

---

## GEMINI VISION API TROUBLESHOOTING

**If image upload fails:**

1. Check file size (max 20 MB)
2. Check file format (JPEG, PNG, WebP, GIF, PDF)
3. Check API quota: Firebase Console → Gemini API → Quota
4. Retry with exponential backoff (1s → 2s → 4s)
5. Fall back to `gemini-1.5-flash` (non-Vision model)

**If annotation returns nonsense:**

- Expected: This is v1.0 baseline (not production-grade yet)
- Don't declare incident; just log feedback
- File as enhancement for Phase 5

---

## AUDIT TRAIL VERIFICATION (First 48h)

**Every 4 hours, spot-check one laudo:**

1. Open any laudo in prod
2. Check `auditTrail/{labId}/events` collection
3. Find events for this laudo ID
4. Verify: `signature`, `operatorId`, `timestamp` present on each event
5. Verify: Hash is 64-character hex string

**If audit trail is EMPTY or MISSING:**

- [ ] Declare P0 (RDC 978 violation)
- [ ] Stop all laudo operations (read-only mode)
- [ ] Investigate Firestore rules + callable logic

| ----------------------------- | --------------------------------------------------------- | -------- |
| Laudo draft save slow (>5s)   | Firestore index building. Wait 30 min, retry.             | P1       |
| SMS delivery delayed (>5 min) | Twilio rate limiting. Resend after 5 min.                 | P1       |
| Gemini Vision API timeout     | Retry with exponential backoff. Fall back to Flash model. | P1       |
| Portal flashing on load       | PWA cache. Ctrl+Shift+R hard reload.                      | P2       |
