## Troubleshooting (Common Blocks)

### "Node version mismatch"
```bash
node --version  # Must be v22.x
nvm install 22 && nvm use 22  # Switch if needed
```

### "gcloud: command not found"
```bash
# Install Google Cloud SDK
# macOS: brew install google-cloud-sdk
# Windows: Download from https://cloud.google.com/sdk/docs/install
gcloud auth login
gcloud config set project hmatologia2
```

### "Lighthouse CLI not found"
```bash
npm install -g "@lhci/cli@0.14.x"
lhci --version  # Verify
```

### "Build fails with TypeScript errors"
```bash
npm run typecheck  # Find errors first
npm run build      # Then rebuild
```

### "Firestore logs return no results"
Check 1: Is Firestore active? (Check console for operations)  
Check 2: Is time range correct? (Try `--freshness 72h` instead of 24h)  
Check 3: Do you have Cloud Logs read permission? (Check gcloud auth)

## Lighthouse CI Detailed Flow

If script says "requires manual Lighthouse":

**Terminal 1:**
```bash
npm run preview
# Waits for server boot at http://localhost:4173
```

**Terminal 2 (after ~3s):**
```bash
npx @lhci/cli@latest autorun \
  --collect.staticDistDir=./dist \
  --collect.url=http://localhost:4173/hub \
  --collect.url=http://localhost:4173/features/bioquimica \
  --collect.url=http://localhost:4173/features/analytics \
  --collect.url=http://localhost:4173/features/export \
  --collect.url=http://localhost:4173/portal \
  --collect.numberOfRuns=3
```

Results saved to `.lighthouseci/` automatically.

**Or: GitHub Actions (Recommended)**
```bash
gh workflow run lighthouse-ci.yml --ref main
# Wait 15 minutes
# Download report from Actions artifacts
```

## Phase 12 Optimization Work (After Baseline)

Once baseline is committed, optimization can begin:

1. **Bundle optimization** (target: -50 KB gzip)
   - Tree-shake Firebase modules
   - Lazy-load `recharts`
   - Code-split analytics

2. **Firestore optimization** (target: p99 <200ms)
   - Add missing composite indexes
   - Batch read queries
   - Cache results client-side

3. **PDF generation** (target: <2.1s)
   - Parallel page rendering
   - Increase Cloud Function memory

4. **Caching** (target: LCP <1.5s)
   - Service Worker optimization
   - `stale-while-revalidate` headers
   - Browser cache TTL
