## Post-Approval Actions (if GO)

### Immediately

1. **Notify team** (Slack): "@channel v1.3 APPROVED for production deploy"
2. **Timestamp decision**: [DATE/TIME] CTO approved

### During Deploy (DevOps executes 2h)

- Monitor in Slack #deployments
- Ask: "Rules live? Functions live? Hosting live?"

### Post-Deploy (QA executes 30 min smoke tests)

- Watch for "Smoke Tests PASS/FAIL" signal
- If FAIL: decide rollback immediately (see DevOps guide)

### Next 24h

- Spot-check 3 users in production
- Monitor Cloud Logs (script running in background)
- Archive v1.3 milestone docs

---

## If NO-GO

### Immediately

1. **Stop deployment** (don't deploy any layer yet)
2. **File issue** with RED findings
3. **Root cause analysis**: CTO + DevOps + Security review
4. **Remediation**: fix, re-test, re-approve

### Timeline

- Bug fix: 1–4 hours typical
- Re-audit: 30 min
- Retry approval: same 15-min checklist
