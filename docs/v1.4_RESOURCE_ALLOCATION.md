# v1.4 RESOURCE ALLOCATION PLAN
## Definitive Staffing Strategy for 22-Week Execution

---

## EXECUTIVE SUMMARY

**v1.4 spans 22 weeks (154 days) across 4 autonomous streams (A–D)**, executing 15 sequential + parallel phases. Total resource envelope: **70–80 FTE-weeks** (fluctuates per phase).

**Key staffing decisions:**
1. **4 stream leads** (A–D) coordinate phases; CTO provides architecture + sign-off
2. **Wave-based execution** (Wave 1: planning; Wave 2: CAPA closure + portal; Wave 3: RDC Part 2 + IA; Wave 4: audit + launch)
3. **Peak FTE: 7–8** (Weeks 9–12, parallel streams A–D concurrent); **Trough: 1–2** (final week)
4. **Skill matrix:** frontend, backend, DevOps, QA, compliance engineering roles
5. **Conflict detection:** flag same person assigned to 2 overlapping phases
6. **Buffer time:** 20% slack per stream (holidays, training, unplanned work)
7. **On-call rotation:** 24h deployment window + 48h post-deploy monitoring (Weeks 16–22)
8. **Auditor availability sync:** blocks CTO/RT Fridays 2h × 8 weeks (Phases 10–15)
