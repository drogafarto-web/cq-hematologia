### v1.3 Incident — HMAC Remediation (2026-05-07)

**Timeline:** `HCQ_SIGNATURE_HMAC_KEY` remained in placeholder `PENDING_SET_*` status for 15 days (2026-04-22 → 2026-05-07) post-v1.2 → v1.3 migration. This allowed ~25 functions to crash silently during audit trail chain validation, creating a 15-day window with **zero chain integrity verification**.

**Remediation Deployed (same day):**
- [ADR-0017](docs/adr/ADR-0017-hmac-baseline-reset-2026-05-07.md): Baseline reset with new 256-bit HMAC key; synthetic event `chain-baseline-reset` inserted into `auditLogs` for full disclosure/compliance with RDC 786 Art. 21, RDC 978 Art. 5.3, Lei 13.787 audit trail requirements
- [ADR-0018](docs/adr/ADR-0018-deploy-gate-secret-status-check.md): Mandatory pre-deploy gate `scripts/preflight-secrets-check.sh` blocks `firebase deploy --only functions` if any secret is `PENDING_SET_*` or empty
- 5 Firestore indexes hot-fixed (OOM prevention); 8 functions memory-bumped (256MB → 512MB)
- 11 orphaned callables surfaced and wired into production

**Audit Trail Continuity:**  
The baseline reset is transparent in the code — historical events retain a `pre-baseline-reset: true` flag. The read path (auditor verification) is unaffected. This approach provides **legal defensibility** (transparent disclosure) while enabling **operational continuity** (no retrospective re-signing required).

## Appendix A: v1.3 Architecture Overview

HC Quality is a **monolithic modular SPA** (Single Page Application) backed by **Firebase Firestore** (auth + data) and **Cloud Functions** (business logic).

### Tech Stack

| Layer | Technology | Version |
|-------|-----------|---------|
| Frontend | React 19 + TypeScript 5.8 | Vite 6, PWA plugin |
| State Management | Zustand 5 | Global auth store |
| UI Framework | Tailwind CSS 3.4 | dark-first design |
| Backend | Firebase Admin SDK 12 | Firestore + Auth + Functions |
| IA/ML | Gemini 2.5 Flash | Vision API (Phase 11) |
| Notifications | Twilio SDK | SMS + SLA tracking (Phase 6) |
| Validation | Zod 3 | Payload validation |

### Key Design Decisions (Locked)

1. **Multi-tenant:** Single-lab mode (v1.3, v1.4). Multi-tenant v2 deferred to v1.5
2. **Audit trail:** HMAC-SHA256 chainHash with baseline reset (ADR-0017)
3. **Signature:** LogicalSignature { hash, operatorId, ts } — not PKI cert (cost/complexity tradeoff)
4. **Callables:** All regulatory writes (v1.4+) via Cloud Function callable (not client-direct)
5. **Rules:** Strict by default — read requires active lab membership; write requires role-based gates

### Security Posture

- Firestore rules enforce field-level access control
- All audit writes append-only (no updates/deletes)
- Secrets management via Google Cloud Secret Manager (with rotation)
- Deploy gates: pre-deploy secret status check (ADR-0018)
- No hardcoded credentials; env-var injection only
