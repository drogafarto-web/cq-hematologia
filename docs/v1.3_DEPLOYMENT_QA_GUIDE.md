## Scenario 1: Bioquímica CIQ (10 min)

**Path:** `/bioquimica` → Create lot → Record run → View chart → Verify signature

### Step 1.1: Load Module
- [ ] Navigate to **https://hmatologia2.web.app/bioquimica**
- [ ] Expected: Page loads in <3s, header visible, "Lotes de Controle" title shown
- [ ] DevTools → Network: No 404 or 5xx errors
- [ ] **If 404 or blank:** FAIL — escalate

### Step 1.2: Create QC Lot
- [ ] Click **"Novo Lote"** button
- [ ] Select material: **"Glucose (Control)"**
- [ ] Enter lot number: `LOT-TEST-001`
- [ ] Click **"Criar"**
- [ ] Expected: Toast notification "Lote criado", lot appears in list
- [ ] **If error:** Screenshot, FAIL

### Step 1.3: Record QC Run
- [ ] Click into newly created lot
- [ ] Click **"Registrar Corrida"**
- [ ] Equipment: Select any (e.g., "Yumizen H550")
- [ ] Result value: `100` (glucose mg/dL)
- [ ] Material lot: Select the control material
- [ ] Click **"Salvar"**
- [ ] Expected: Run recorded, assinatura (chainHash) visible in run details
- [ ] **If no chainHash:** FAIL (signature generation failed)

### Step 1.4: View Levey-Jennings Chart
- [ ] Scroll down to chart section
- [ ] Expected: Chart renders in <3s with:
  - X-axis: run sequence
  - Y-axis: glucose results
  - Control limits: ±2SD, ±3SD lines visible
  - Current point plotted
- [ ] **If chart blank or timeout:** FAIL

### Step 1.5: Verify Signature
- [ ] In run details, check **"Assinatura"** field
- [ ] Expected: 64-character hex string (SHA256 hash)
- [ ] Format check: `^[a-f0-9]{64}$` (hex, 64 chars)
- [ ] **If missing or wrong format:** FAIL (security violation)

### Step 1.6: Console Check
- [ ] DevTools → Console tab
- [ ] Expected: 0 red ERROR messages
- [ ] Yellow warnings OK
- [ ] **If red errors visible:** Log them, note as FAIL

### Result: Scenario 1
- [ ] ✅ PASS (all steps completed, 0 errors)
- [ ] ❌ FAIL (any step failed, screenshot and escalate)

## Troubleshooting

### Scenario 1: Chart Doesn't Load
- Clear browser cache: **Ctrl+Shift+Delete**
- Hard reload: **Ctrl+F5**
- If still blank: Check Cloud Logs for Gemini API errors (SecuritySign-Off documents)

### Scenario 2: OAuth Popup Blocked
- Allow popups for hmatologia2.web.app in browser settings
- If popup still doesn't open: Check browser console for OAuth errors
- Contact DevOps if DRIVE_OAUTH_CLIENT_ID missing

### Scenario 2: Import Fails
- Verify Google account has Drive files in target folder
- Check browser console for `403 Forbidden` (permission issue)
- Contact DevOps if OAuth credentials need refresh

### Any Module: 404 Error
- Check URL spelling (case-sensitive)
- Refresh browser
- If still 404: Check `firebase.json` rewrites (DevOps responsibility)

### Any Module: 5xx Error
- Wait 30s (services may still be warming up)
- Refresh
- Check Cloud Logs for function errors
- Escalate to DevOps
