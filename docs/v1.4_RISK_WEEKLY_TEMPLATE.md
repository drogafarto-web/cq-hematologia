### 2. Trigger Events Fired This Week (10 min)

**Did ANY of these happen?**

- [ ] RISK-402: Auditor RFI cycle exceeded 5 business days
- [ ] RISK-406: Schema bug detected OR rules test failed
- [ ] RISK-407: ≥2 CAPAs missing evidence
- [ ] RISK-408: Bug rate >3/phase OR engineer reports >20% context switch
- [ ] RISK-411: Accuracy <85% OR false negative on critical disease
- [ ] RISK-416: Customer/sales escalates multi-tenant demand
- [ ] RISK-404: Orphan DICQ count >5
- [ ] RISK-412: Customer requests LIS sync
- [ ] RISK-413: Lighthouse CI fails OR metric regresses >10%
- [ ] RISK-418: Any cloud service >80% monthly budget

**If ANY box is checked:**

```
Triggered Risk: [RISK-XXX]
Date/Time: [when did it happen]
Evidence: [link to log, screenshot, email thread]
Impact: [phase delay days, cost impact, scope impact]
Response: [mitigation step activated, owner, timeline]
Escalation: [to whom, when]
```

**Example:**
```
Triggered Risk: RISK-402
Date/Time: 2026-05-20 Tuesday 14:00 BRT
Evidence: Email from auditor (auditor@lab.com) saying RFI response delayed
Impact: CAPA-001 closure delayed 2 days; Phase 4 Week 1 at risk
Response: QA Lead contacted lab director; expedited review initiated
Escalation: CTO notified; Phase 4 contingency window activated
```

### 3. New Risks Identified (10 min)

**Brainstorm:** Did any NEW risks emerge this week that aren't on the register?

**If YES, propose as:**

```
RISK-NEW-XXX (Proposed):
  Title: [≤8 words]
  Description: [context, trigger, impact]
  Probability: [1–3] | Impact: [1–3] | Score: [P×I]
  Owner: [assign]
  Mitigation: [proposed steps]
  
  Decision: ☐ Accept (add to active register)
           ☐ Monitor (watch for 2 weeks, re-assess)
           ☐ Defer (add to v1.5 register)
           ☐ Close (not a v1.4 risk)
```

**Example (Week 4):**
```
RISK-NEW-601:
  Title: Database index creation timeout on staging
  Description: Phase 2 Firestore schema changes trigger auto-index creation. Staging rebuild took 8h; prod may take 12h+. Impacts Phase 2 deploy window + v1.3 downtime.
  Probability: 2 | Impact: 2 | Score: 4
  Owner: Stream D (DevOps)
  Mitigation: Pre-create indexes manually on prod before schema deploy; increase deploy window to 24h; monitor index creation logs
  
  Decision: ☑ Accept (add to Phase 2 deploy checklist)
```

## Pre-Meeting Preparation (Sunday EOD)

**Checklist for QA Lead (due by Sunday 23:59):**

- [ ] Read all Top 10 risk descriptions (Section 3 of main doc)
- [ ] Check if any trigger events fired this week (emails, logs, Slack)
- [ ] Poll stream leads for status updates (informal Slack poll OK)
- [ ] Fill in "Last Action" + "Next Action" columns in status table
- [ ] Identify any new risks (brainstorm with QA team)
- [ ] Prepare agenda doc (copy template below, fill in specific week)
- [ ] Send agenda to attendees by Monday 06:00 BRT (give 3h notice)

**Pre-meeting Agenda Template:**

```markdown
## Weekly Risk Review — Week X, Phase Y (2026-MM-DD)

**Attendees:** CTO, QA Lead, Stream A/B/C/D Leads
**Time:** Monday 09:30–10:00 BRT (30 min)
**Location:** [Zoom link / Slack huddle]

### Status Summary (Incoming)
- Active risks: 18
- Mitigated risks: 2
- Closed risks: 0
- 🔴 Escalations this week: [0 / N]
- ⏳ Action items due: [list key deadlines]

### Agenda
1. [5 min] Status updates — Top 10 risk owners report last action + next step
2. [10 min] Trigger events — Did any risks fire? (review evidence + response)
3. [10 min] New risks — Any emerging issues identified by stream leads?
4. [5 min] Closures — Any risks ready to close? (propose + approve)

### Key Dates This Week
- [If Phase 3 kickoff] Schema review audit scheduled
- [If Phase 4 active] CAPA evidence checklist tracking (weekly)
- [If Week 1] Stream allocation doc due by Friday EOD
- [Etc.]
```

## Red Flags (Escalate Immediately if ANY occur)

🚩 **RISK-402:** Auditor cancels 2+ times in 4 weeks  
🚩 **RISK-406:** Rules test fails OR schema change breaks prod  
🚩 **RISK-407:** ≥3 CAPAs incomplete by Phase 4 Week 7  
🚩 **RISK-408:** Bug count >5 in any phase  
🚩 **RISK-411:** IA accuracy <80% OR patient safety incident  
🚩 **RISK-416:** Customer escalates to CEO OR lost deal >$50K  
🚩 **RISK-413:** Lighthouse CI blocks deploy OR bundle >400KB  
🚩 **RISK-418:** Cloud spending >hard limit in same month  
🚩 **ANY RISK:** Trigger fires 2x in same week → escalate to CTO immediately
