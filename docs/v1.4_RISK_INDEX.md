## How to Use This Document Set

### Daily Operations

1. **Print v1.4_RISK_QUICK_REFERENCE.md** → keep on desk
   - Check red flags daily
   - Reference escalation thresholds
   - Bookmark section 6 (triggers + timeline)

2. **Track cost weekly (Friday)**
   - Stream D updates v1.4_RISK_DASHBOARD.json
   - Check Gemini/Twilio/Firebase vs budgets ($500/$200/$300)
   - Alert CTO if any service >80% cap

### Weekly Review (Every Monday, 09:30 BRT)

1. **Sunday (Pre-meeting)**
   - QA Lead: Read v1.4_RISK_MITIGATION_TRACKING.md Section 3 (Top 10)
   - QA Lead: Copy template from v1.4_RISK_WEEKLY_TEMPLATE.md
   - QA Lead: Check if any triggers fired (email logs, Slack)
   - Stream leads: Send status updates (via Slack)
   - QA Lead: Send agenda by Monday 06:00 BRT

2. **Monday (Meeting)**
   - Use v1.4_RISK_WEEKLY_TEMPLATE.md agenda (30 min)
   - Fill in status table, trigger events, new risks, closures
   - Escalations decided in meeting

3. **Monday (Post-meeting, <4h)**
   - Update v1.4_RISK_MITIGATION_TRACKING.md with decisions
   - Email escalation notifications (if any)
   - Post summary to #v1.4-risks Slack

### Phase Planning

1. **Before Phase X kickoff:**
   - CTO: Read v1.4_RISK_MITIGATION_TRACKING.md Section 7 (Historical Log)
   - Tech Lead: Read Section 3 (Top 10 Risks) for phase-specific risks
   - Stream Leads: Review risks assigned to their phase

2. **During Phase X:**
   - Weekly Monday review (track status)
   - Daily standup: mention any trigger risks
   - Friday: Stream D cost report → dashboard update

3. **After Phase X:**
   - Monthly summary (v1.4_RISK_WEEKLY_TEMPLATE.md template)
   - Post-incident analysis (if any risk triggered)
   - Update historical log with lessons learned

### Auditor Interaction

1. **Pre-Alignment Call (Week 1):**
   - Reference RISK-402 mitigation (auditor pre-call scheduled)
   - Show v1.4-ASSUMPTIONS-AND-CONSTRAINTS.md (to publish)
   - Confirm RFI SLA <5 business days

2. **Phase 4 CAPA Review (Weeks 4–7):**
   - Weekly evidence checklist (RISK-407 template)
   - CAPA-CLOSURE-REPORT draft (Week 4 feedback)
   - Escalation path for RFI delays >5 days

3. **Phase 13 Final Audit:**
   - v1.4-DICQ-COVERAGE-MATRIX.md (all orphaned items resolved)
   - Pre-audit dry run (Week 17 Phase 13 Week 1)
   - Escalation: >5 P0 gaps → CTO immediate

## Decision Rights & Escalation Paths

### Quick Escalation Guide

**🔴 Critical (Score 7–9)** → CTO

- Immediate: Slack DM + email
- Action: War room <4h, daily standup until score ≤6
- Examples: None currently (RISK-401, 403 mitigated)

**🟡 High (Score 4–6)** → QA Lead

- Timeline: <48h notification
- Action: Owner remediation plan, weekly tracking
- Examples: RISK-402, 406, 407, 411, 416, 418

**🟢 Low (Score 1–3)** → Phase Lead

- Timeline: Weekly acknowledgment
- Action: Standard engineering, escalate if trigger fires
- Examples: RISK-405, 410, 414, 415, 417, 419

**🚨 Auto-Escalate to CTO (Regardless of Score):**

- Patient safety incident
- Data exposure
- RDC 978 / DICQ breach
- Phase delay ≥5 days
- Budget overage ≥$200/month
- Security finding (CVE ≥6)
