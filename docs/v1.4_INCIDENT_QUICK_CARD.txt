7 CRITICAL SCENARIOS — IN ORDER OF APPEARANCE
─────────────────────────────────────────────────────────────────────────────

[1] RULES DENY USERS (RDC Lockout)
    Detection:  "Permission denied" errors in Cloud Logs (>10 in 15 min)
    Decision:   All users blocked? → ROLLBACK. Subset? → FIX & MONITOR.
    Action:     Run: gcloud logging read 'text contains "Permission denied"' \
                     --project=hmatologia2 --limit=50
    Fix:        See firestore.rules, deploy: firebase deploy --only firestore:rules
    Rollback:   git checkout HEAD~1 -- firestore.rules && firebase deploy --only firestore:rules

[2] CLOUD FUNCTIONS 5xx (Function Crash)
    Detection:  httpStatus=500+ in Cloud Logs (>50 errors in 10 min)
    Decision:   Single function or all? Check function name in logs.
    Action:     Rollback single: firebase deploy --only functions:<name>
                Rollback all:    firebase deploy --only functions
    Verify:     Tail logs for 5 min: gcloud logging read 'severity >= ERROR' --follow

[3] FIRESTORE QUOTA BREACH (Read/Write Spike)
    Detection:  "Request rate exceeded" OR quota >95% in GCP Console
    Decision:   Expected spike? → Disable feature. Unexpected? → FIX CODE.
    Action:     Identify culprit: grep -r "onSnapshot\|polling" src/features/<MODULE>
                Check polling interval >=30s, query has .limit()
    Fix:        Reduce interval, add rate limiting, redeploy: firebase deploy --only hosting

[4] NOTIVISA API TIMEOUT (Gov Endpoint Down)
    Detection:  "NOTIVISA" in Cloud Logs + "deadline exceeded"
    Decision:   ANVISA down? → QUEUE for retry. Code issue? → FIX.
    Action:     Check: https://notivisa.anvisa.gov.br/status
                Queue: gcloud tasks queues create notivisa-retry-queue
    Note:       This is NOT a blocker — notifications will retry automatically.

[5] GEMINI VISION API COST SPIKE (Phase 11 IA)
    Detection:  Gemini tokens >10x baseline in logs, daily cost >$100
    Decision:   This is a P2 cost issue, not availability.
    Action:     Reduce confidence threshold in analyzer, redeploy functions
                If critical: DISABLE_PHASE_11_IA = false in code

[6] SMS DELIVERY FAILURE (Twilio Down)
    Detection:  "Twilio" + "ERROR" in Cloud Logs (>10 failures in 15 min)
    Decision:   Twilio down? → FALLBACK to email. Code issue? → FIX.
    Action:     Check: https://status.twilio.com
                Fallback: Add email notification in sendNotification function
    Note:       Lab notifications will work via email — not a data blocker.

[7] CRITICAL DATA LOSS (Accidental Delete)
    Detection:  RT reports: "Document was deleted accidentally"
    Decision:   EMERGENCY. Restore from backup immediately.
    Action:     Identify when: gcloud firestore query (check audit trail)
                Restore: gcloud firestore restore --backup-id=<ID>
    Verify:     Query restored doc: gcloud firestore documents get /path/{id}

[8] CASCADING FAILURE (Multiple Systems Down) — FULL ROLLBACK
    Detection:  >2 scenarios simultaneous OR >1% of requests 5xx
    Decision:   DECLARE INCIDENT. Full rollback to v1.3.
    Action:     git checkout <v1.3-commit> -- firestore.rules functions/ src/
                firebase deploy --only firestore:rules,functions,hosting
    Verify:     Tail logs for 5 min — should be clean
