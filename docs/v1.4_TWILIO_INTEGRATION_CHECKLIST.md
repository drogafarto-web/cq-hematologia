## Appendix A: Quick Reference — Twilio API Codes

| Error Code | Meaning                        | Action                                            |
| ---------- | ------------------------------ | ------------------------------------------------- |
| `20003`    | Invalid phone number format    | Check number includes country code (+55)          |
| `21609`    | Invalid phone number           | SMS cannot be sent to this carrier in Brazil      |
| `20429`    | Rate limit exceeded            | Backoff & retry (exponential: 2s → 10s → 60s)     |
| `21610`    | Account not authorized for SMS | Upgrade from trial to production (Twilio support) |
| `11200`    | HTTP client error (400–499)    | Check request body schema, token auth             |
| `11201`    | HTTP server error (500–599)    | Twilio service issue, retry later                 |

## Appendix B: Firestore Schema — SMS Audit Log

```typescript
// /labs/{labId}/sms-audit-log/{docId}
interface SMSAuditLog {
  messageId: string; // Twilio message SID
  recipientPhone: string; // Plain phone number (logged only during send)
  context: 'shift_change' | 'deploy_alert' | 'patient_portal';
  status: 'queued' | 'sent' | 'delivered' | 'failed' | 'undelivered';
  errorCode?: string; // Twilio error code (if failed)
  errorMessage?: string; // Human-readable error (if failed)
  body: string; // SMS body text
  costBRL: number; // Cost in Brazilian Reals (0 if sandbox)
  timestamp: Timestamp; // When SMS was sent
  webhookReceivedAt?: Timestamp; // When delivery status webhook received
  operatorId: string; // Who triggered the SMS (auth.uid)
}
```
