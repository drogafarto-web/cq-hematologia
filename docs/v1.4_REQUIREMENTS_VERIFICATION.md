## SECTION 1: REQ-BY-REQ VERIFICATION TABLE

This table is the **master reference**. Each row is a requirement with its full lineage.

### Legend
- **REQ-ID:** User-story requirement (REQ-401–415) or technical debt (TD-401–405)
- **Phase:** ROADMAP phase (1–15) where REQ closes. Multi-phase REQs listed with (Phase A → Phase B)
- **Module(s):** Which feature modules affected
- **Stream:** A (Compliance/CIQ), B (Portals), C (IA Foundation), D (DevOps/Perf)
- **Artifact(s):** Deliverable (code, schema, CF, test, doc)
- **Status:** DRAFT (not started), IN PROGRESS, COMPLETE
- **Success Criteria:** Quantifiable acceptance criteria (from REQUIREMENTS.md)
- **DICQ Block(s):** Which DICQ 4.X blocks satisfied
- **RDC Article(s):** Which RDC 978 / RDC 222 / RDC 986 articles satisfied
- **Owner:** Agent or role assigned
- **Effort:** Story points (s = small, m = medium, l = large)
