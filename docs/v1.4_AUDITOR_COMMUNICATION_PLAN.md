# v1.4 Auditor Communication Plan

**Version:** 1.0  
**Date Created:** 2026-05-07  
**Status:** ACTIVE (May 2026 – October 2026)  
**Audience:** External Auditor (Clinical Lab Compliance)  
**Scope:** Full v1.4 milestone (Phases 4–15) leading to 2026-08-31 external audit

---

## Executive Summary

This document defines the cadence, mechanics, and templates for continuous alignment with the external auditor throughout v1.4 execution. The goal is **zero surprises at audit time** through structured, proactive communication.

**Key Principles:**

- Auditor is informed **weekly**, not at phase end
- RFI (Request for Information) is **async-first**, with documented response time SLAs
- Phase gates require auditor pre-approval or explicit deferral
- Evidence packages are **assembled concurrently**, not retrofitted
- External audit (target: 2026-10-15) has 6-week buffer post-launch for CAPA closure

## 2. RFI Workflow (Request for Information)

### Process Flow

**Trigger:** Auditor submits written question via email (subject: `[RFI] [Phase] [Topic]`)

```
┌─────────────────────────────────────────────────────┐
│ Auditor emails RFI                                  │
│ Subject: "[RFI] Phase 5 | Critical Value Escalation"│
└────────┬────────────────────────────────────────────┘
         │
         ├─→ Auto-reply (same day): ACK + ref number (RFI-001)
         │
         ├─→ CTO logs in shared tracker (Google Sheet)
         │
         ├─→ Owner assigned (within 24 hours)
         │
         ├─→ Response drafted (within 5 business days)
         │
         ├─→ Evidence gathered (test results, screenshots, logs, code)
         │
         ├─→ CTO reviews + approves
         │
         └─→ Email sent to auditor (within 5 bdays) with:
             - Answer (1–2 paragraphs)
             - Evidence (attachments + links)
             - Follow-up Q: "Does this satisfy your concern?"
             - Next step (if evidence requires audit verification)
```

### RFI Response Template

```
Subject: RE: [RFI-###] [Phase] [Topic] — Response

Hi [Auditor Name],

**RFI Reference:** RFI-###
**Question:** [Restate the auditor's question clearly]

**Response:**
[1–2 paragraph answer with specific details + evidence references]

**Evidence Attached:**
- Document 1: [description]
- Document 2: [description]
- GitHub link: [commit SHA + test file]
- Cloud Logs screenshot: [timestamp]
- Test report: [reference]

**Verification Opportunity:**
Would you like to [review the code live | watch a demo | run a test scenario] to validate this?

**Follow-Up:**
Does this fully address your concern, or are there additional gaps to investigate?

Best regards,
[CTO Name]

---
Auditor Communication Tracker: [Google Sheet link]
```

### SLA for RFI Response

| Category                  | SLA                        | Notes                            |
| ------------------------- | -------------------------- | -------------------------------- |
| **Acknowledgment**        | Same business day          | Auto-reply email                 |
| **Initial response**      | 5 business days            | Answer + initial evidence        |
| **Follow-up evidence**    | +3 business days           | If deeper investigation needed   |
| **Live demo/audit**       | +5 business days           | Scheduled after initial response |
| **Escalation** (if stuck) | CTO override gate (see §7) | If owner unavailable >7 days     |

## 7. Escalation Path (If Auditor Unavailable >7 Days)

### Scenario: Auditor Unresponsive to RFI or Pre-Phase Approval

**Escalation Chain:**

```
Day 1–5: Standard RFI/approval process (no response yet)
         ↓
Day 6–7: CTO sends escalation email (subject: "[URGENT] RFI-### Follow-Up")
         - References original request + deadline
         - Proposes default action if no response by [DATE]
         ↓
Day 8: CTO invoices escalation override gate
       - Example: "If no auditor response by EOD [DATE],
         we will proceed with Phase kickoff as-is, with
         auditor catch-up call scheduled for [DATE]"
       ↓
Day 9+: CTO override executes (if still no response)
        - Phase proceeds as planned
        - Auditor notified of override + reasons
        - Catch-up call scheduled within 48h
        - All evidence retroactively reviewed
```

### CTO Override Email Template

```
Subject: ESCALATION | [RFI-###] CTO Override Gate — Phase [X] Proceeding

Hi [Auditor Name],

**Original RFI:** [DATE] — no response after 7 business days.

**CTO Override Decision:** Phase [X] kickoff proceeding [DATE] as originally planned.

**Reason:** RFI must be resolved within 5 bdays per SLA. Deferral would delay entire roadmap.

**What happens next:**
1. Phase [X] launches [DATE] as scheduled
2. All RFI evidence + auditor catch-up call scheduled [DATE] [TIME]
3. You will review all Phase [X] evidence (expedited package sent tomorrow)
4. Any concerns retroactively addressed in Phase [X+1]

**Your availability request:** Please confirm by [DATE] [TIME] if catch-up call [DATE] works.

This escalation is a last resort (first occurrence in v1.4). We value your input and aim for real-time alignment.

Contact me directly if there's a barrier to communication.

Best,
[CTO Name]
```

### Escalation Approval (CTO Solo Authority)

- **Trigger:** Auditor >7 days unresponsive to RFI or pre-phase approval
- **CTO authority:** Full override (proceed with phase as-is)
- **Notification:** Auditor copy on all override emails
- **Recovery:** 48h catch-up call + expedited evidence package
- **Post-audit impact:** Any override is disclosed in audit findings (transparency)

## 10. Failure Modes & Backups

### If Weekly Call Misses (Auditor or CTO)

- **Backup:** Email status (Template A above) sent same day
- **Rescheduling:** Next call bumped 1 week forward (Friday 10:00 BRT)
- **Recording:** If call is async (email-based), store in shared folder with timestamp

### If RFI Response Exceeds 5 bdays

- **Day 6:** CTO sends progress email (not final answer, but roadmap to answer)
- **Day 7–9:** CTO follow-up with ETA + interim evidence
- **Day 10:** Escalation gate activated (see §7)

### If Auditor Rejects Phase Scope

- **Response timeline:** Within 2 business days of auditor's objection
- **Negotiation:** Call to discuss alternatives (defer features, split phase, etc.)
- **Resolution:** Either scope revised + re-approved, OR phase deferred to next in roadmap
- **Impact:** Logged in CAPA register + road map adjusted + timeline recalculated

### If Production Issue Found During Audit (v1.3 or v1.4)

- **Tier 1 (High):** CTO notified immediately | Escalation email within 1 hour
- **Tier 2 (Medium):** RFI sent normally | 5-day response SLA
- **Tier 3 (Low):** Logged in CAPA register | No immediate response required
