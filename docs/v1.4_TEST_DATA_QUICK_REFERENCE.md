**TL;DR:** Fast lookup for test data locations, reset procedures, and mock user credentials.

## Quick Commands

### Load Test Data for a Phase

```bash
# Phase 4 (Portal + NOTIVISA)
node test/utils/load-fixtures.mjs --phase 4 --firebase-project hmatologia2-staging

# Phase 5 (Critical Values + IA)
node test/utils/load-fixtures.mjs --phase 5 --firebase-project hmatologia2-staging

# All phases
node test/utils/load-fixtures.mjs --all --firebase-project hmatologia2-staging
```

### Reset Test Data (Delete + Reload)

```bash
# Reset single phase (full reset)
node test/utils/reset-staging.mjs --phase 4 --firebase-project hmatologia2-staging

# Reset with backup first
node test/utils/reset-staging.mjs --phase 4 --backup-first --firebase-project hmatologia2-staging

# Quick reset (emulator only, no backup)
firebase emulators:exec --project hmatologia2 'npm run test:load-fixtures -- --phase 4'
```

### Validate Fixtures Before Deploy

```bash
# Validate single phase
npm run test:validate-fixtures -- --phase 4

# Validate all phases
npm run test:validate-fixtures -- --all

# Detailed validation report
npm run test:validate-fixtures -- --phase 4 --verbose
```

### Create/Delete Test Users

```bash
# Create all users for Phase 4
node test/utils/create-test-users.mjs --phase 4 --firebase-project hmatologia2-staging

# Delete all users for Phase 4
node test/utils/delete-test-users.mjs --phase 4 --firebase-project hmatologia2-staging

# List active test users
node test/utils/list-test-users.mjs --phase 4 --firebase-project hmatologia2-staging
```

## Troubleshooting Quick Fixes

### Fixture File Not Found

```bash
# List available fixtures
ls -la test/fixtures/phase-4/

# Check cwd (must be project root)
pwd  # Should be /path/to/hc-quality

# Reload from backup
node test/utils/reset-staging.mjs --phase 4 --firebase-project hmatologia2-staging
```

### Test Users Not Logging In

```bash
# Verify users exist
node test/utils/list-test-users.mjs --phase 4 --firebase-project hmatologia2-staging

# Recreate users
node test/utils/delete-test-users.mjs --phase 4 --firebase-project hmatologia2-staging
node test/utils/create-test-users.mjs --phase 4 --firebase-project hmatologia2-staging

# Check .env.test for password (should be Test@2026!4)
grep "TEST_PASSWORD" .env.test
```

### Laudos Not Visible in Portal

```bash
# Check data was loaded
firebase firestore:query --project hmatologia2-staging /labs/lab-001-staging/laudos

# Verify status = PUBLISHED
npm run test:validate-fixtures -- --phase 4 --verbose

# Reset and reload
node test/utils/reset-staging.mjs --phase 4 --firebase-project hmatologia2-staging
```

### Critical Value SMS Not Sent

```bash
# Check Twilio sandbox is enabled
grep "TWILIO_" .env.test

# Verify phone number in config (must be E.164 format)
cat test/fixtures/phase-5/criticos-config.json | grep smsRecipients

# Check escalation logs
firebase functions:log --project hmatologia2-staging | grep "escalation"
```

### Import Stuck on Invalid Records

```bash
# Show invalid records
node test/utils/analyze-legacy-import.mjs --invalid-only

# Skip invalid, keep valid
node test/utils/reset-staging.mjs --phase 9 --skip-invalid-legacy

# Re-run validation
npm run test:phase-9 -- --legacy-import
```
