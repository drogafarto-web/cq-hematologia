## Critical Filters

**Copy + paste into Cloud Console or use with `gcloud logging read`:**

### 1. All Errors (Last Hour)
```
severity >= ERROR AND timestamp > now - 60m
```

### 2. Cloud Functions Only
```
resource.type="cloud_function" AND severity >= ERROR
```

### 3. Firestore Permission Errors
```
resource.type="cloud_firestore" AND textPayload=~".*Permission.*"
```

### 4. Hosting 5xx Errors
```
resource.type="cloud_run" AND httpRequest.status >= 500
```

### 5. Function Timeouts
```
resource.type="cloud_function" AND textPayload=~".*Exceeded timeout.*"
```

## What to Watch For (Red Flags)

| Log Signature | Severity | Action |
|---|---|---|
| `"Exceeded timeout of X seconds"` | 🔴 BLOCK | Check async handlers; increase timeout or split work |
| `"Permission denied"` on `/labs/{labId}/*` | 🔴 BLOCK | Firestore rules regression; compare to git HEAD |
| HTTP 502/503 sustained >5 min | 🔴 BLOCK | Hosting/runtime failure; consider rollback |
| `"undefined is not a function"` | 🔴 BLOCK | Missing dependency; check `functions/package.json` |
| `"Request rate exceeded"` (Firestore) | 🟡 OK | Expected during load; auto-backoff handles |
| Cold-start latency on first invocation | 🟡 OK | Expected; not an error |

## gcloud Command Cheatsheet

```bash
# List all errors (last 24h)
gcloud logging read "severity >= ERROR" \
  --project=hmatologia2 \
  --limit=100

# Real-time tail (Ctrl+C to stop)
gcloud logging read "severity >= ERROR" \
  --project=hmatologia2 \
  --follow

# Export to JSON file
gcloud logging read "severity >= ERROR" \
  --project=hmatologia2 \
  --limit=500 \
  --format=json > errors.json

# Count errors per function
gcloud logging read "resource.type='cloud_function' AND severity >= ERROR" \
  --project=hmatologia2 \
  --format=json | jq -r '.[] | .labels.function_name' | sort | uniq -c
```
